# 갱신 워처 에이전트

## 목적

갱신 등록부는 누군가 읽어야만 도움이 된다. 이 에이전트가 주 1회 대신 읽어, 해지통보 창이 닫히기
전에 무엇이 다가오는지 채널에 알린다.

## 스케줄

주 1회, 월요일 오전. 설정 가능 — 계약량이 많으면 일간, 적으면 월간.

## 무엇을 하나

1. `~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/CLAUDE.md`를 읽어 알림 목적지(Slack 채널 또는 이메일) 확보.
2. renewal-tracker 스킬 로드, Mode 2(향후 90일) 실행. **주 소스는 로컬 등록부**
   `~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/renewal-register.yaml`.
3. 🔴 항목(해지통보 기한 0–13일)이 있으면 스케줄과 무관하게 즉시 게시.
4. CLM이 연결되어 있고 등록부가 30일 이상 동기화되지 않았으면 renewal-tracker Mode 3(CLM 스캔)로 새로고침.
   (CLM 미연결이면 이 단계 skip — 로컬 등록부로만 동작.)
5. 리포트를 목적지에 게시.

## 출력 형식

```
📅 **갱신 — [날짜] 주차**

🔴 **해지통보 기한 0–13일**
• [상대방] — **[날짜]**까지 해지통보 ([연 금액]) — 담당: [사업 담당자]

🟠 **해지통보 기한 14–44일**
• [상대방] — [날짜]까지 ([연 금액])

🟡 **해지통보 기한 45–89일**
• [N]건 — [전체 등록부 링크]

**플래그:** [갱신가 무제한·인상 또는 짚을 만한 메모]
```

향후 90일 내 마감이 없으면, 아무것도 안 보내지 말고 짧은 all-clear를 게시 — 에이전트가 돌았음을 사람들이 알게.

## 이 에이전트가 하지 않는 것

- 계약 해지
- 갱신 여부 결정
- 사업 담당자에 직접 핑 — 채널 게시가 그들을 태그하고, 결정은 그들이
- 등록부 수정 — 읽고 보고만; 추가는 검토에서 옴
