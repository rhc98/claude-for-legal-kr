# /commercial-legal:review-proposals

플레이북 모니터 에이전트가 쌓아둔 대기 중 플레이북 업데이트 제안을 하나씩 훑고, 승인된 변경을
`~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/CLAUDE.md`(실무 프로파일)에 적용한다.

이 스킬은 **플레이북 모니터 에이전트와 한 쌍**이다. 에이전트가 이탈 로그에서 패턴을 감지해 제안을
초안하고 파일에 쓰면(에이전트 Step 1–4), 이 커맨드가 그 제안을 사용자에게 제시하고 승인/거부/편집/
보류를 받아 적용한다(에이전트 Step 5). 따라서 이 스킬의 흐름·파일 경로·수용/거부/편집/보류 의미·
완료 시 아카이브·약관규제법 무효 normalize caveat는 모두
`~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/agents/playbook-monitor.md`의
**Step 5 — 검토·승인**과 동일하게 유지된다. 둘이 충돌하면 에이전트 Step 5가 정본이다.

## 가드레일과 컨텍스트

실무 수준 `~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/CLAUDE.md`가 정본이다 —
공유 가드레일·플레이북·출처 태그·결정 자세·검증 로그 규칙은 거기서 읽는다. 이 텍스트와 충돌하면
그 파일이 우선한다. 특히:

- **결정 자세.** 어떤 제안이든 약관규제법 무효 가능성·법적 판단을 건드리면, 단정하지 않고 `[검토]`로
  플래그한다(아래 약관규제법 caveat 참조).
- **출처 태그.** 제안 블록에 인용이 들어 있으면 그 출처 태그(`[모델 지식 — 검증 필요]`,
  `[법망]` 등)를 그대로 보존한다 — 검토 과정에서 승격하지 않는다.
- **검증 로그.** 검토 중 사용자가 인용·임계점을 1차 소스로 확인하면
  `~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/verification-log.md`에 기록.

## 매터 컨텍스트

플레이북·실무 프로파일은 **실무 수준**에서 산다 — 이 커맨드는 매터별로 분기하지 않는다. 실무 수준
CLAUDE.md `## 매터 워크스페이스`가 활성이어도 제안은 실무 수준 CLAUDE.md를 갱신한다(매터별 CLAUDE.md가
아님). 매터 워크스페이스 머시너리는 이 커맨드에 적용되지 않는다.

## 지침

1. **플레이북 모니터 에이전트를 로드하고 Step 5(검토·승인 흐름)를 실행한다.** 아래 단계는 그 Step 5의
   사용자 대면 실행이며, 에이전트 정의와 동일한 의미를 갖는다.

2. **제안 파일을 읽는다.** `~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/playbook-proposals.md`.
   파일이 없거나 비었으면:
   > *"대기 중 제안 없음. 플레이북이 최신입니다."*

   여기서 중단. 더 진행하지 않는다. (제안은 플레이북 모니터 에이전트가 임계점을 넘은 이탈 패턴을
   감지했을 때만 생긴다 — 제안이 없다는 것은 플레이북이 실무와 어긋나지 않았다는 신호다.)

3. **제안을 하나씩 제시한다.** 각 제안마다 에이전트 Step 3에서 초안된 전체 제안 블록(패턴·현재 문구·
   제안 새 문구·근거 데이터·권고)을 그대로 보여주고, 네 가지 선택지를 제시한다:

   ```
   제안 [N] / [전체]: [조항명]

   [전체 제안 블록 — 패턴 / 현재 플레이북 문구 / 제안 새 문구 / 근거 거래 / 권고]

   무엇을 하시겠습니까?
   [A] 수용 — 제안 문구를 CLAUDE.md에 적용
   [R] 거부 — 현재 문구 유지
   [E] 편집 — 원하는 문구를 입력
   [D] 보류 — 다음 주기에 다시 알림
   ```

   **권고가 "논의 플래그(Flag for discussion)"인 제안은 그 라벨을 그대로 surface한다.** 에이전트가
   "논의 플래그"를 단 것은 그 이탈 패턴이 단순 사업 양보가 아니라 **약관규제법 무효 위험을
   normalize**하는 신호일 수 있다는 뜻이다(아래 caveat). 이 경우 제안을 "개정 권고"처럼 다루지
   말고, 그 약관규제법 플래그를 사용자에게 명시한 뒤 선택을 받는다.

4. **[A] 수용 — 쓰기 전 정확한 diff를 보여준다.** CLAUDE.md에 적용될 변경을 명시적으로 표시:

   ```
   CLAUDE.md 갱신:

   - [현재 텍스트]
   + [제안 텍스트]

   확인? (예 / 아니오)
   ```

   **사용자가 명시적으로 확인한 후에만** CLAUDE.md에 쓴다. diff 없이, 확인 없이 절대 쓰지 않는다.

5. **[E] 편집 — 사용자가 선호 문구를 입력한다.** 사용자가 준 문구로 diff를 다시 만들어 보여주고,
   같은 방식으로 명시적 확인을 받은 후에만 쓴다.

6. **[R] 거부 / [D] 보류 — CLAUDE.md를 수정하지 않는다.** 결정을 사유와 함께
   `~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/playbook-monitor-log.yaml`에 로그한다.
   - **거부:** 거부일 이후 새 패턴이 생기기 전까지 같은 제안을 다시 제기하지 않는다(에이전트가 그 로그를 읽는다).
   - **보류:** 다음 주기에 다시 알리도록 표시.

7. **모든 제안 해결 후 요약을 보여준다:**

   ```
   검토 완료.
   [N] 수용·적용 | [N] 거부 | [N] 보류 | [N] 편집·적용
   CLAUDE.md 최종 갱신: [timestamp]
   다음 플레이북 점검: [N]건 더 로그된 후
   ```

8. **제안 파일을 아카이브한다.** `playbook-proposals.md`를 `playbook-proposals-[YYYYMMDD].md`로 rename.
   활성 제안 파일은 비워진다 — 다음 플레이북 모니터 실행이 새로 덮어쓴다. (에이전트가 stale 제안을
   누적하지 않고 매 실행마다 활성 파일을 덮어쓰는 것과 짝을 이룬다.)

## 약관규제법 무효 — "논의 플래그" 제안의 normalize caveat

플레이북 모니터 에이전트는 일관되게 수용된 조항이 단순 사업 위험이 아니라 **약관규제법 무효 위험을
normalize**할 수 있을 때 "논의 플래그"를 단다 — 예: 면책조항(약관규제법 §7) 또는 과도한 손해배상액
예정(§8)을 반복 수용하는 패턴은 집행 불가능할 수 있는 조항을 플레이북 표준으로 굳히는 신호일 수 있다.

이런 제안을 검토할 때:

- **무효를 단정하지 않는다.** 무효는 법원·공정위가 내리는 결론이다. 제안을 수용해도 그 조항이
  유효해지지 않고, 거부해도 무효가 확정되지 않는다.
- 제안의 `[검토 — 약관규제법 무효 가능성, vendor-agreement-review로 확인]` 플래그를 사용자에게
  그대로 보여준다. "이 패턴을 플레이북 표준으로 굳히기 전에, 반복 수용 중인 조항이 약관규제법
  §[X]에 걸릴 가능성을 검토하시는 게 좋습니다 — 무효라면 표준화는 집행 불가능한 입장을 코드화하는
  것입니다." `[검토]`
- 사용자가 그래도 수용을 택하면 적용하되, 그 결정을 로그에 약관규제법 플래그와 함께 남긴다. 사용자가
  보류·거부를 택하면 그대로 로그.
- 약관규제법 적용 여부·무효 가능성의 깊은 분석은 이 커맨드가 수행하지 않는다 — `/commercial-legal:vendor-agreement-review`
  또는 검토 스킬의 약관규제법 overlay 패스로 라우팅한다.

## 이 커맨드가 하지 않는 것

- 명시적 건별 diff 확인 없이 CLAUDE.md 수정 — 모든 [A]/[E] 적용은 diff + 명시적 "예" 후에만.
- 제안 생성 — 그건 플레이북 모니터 에이전트(Step 1–4). 이 커맨드는 이미 쓰인 제안만 소비한다.
- 조항의 약관규제법 무효 단정 — "논의 플래그" 제안은 `[검토]` + 라우팅 제안만.
- stale 제안 누적 — 완료 시 활성 파일을 아카이브로 rename.
- 거부된 제안을 새 패턴 전까지 다시 제기.
