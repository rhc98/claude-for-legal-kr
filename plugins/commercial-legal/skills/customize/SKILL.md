# /customize

## 실행 조건

사용자가 `/commercial-legal:customize` 입력. 콜드스타트 인터뷰 전체 재실행 없이, YAML 수동
편집 없이 실무 프로파일의 무엇 — 위험 성향·에스컬레이션 담당자·플레이북 입장·하우스 스타일·
산출물 형식 — 을 변경하고 싶음.

## 할 일

1. **config 읽기.** `~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/CLAUDE.md`
   (와 한 단계 위 `~/.claude/plugins/config/claude-for-legal-kr/company-profile.md`) 읽기.
   플러그인 config 없거나 `[PLACEHOLDER]` 값 남아 있으면:

   > 셋업을 아직 안 하셨습니다. 먼저 `/commercial-legal:cold-start-interview` 실행 —
   > customize는 이미 가진 프로파일을 조정하기 위한 것.

2. **맞춤화 가능 맵 보여주기.** 프로파일에 있는 것을 그룹화, 현재 값의 한 줄 요약과 함께:

   - **회사 / 우리는 누구인가** — 회사명·업종·관할·단계·실무 환경·매도인측/매수인측
     orientation *(모든 12 플러그인 공유 — `company-profile.md` 통해 변경 전파)*
   - **위험 성향** — 보수·중간·적극, 각각의 의미 (폴백 조건·승인 임계점·`[검토]` 바 영향)
   - **인력** — 에스컬레이션 체인, 금액 임계점·조항 유형별 승인권자
   - **플레이북 — 매도인·공급자측** — 책임 한도·면책·개인정보·기간/해지·준거법 입장과 각 폴백
   - **플레이북 — 매수인·구매자측** — 책임 한도·면책·개인정보·기간/해지·준거법 입장과 각 폴백
   - **에스컬레이션** — 승인 매트릭스·금액 임계점·금액 무관 자동 에스컬레이션 트리거
   - **하우스 스타일** — 레드라인 톤·이해관계자 요약·산출물 보관 위치·갱신 알림 목적지
   - **검토 선호** — `confirm_routing` 등 라우팅 동작
   - **NDA 트리아지 선호** — `closing_action` 등 NDA 트리아지 출력 마무리 동작
   - **플레이북 모니터 설정** — playbook-monitor 에이전트의 `pattern_threshold`·`lookback_months`
   - **통합** — CLM(Ironclad·Agiloft)·전자서명·문서 저장소·Slack·**국가법령정보 MCP** 상태·폴백

3. **무엇을 바꿀지 묻기.**

   > 무엇을 조정할까요? 섹션을 고르거나 자기 말로 변경 설명.

4. **변경 수집.** 사용자가 섹션을 고르면 그 섹션의 현재 값을 보여주고 새 값 묻기. 사용자가
   자유 형식으로 설명하면 어느 섹션이 영향인지 식별하고 mapping 확인.

5. **변경 전 검증.** 변경에 법적 인용·임계점·기한이 있으면 국가법령정보 대조 검증:
   ```
   mcp__korean-law__get_law_text "[조항]"
   ```
   인용이 안 맞으면 사용자에게 확인 ("당신은 X라 하셨는데, 국가법령정보의 현행 텍스트는 Y. 어느
   것으로 갈까요?").

6. **변경 적용.** CLAUDE.md의 해당 섹션을 갱신. 변경 전후 diff 보여주기.

   예시:
   - *매도인측 책임 한도 폴백 12개월 → 6개월:* "이제 검토 스킬이 6개월 초과를 이탈로
     플래그합니다; 기존 deal-debrief 기록은 그대로 유지."
   - *새 에스컬레이션 승인권자:* "당신 권한을 초과하는 레드라인은 이제 이 승인권자로
     라우팅 — escalation-flagger가 해당 위험 밴드에서 디폴트로 포함."
   - *위험 성향 중간 → 적극:* "더 벤더 친화적 입장을 플래그 없이 수용하고 `[검토]`
     바를 높입니다."

7. **확인 + 검증 로그 기록.**
   ```
   [YYYY-MM-DD] customize - [성명]이(가) [섹션]에서 [구체적 변경]
   ```
   `~/.claude/plugins/config/claude-for-legal-kr/commercial-legal/verification-log.md`에 추가.

## 흔한 맞춤화 패턴

**위험 성향 변경:** "보수에서 중간으로" → 영향 받는 모든 디폴트 다시 보여주고 사용자
확인 받기 (폴백 조건 수용 범위, 자동 에스컬레이션 임계점 등).

**에스컬레이션 담당자 변경:** 옛 담당자에서 새 담당자로 — 단순 이름 교체. 매트릭스 전체
보여주기.

**플레이북 입장 단 한 줄 변경:** "매수인측 책임 cap 12개월에서 6개월로" — 영향: 모든
미래 계약 검토·평가.

**하우스 스타일 갱신:** "갱신 알림을 이메일에서 Slack 채널로" → 영향 받는 스킬·에이전트
(renewal-watcher 등) 식별.

**국가법령정보 MCP 추가·제거:** `.mcp.json` 직접 편집 안내 또는 자동화.

## 가드레일

- **섹션을 삭제하지 않는다.** 사용자가 "제거"를 원하면 `[미구성]`으로 설정하고 그것이
  플러그인 동작에 무엇을 의미하는지 설명.
- **내부 모순 플래그.** 변경이 프로파일을 모순되게 만들면(위험 성향 적극 + "모든 레드라인
  GC 승인 필요"; 또는 "매도인측" + 매수인측 플레이북 입장), 긴장을 플래그하고 어느 것을
  원하는지 묻기.
- **가드레일 약화 플래그.** 사용자가 가드레일을 끄려 하면(`[검토]` 플래그 제거, 문서 헤더
  생략, `[검증 필요]` 태그 제거), 그 가드레일이 무엇을 막는지 설명하고 trade-off를 이해하는지
  확인. `[검토]` 플래그·출처 attribution 태그·인용 조문의 `[검증 필요]` 태그는 load-bearing이며
  제거하지 않는다.
- **한 번에 하나.** 전체 인터뷰를 다시 묻지 않는다.
