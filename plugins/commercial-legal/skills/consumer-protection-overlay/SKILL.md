# 소비자 보호 overlay 평가

## 매터 컨텍스트

실무 수준 CLAUDE.md `## 매터 워크스페이스` 확인. 비활성이면 skip. 활성이면 어느 매터인지 묻기.

## 목적

상대방이 **소비자**일 때, 한국 소비자 보호법령(전자상거래법·할부거래법·방문판매법·소비자기본법·
표시광고법)은 **계약으로 배제할 수 없는 강행규정**을 부과한다 — 청약철회권(cooling-off), 필수
고지사항, 금지행위, 부당 표시·광고. B2B 거래에 맞춘 일반 플레이북을 B2C에 그대로 적용하면
강행규정 위반을 놓친다.

**왜 별도 스킬인가:** 원본 플러그인은 B2B 중심이다. 한국 소비자법은 사업자에 일방적 의무를 지우고
소비자에 강한 권리를 부여하며, **B2C를 B2B로 오분류하는 것이 최상위 실패 모드**다. 미국·EU 등가물과
요건이 다르다.

## 핵심 조문

국가법령정보에서 현행 텍스트 가져오기 (청약철회 기간·고지사항·금지행위는 **반드시 현행 확인** — 시행령·고시로
정해지고 개정됨):

```
mcp__korean-law__get_law_text "전자상거래 등에서의 소비자보호에 관한 법률"
mcp__korean-law__get_law_text "할부거래에 관한 법률"
mcp__korean-law__get_law_text "방문판매 등에 관한 법률"
mcp__korean-law__get_law_text "표시·광고의 공정화에 관한 법률"
mcp__korean-law__legal_analysis {"mode": "verify_citations", "citation": "전자상거래 등에서의 소비자보호에 관한 법률 제17조"}
```

국가법령정보 미연결 시 모든 인용 `[모델 지식 — 검증 필요]`. **한국소비자원·소비자분쟁조정위원회 결정례,
공정위 고시는 국가법령정보 미커버일 수 있음** — `[소비자원 — 직접 검증 필요]` / `[공정위 — 직접 검증 필요]`.

## Step 1 — B2C vs B2B 분류 게이트 (핵심 — 단정 전 확인)

**이 분류가 적용 게이트다.** 같은 계약도 상대방이 소비자면 강행규정이 적용되고, 사업자면 적용되지
않는다. 오분류는 "확신에 차고 틀린" 출력을 만든다. **분류를 명시적 `[검토]` 결정점으로 둔다.**

| 질문 | 본 거래 |
|---|---|
| 상대방이 개인 소비자인가, 사업자인가? (사업 목적 거래는 소비자 아님) `[검토]` | [판정] |
| 거래 형태는? (온라인 통신판매 / 방문·전화권유 / 할부 / 일반 매장) | [판정] |
| 혼합(B2B2C)인가 — 우리가 플랫폼·중개자인가? | [검토] |

→ 소비자면 Step 2로. 사업자면 소비자법 미적용(단, 약관규제법은 B2B에도 적용 — Step 5). 애매하면:

> "이 거래의 상대방이 소비자인지 사업자인지가 적용 법령을 가릅니다 — 소비자면 청약철회권·필수
> 고지 등 강행규정이 적용되고, 계약으로 배제할 수 없습니다. `[검토 — B2C/B2B 분류]`"

## Step 2 — 적용법 라우터 (소비자일 때)

거래 형태로 적용 법령 라우팅(중복 적용 가능):

| 거래 형태 | 적용 법령 |
|---|---|
| 온라인·통신판매 | 전자상거래법 `[검증 필요]` |
| 할부·신용제공 | 할부거래법(+ 선불식 할부 별도) `[검증 필요]` |
| 방문판매·전화권유·다단계 | 방문판매법 `[검증 필요]` |
| 플랫폼·오픈마켓 중개 | 전자상거래법 통신판매중개자 규정 `[검증 필요]` |
| 일반 소비자 거래 | 소비자기본법(일반법) `[검증 필요]` |

## Step 3 — 청약철회권 + 필수 고지사항 체크리스트

각 항목 국가법령정보 현행 대조(기간·항목은 **국가법령정보에서 가져온다** — 아래는 점검 항목):

| 의무 | 점검 내용 | 상태 |
|---|---|---|
| 청약철회권 | 법정 철회기간을 보장하나? 계약이 철회권을 부당 제한·배제하나? 철회 제한 사유(주문제작 등) 고지? `[검증 필요 — 현행 기간, 국가법령정보]` | ☐ |
| 청약철회 효과 | 대금 환급 기한·방법, 반환비용 부담 규정이 법정과 일치? `[검증 필요]` | ☐ |
| 필수 고지사항 | 사업자 정보·거래조건·청약철회 안내 등 법정 표시·고지 항목을 모두 제공? `[검증 필요]` | ☐ |
| 자동결제·갱신 | 정기결제·자동갱신 시 사전 고지·동의·해지 절차가 법정 요건 충족? `[검토]` | ☐ |
| 금지행위 | 기만적 청약유인, 거짓 표시, 부당한 회원탈퇴 방해 등 금지행위 해당 여부? `[검토]` | ☐ |

## Step 4 — 표시광고법 부당표시·광고 점검

광고·표시 문구가 부당한 표시·광고(거짓·과장·기만·부당비교·비방)에 해당하는지 플래그(`[검토]` —
위반 여부는 공정위 판단). 중요정보 고시 대상이면 표시 의무 `[검증 필요 — 공정위 고시]`.

## Step 5 — 약관규제법 무효 overlay 교차

이용약관이 정형 약관이면 CLAUDE.md `## 약관규제법 무효 overlay` 패스 — §6–14 무효 가능성 `[검토]`.
(소비자법 강행규정 위반과 약관규제법 무효는 별개 트랙 — B2B여도 약관규제법은 적용.)

개인정보를 수집·처리하면 PIPA — `/privacy-legal:` 핸드오프(처리방침·동의·국외이전).
