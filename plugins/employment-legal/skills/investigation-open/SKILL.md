# /investigation-open

## 매터 컨텍스트

실무 수준 CLAUDE.md `## 매터 워크스페이스` 확인. 활성화 상태이면 활성 매터 확인 필수.
활성 매터 없으면 작업 전에 먼저 묻는다:

> "어느 사건 매터입니까? `/employment-legal:matter-workspace new <slug>`로 새 조사 매터를
> 만들거나 `/employment-legal:matter-workspace switch <slug>`로 전환하세요.
> 괴롭힘·성희롱 조사 매터는 비밀 등급 '강화'가 디폴트입니다."

매터 간 컨텍스트가 `on`이 아니면 다른 매터 파일 절대 안 읽음.

산출물 경로: `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/matters/<매터-slug>/outputs/`
로그 경로: `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/investigations/<매터-slug>/incident-log.md`

---

## 목적지 점검 (최엄격)

**산출물 생성 전 반드시 확인.** 이 사건 파일은 비밀누설 금지 의무(근기법 §76-3⑧·남녀고용평등법
§14⑥) 하에 있다. `[모델 지식 — 검증 필요]`

수신자 확인:
- **허용:** 조사자·HR 책임자·법무담당·필요 최소한의 경영진
- **불가:** 공개 채널·전사 배포·인사 전체 공지·당사자(피해자·행위자·참고인) 직접 전달·외부

목적지가 모호하거나 서클 외부이면: 중단하고 (a) 법무용 비밀유지 버전, (b) sanitize 버전,
(c) 둘 다 중 무엇을 줄지 묻는다.

---

## 목적

신규 사건 개시 시 초기 정보를 수집하고, 즉시 보호조치를 확인하며, 조사 사건 파일을 구조화한다.
피해자 진술을 최초 접수하는 시점이므로 2차 피해 방지를 최우선으로 한다.

---

## 초기 신고 내용 수집 (5W)

아래 정보를 순서대로 수집한다. 한 번에 모두 묻지 않는다 — 피해자가 편안하게 답할 수 있도록
속도를 조절한다. `[검토]`

| 항목 | 확인 내용 | 기재 |
|------|-----------|------|
| **누가 (Who)** | 피해자 성명·직책·부서 / 행위자 성명·직책·부서·피해자와의 관계 | |
| **언제 (When)** | 최초 발생일 / 가장 최근 발생일 / 빈도·지속 기간 | |
| **어디서 (Where)** | 발생 장소 (사무실·회의실·사업장 외·온라인 등) | |
| **무슨 행위 (What)** | 행위 유형 및 구체적 내용 (직접 인용 vs 요약 구분 기재) | |
| **어떤 영향 (Impact)** | 피해자가 경험한 신체적·정신적 고통, 근무환경 변화, 업무 영향 | |

**추가 확인:**
- 목격자 또는 참고인이 있는가?
- 관련 문서·메시지·녹취 등 증거가 있는가?
- 이전에 내부 신고 또는 시정 요청을 한 적 있는가?
- 현재 형사·민사 절차를 병행하거나 고려 중인가?

**기재 원칙:**
- 직접 인용(`" "`)과 조사자 요약을 명확히 구분한다
- 피해자의 표현을 가치 판단 없이 있는 그대로 기재한다
- 조사자의 평가·의견은 별도 `[조사자 관찰]` 태그로 분리한다

## 즉시 보호조치 체크리스트

**피해자 의향을 먼저 확인한다.** 보호조치가 오히려 신분 노출이나 불이익으로 이어지지 않도록
피해자와 협의하여 결정한다. `[검토]`

피해자가 모든 보호조치를 거부하더라도 그 의향과 이유를 기록한다.

## 비밀준수 서약 안내

조사 관여자(조사자·참고인 면담자·HR 담당자 등) 전원에게 비밀준수 서약을 받는다.

안내 내용:
- 조사 과정에서 알게 된 일체의 내용은 직무상 목적 외 누설 금지
- 위반 시 근기법 §76-3⑧에 따른 500만원 이하 과태료 `[모델 지식 — 검증 필요]`
- 서약 서명 일자와 서명자를 incident-log에 기재

## 사건 파일 초기화

아래 구조로 사건 파일을 생성한다.

```
~/.claude/plugins/config/claude-for-legal-kr/employment-legal/investigations/<매터-slug>/
├── incident-log.md        # 사건 번호·신고 내용·타임라인·보호조치·진술 색인
└── outputs/               # 조사 메모·보고서 초안 (목적지 점검 후 생성)
```

`incident-log.md` 초기 항목:

```markdown
# 조사 사건 로그 — [매터-slug]

**비밀유지:** 이 파일은 비밀누설 금지 의무(근기법 §76-3·남녀고용평등법 §14) 하에 작성됨.
조사 목적 외 열람·공유 금지.
```
