# /leave-tracker

가드레일·헤더·검토자 메모·태그 어휘·결정 자세·산출물 포맷·대시보드 제안은 모두 이 플러그인의
`~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md`가 정본이다 — 여기서
재진술하지 않고 가리킨다. 충돌 시 그 CLAUDE.md가 우선한다.

## 매터 컨텍스트

`~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md` 읽기.
- 매터 워크스페이스가 활성이면: 활성 매터의 `matter.md`를 읽고 해당 매터의 leave-register 경로를 확인.
  매터-특화 override(예: "이 의뢰인 사업장 상시 근로자 수, HRIS 상황")를 적용.
- 사내 사용자(워크스페이스 off): 실무 수준 CLAUDE.md에서 `## 휴가·휴직 등록부 설정` 블록을 읽어
  등록부 경로·알림 설정을 가져온다.
- CLAUDE.md가 없거나 `[PLACEHOLDER]`이면: substantive 작업 전 멈추고
  `/employment-legal:cold-start-interview`로 안내.

## 목적지 점검

leave-register.yaml은 근로자명·휴직사유 등 개인정보를 포함한다. 산출물 생성·공유 전:
- 등록부 조회 결과를 Slack·이메일 등 외부로 보내려는 경우, 비밀유지 서클 내부인지 확인.
- 익명 코드로만 관리하는 사업장은 실명 출력 전 확인.
- `## 산출물` (CLAUDE.md)의 역할별 헤더 적용.

## 목적

한국 법정 휴가·휴직의 기한 관리:

- **연차유급휴가(§60)** — 발생 일수, 소멸일, 사용촉진 절차 기한
- **연차사용촉진(§61)** — 1차 촉구·2차 지정 기한 충족 여부 (미이행 시 미사용수당 지급 의무 발생)
- **출산전후휴가(§74)** — 종료 예정일, 복직 예정일
- **배우자 출산휴가(남녀고평법 §18-2)** — 사용 마감일 (출산일 + 90일 이내)
- **육아휴직(남녀고평법 §19)** — 신청 기한, 종료일, 복직 예정일, 해고 금지 기간
- **육아기 근로시간 단축(§19-2)** — 종료일, 복직 예정일

> **한국 법체계 포인트.** 이 스킬은 한국 법 전용이다. FMLA·CFRA·USERRA·ADA는 적용되지 않는다.
> 한국은 전국 단일 법체계 — 미국의 "state supplement" 개념 없음. 신청 기한·부여 의무·불리한 처우
> 금지는 근기법 + 남녀고용평등법이 지배한다. `[모델 지식 — 검증 필요]`

## 플레이북 먼저 로드

CLAUDE.md `## 휴가·휴직` 섹션과 `## 휴가·휴직 등록부 설정` 블록을 읽는다. 여기에:
- `register_path`: leave-register.yaml 위치
- `alert_destination`: Slack 채널·이메일·인라인 중 어느 것
- `lead_times_days`: 임박 기한 알림 리드타임 (예: [30, 14, 7])
- `hris`: HRIS 연결 여부 + 시스템명

HRIS가 연결되어 있으면 실시간 pull을 우선 시도. 미연결이면 로컬 등록부로만 동작.

## 등록부 읽기

`register_path`에서 `leave-register.yaml`을 읽는다. 없으면 빈 등록부로 초기화:

```yaml
# leave-register.yaml — 휴가·휴직 등록부
# 생성: [YYYY-MM-DD] | 플러그인: employment-legal
leaves: []
```

등록부 키는 machine-stable 영문, 사람이 보는 값·코멘트는 한국어.

## 액션 라우팅

`$ARGUMENTS`의 첫 토큰으로 디스패치:

### `--list` / 조회 (인수 없음 시 디폴트)

전체 등록부를 표 형식으로 출력:

```markdown
## 휴가·휴직 등록부

| 근로자 | 휴가 종류 | 시작일 | 종료(예정)일 | 상태 | 다음 법정 기한 | D-day |
|---|---|---|---|---|---|---|
| [근로자] | [종류] | [날짜] | [날짜] | [상태] | [기한 설명] | D-[N] |
```

상태: `진행중` / `종료` / `촉진중` / `신청대기`

근로자명이 익명 코드이면 코드 그대로 출력. 실명 출력이 필요하면 목적지 점검 후 진행.

### `--add` / 새 등록

`/employment-legal:log-leave`로 안내:
> "새 휴가·휴직 등록은 `/employment-legal:log-leave`를 사용하세요. 법정 사항을 자동으로 계산하고
> 등록부에 추가할 YAML 블록을 생성합니다."

### `--alert` / 임박 기한

오늘 날짜 기준으로 `lead_times_days` 이내 기한 항목을 필터링. CLAUDE.md의 리드타임 설정을 따른다
(미설정이면 디폴트 [30, 14, 7]).

```markdown
## 임박 기한 알림

### 🔴 7일 이내

| 근로자 | 기한 종류 | 기한일 | D-day | 필요 조치 |
|---|---|---|---|---|

### 🟠 14일 이내

[같은 표]

### 🟡 30일 이내

[같은 표]
```

각 항목에 필요 조치 한 줄 첨부 (예: "연차사용촉진 1차 촉구 서면 발송 필요").

### `--report` / 기간별 보고서

사용자가 기간을 명시하지 않으면 묻는다: "어느 기간의 집계가 필요하신가요? (예: 2025년 1분기)"

```markdown
## 휴가·휴직 집계 보고서 — [기간]

| 휴가 종류 | 건수 | 진행중 | 종료 | 연차미사용수당 위험 |
|---|---|---|---|---|
```

## 법정 기한 계산기

등록부의 시작일·종료일·출산일을 기준으로 자동 계산. 계산 결과는 `[모델 계산 — 해당 법조항 대조 검증 필요]` 태그.

### 연차유급휴가 (근기법 §60)

**1년 미만 근로자:**
- 1개월 개근 시 1일 발생 (최대 11일) `[모델 지식 — 검증 필요]`
- 소멸일: 입사 1주년 (또는 해당 회계연도 연차 마감일 — 취업규칙 확인)

**1년 이상 근로자:**
- 출근율 80% 이상: 15일
- 3년 이상 근속: 2년마다 1일 가산 (최대 25일) `[모델 지식 — 검증 필요]`
- 소멸일: 발생일로부터 1년 (취업규칙에 다른 정함이 없는 한)

**출근율 산정·가산 일수는 `[검토]` — 산정 방식·공제 사유에 따라 달라짐.**

### 연차사용촉진 (§61)

연차 소멸일을 기준으로:

- **1차 촉구 기한:** 소멸일 - 6개월 (이 날까지 미사용 일수 통보 + 사용 촉구 서면 발송)
- **2차 지정 기한:** 1차 촉구일로부터 10일 이내 근로자가 사용 시기를 지정하지 않으면,
  사용자가 시기를 지정하여 서면 통보 — 단, 소멸일 - 2개월 이전까지
- **절차 이행 효과:** 미사용 연차수당 지급 의무 면제
- **절차 미이행 효과:** 미사용수당 전액 지급 의무 발생

> **주의.** 1차 촉구·2차 지정 기한을 하루라도 어기면 절차 이행의 효과가 없다 — 미사용수당
> 지급 의무가 살아난다. `[모델 지식 — 검증 필요, 법망 또는 고용부 행정해석 대조]`

### 출산전후휴가 (§74)

- **단태아:** 시작일 + 90일 = 종료 예정일; 출산 후 45일 이상 부여 필수
- **다태아:** 시작일 + 120일 = 종료 예정일; 출산 후 60일 이상 부여 필수
- **복직 예정일:** 종료 예정일 + 1일
- **해고 금지 기간:** 출산전후휴가 기간 + 그 후 30일 (근기법 §74⑦) `[모델 지식 — 검증 필요]`
- 다태아 여부는 등록 시 입력 (`multiple_birth: true`).

### 배우자 출산휴가 (남녀고평법 §18-2)

- **부여 일수:** 10일 유급 `[모델 지식 — 2024년 기준, 개정 여부 검증 필요]`
- **사용 마감:** 출산일 + 90일 이내
- **분할 사용:** 가능 (분할 횟수 제한 — 법망 또는 고용부 사이트 확인)

### 육아휴직 (남녀고평법 §19)

- **대상:** 만 8세 이하 또는 초등학교 2학년 이하 자녀
- **기간:** 최대 1년 (부모 각각; 가구당 최대 2년)
- **신청 기한:** 시작 희망일 30일 전 서면 신청 (예외: 긴급 사유 — 법망 확인)
- **신청 마감 계산:** 희망 시작일 - 30일
- **종료일:** 등록된 종료일
- **복직 예정일:** 종료일 + 1일
- **해고 금지 기간:** 육아휴직 신청 시부터 복직 후 일정 기간 (남녀고평법 §19④) `[모델 지식 — 검증 필요]`
- **육아휴직급여:** 고용보험에서 지급 — 구체 금액은 고용보험 사이트(https://www.ei.go.kr) 확인.
  **절대 금액을 하드코딩하지 않는다.**

### 육아기 근로시간 단축 (§19-2)

- **주 근로시간:** 15~35시간 범위 `[모델 지식 — 검증 필요]`
- **종료일:** 등록된 종료일
- **복직 예정일:** 종료일 + 1일

## 임박 기한 알림

등록부의 각 항목에 대해 "다음 법정 기한"을 계산하고, `lead_times_days` 이내이면 알림 항목에 포함.

**심각도 기준 (CLAUDE.md `lead_times_days` 설정 우선):**
- 🔴 7일 이내
- 🟠 8~14일
- 🟡 15~30일
- (30일 초과는 `--list`에서 확인 가능하나 `--alert` 디폴트 출력 제외)

**리드타임 미설정이면 디폴트 [30, 14, 7] 사용.**

각 알림 항목:
- 근로자 (익명 코드 또는 실명 — 목적지 점검 후)
- 기한 종류 + 법조항
- 기한일 + D-day
- 필요 조치 한 줄
- 담당자 (등록부에 기재된 경우)

## 대시보드 제안

등록부 항목이 10건 이상이거나 사용자가 요청하면 HTML 대시보드를 제안한다:

> 📊 **대시보드로 볼래요?** 휴가 종류별 현황, D-day 타임라인, 연차사용촉진 진행 상황을
> 인터랙티브 뷰로 보여드립니다. Claude Code에서는 [산출물 폴더]에 HTML 파일로 저장.

대시보드는 미리 만들지 않고 제안 → 사용자 확인 후 생성.

**대시보드 출력은 신뢰 못 할 입력을 이스케이프한다** — 근로자명·사유 등 외부 입력은 HTML render
전 escape. 셀 텍스트는 `textContent`로만 설정 (`innerHTML` 금지).

## 의사결정 트리

> **다음 단계? 하나 골라주세요:**
> 1. **연차사용촉진 절차 시작** — 1차 촉구 서면 초안 작성. 대상 근로자와 미사용 일수를 알려주세요.
> 2. **새 휴가·휴직 등록** — `/employment-legal:log-leave` 실행.
> 3. **leave-tracker-agent 설정** — 주간 자동 알림 에이전트 설정. `/employment-legal:leave-tracker-agent --config`
> 4. **에스컬레이션** — 기한 초과·절차 미이행 위험 항목을 CLAUDE.md 에스컬레이션 담당자에게 보낼 메모 초안.
> 5. **다른 것** — 무엇을 하고 싶은지 알려주세요.

## 이 스킬이 하지 않는 것

- 연차사용촉진 서면을 자동으로 발송하지 않는다 — 초안을 만들고 사용자가 검토·발송.
- 육아휴직급여 구체 금액을 계산하지 않는다 — 고용보험 사이트 안내.
- 출근율 산정을 단독으로 결론짓지 않는다 — `[검토]` 플래그.
- 해고 금지 위반 여부를 단정하지 않는다 — `/employment-legal:termination-review`로 핸드오프.

## 등록부 레코드 형식 참조

```yaml
leaves:
  - id: "EMP001-AL-2025"           # machine-stable ID
    employee: "김OO"               # 실명 또는 익명 코드
    leave_type: "연차"             # 연차 | 출산전후휴가 | 배우자출산휴가 | 육아휴직 | 육아기근로시간단축 | 기타
    start_date: 2025-01-01
    end_date: 2025-12-31           # 종료 예정일 (연차: 소멸일)
    status: "진행중"               # 신청대기 | 진행중 | 촉진중 | 종료
    multiple_birth: false          # 출산전후휴가 시 다태아 여부
    annual_days_total: 15          # 연차: 해당 연도 총 발생 일수
    annual_days_used: 5            # 연차: 사용 일수
    annual_days_remaining: 10      # 연차: 잔여 일수
    expiry_date: 2025-12-31        # 연차 소멸일
    promotion_1st_deadline: 2025-06-30   # 연차사용촉진 1차 촉구 기한
    promotion_1st_sent: false            # 1차 촉구 발송 여부
    promotion_2nd_deadline: 2025-10-31   # 연차사용촉진 2차 지정 기한
    promotion_2nd_sent: false            # 2차 지정 발송 여부
    reinstatement_date: null       # 복직 예정일 (육아휴직·출산휴가)
    dismissal_prohibition_end: null  # 해고 금지 종료 예정일
    application_deadline: null     # 신청 기한 (육아휴직: 시작일 - 30일)
    owner: "인사팀"                # 담당자
    notes: ""
    provenance: "[모델 계산 — 법조항 대조 검증 필요]"
```
