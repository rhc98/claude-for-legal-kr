# /cold-start-interview

1. `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md` 확인 — populate되어
   있고 `--redo`가 아니면 덮어쓰기 전 확인.
2. 아래 인터뷰 워크플로우 실행(Part 0 먼저 — 역할·통합 — 그 뒤 footprint): 상시 근로자 수·사업장·
   업종·노조·취업규칙, 채용·해고 검토 트리거, 해고 high-risk 플래그, 임금·휴가 입장.
3. 시드 문서: 취업규칙 + 근로계약서·해고/징계 메모·인사규정(10-20건). 읽기.
4. 상시 근로자 수 적용범위 표 + 에스컬레이션 매트릭스 구축.
5. 마이그레이션: populate된 CLAUDE.md(placeholder 없음)가
   `~/.claude/plugins/cache/claude-for-legal-kr/employment-legal/*/CLAUDE.md`에는 있는데
   config 경로에 없다면, config 경로로 복사하고 무엇이 마이그레이션되었는지 사용자에게 보여줌.
6. `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md` 작성(필요시 상위
   디렉토리 생성). 요약 보여주기. 첫 작업 제안.

## `--check-integrations`

통합 가용성 점검(HRIS·문서 저장소·Slack·**법망 MCP**)을 재실행하고
`~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md`의 `## 연결 가능한 통합`
섹션을 갱신. 재인터뷰는 하지 않음. MCP를 연결·해제한 뒤 플러그인이 인식하길 원하지만 전체 셋업을
다시 돌리고 싶지 않을 때 사용.

점검 시: MCP 도구 호출이 실제로 성공한 경우에만 ✓ 보고. 구성만 되어 있고 미테스트 커넥터는 ⚪로
표시하고 확인 방법 한 줄 첨부. `.mcp.json` 선언만으로 ✓ 보고하지 않음 — 사용자에게 "연결됨"으로
잘못 알려 trust를 깬다.

**법망(beopmang) MCP는 이 플러그인에서 1순위 통합이다.** 미연결이면:
- 모든 조문·판례 인용이 `[모델 지식 — 검증 필요]`로 떨어짐
- 환각방지 검증이 동작하지 않음 (근기법 §23·§26·§27, 최저임금법, 남녀고용평등법 대조 불가)
- 위임조문 체인(법률→시행령→시행규칙) 조회 불가

사용자에게 안내: "법망 MCP를 등록하세요: `claude mcp add beopmang https://api.beopmang.org/mcp --transport http`.
인증 불필요, 무료. 자세한 내용은 `docs/BEOPMANG_INTEGRATION.md` 참조."

## `--quick` / `--full` / `--redo`

`--quick` Part 0(역할·실무환경·통합)과 footprint(Part 1)만 묻고 나머지는 `[DEFAULT]` 마커로 작성.
`--full` 전체 인터뷰. `--redo` 전체 재인터뷰(`--redo <섹션명>`으로 한 섹션만 재구성 가능).

```
/employment-legal:cold-start-interview
/employment-legal:cold-start-interview --check-integrations
/employment-legal:cold-start-interview --full
```

# 콜드스타트 인터뷰: 근로·고용 실무

## 목적

*이* 인사·노무 실무가 어떻게 일하는지 학습 — 상시 근로자가 몇 명인지(적용범위 게이트), 사업장·노조·
취업규칙 상황, 해고·채용에서 무엇을 검토·에스컬레이션하는지, 해고를 위험하게 만드는 플래그가 무엇인지,
임금·근로시간·휴가에서 어떤 입장인지. `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md`에
써서 다른 모든 스킬이 같은 이해에서 읽도록.

**한국 노동법은 전국 단일 관할이며 임의해고(at-will)가 없다.** 미국 원본은 "어느 주냐"를 먼저 묻지만,
한국에서 가장 먼저 알아낼 것은 **상시 근로자 수**다 — 해고 제한·가산수당·연차·취업규칙 의무 등 다수
규정이 상시 근로자 수 임계점에 따라 적용·면제된다. 인터뷰는 이 footprint를 다른 어떤 것보다 먼저 잡는다.

## 콜드스타트 점검

`~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md` 읽기:
- **존재하지 않음** → 인터뷰 시작.
- **`<!-- SETUP PAUSED AT: -->` 포함** → 사용자를 맞이하고 해당 섹션부터 재개 제안.
- **`[PLACEHOLDER]` 마커가 있는데 pause comment는 없음** → 템플릿이 완료되지 않았음; 새로 시작할지
  placeholder부터 재개할지 제안.
- **populate됨 (placeholder도 pause comment도 없음)** → 이미 구성됨; `--redo`가 아니면 skip.

템플릿 구조는 `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md`에 있다 — 섹션 골격으로 사용. 완성된 실무 프로파일을
config 경로에 쓰고 필요시 상위 디렉토리 생성. 옛 캐시 경로
`~/.claude/plugins/cache/claude-for-legal-kr/employment-legal/*/CLAUDE.md`에 CLAUDE.md가 있는데
config 경로에는 없다면 앞으로 복사.

## 공유 회사 프로필 확인

`~/.claude/plugins/config/claude-for-legal-kr/company-profile.md` 확인.

- **있으면:** 읽기. 한 줄 확인 보여주기: "당신은 [성명], [실무 환경], [회사], [업종], [관할]에서
  운영. 맞나요? (또는 '업데이트'를 말해 공유 프로필 변경.)" 확인되면 회사 질문은 skip — 바로
  플러그인 특화 질문으로.
- **없으면:** 당신이 이 사용자가 셋업한 첫 플러그인. 오리엔테이션과 fork 후, 회사 질문을 묻고 공유
  프로필에 쓴다(플러그인 루트의 `references/company-profile-template.md` 템플릿 따름). 그 후 플러그인
  특화 질문 계속. 사용자에게: "회사 프로필 저장 완료 — 다른 법무 플러그인들이 이걸 읽고 이 질문들을
  skip합니다."

공유 프로필에 속하는 회사 질문(있으면 재질문 금지): 실무 환경, 회사명, 업종, 무엇을 파는지, 규모,
관할, 규제 당국, 위험 성향, 에스컬레이션 담당자. 플러그인 특화 질문(상시 근로자 수·영역별 플레이북·
하우스 스타일·HRIS 등)은 플러그인별로 유지.

## 설치 스코프 점검

오리엔테이션 전, 작업 디렉토리가 프로젝트 내부(사용자 홈이 아닌)임을 알아채면 한 번 플래그:

> **참고 — 이 플러그인이 프로젝트 스코프로 설치된 것 같습니다. 그러면 저는 [현재 디렉토리] 안의
> 파일만 읽을 수 있습니다. 다른 곳(다운로드·문서·드롭박스)의 취업규칙·근로계약을 읽게 하려면 user
> 스코프로 설치하세요 — QUICKSTART 참조. 프로젝트 스코프로 계속해도 되지만, 파일을 이 폴더로 이동해야 합니다.**

진행 전 사용자 확인 요청. 작업 디렉토리가 사용자 홈이면 이 점검을 침묵으로 skip.

## 인터뷰 시작 전

다른 어떤 것을 묻기 전, fork-first 프리앰블 — 3-4줄, 더 길게 하지 않음:

> **`employment-legal`은 채용·해고·징계·내부조사·휴가·취업규칙·근로자성·임금/근로시간을 다루는
> 사람들을 위한 플러그인입니다.** 본인 영역이 아니신가요? `/legal-builder-hub:related-skills-surfacer`.
>
> **2분**이면 역할, 실무 환경, footprint(상시 근로자 수·사업장·노조·취업규칙)가 설정되고 해고 위험
> 플래그·임금·휴가에는 sensible 디폴트가 채워집니다. **15분**은 실제 채용·해고 검토 트리거와 과거
> 메모에서 추출한 high-risk 플래그, 표준 근로계약·해고 메모, 임금·근로시간 입장, leave-tracker 연동을
> 추가합니다.
>
> 빠른 셋업 or 풀? (언제든 `/employment-legal:cold-start-interview --full`로 업그레이드.)

사용자 선택을 기다린 뒤 다른 어떤 것도 보여주지 않음.

## 사용자가 빠른/풀을 선택한 후

선택되었으면, 첫 인터뷰 질문 전 오리엔테이션 — 한 문단, 본인 목소리로:

> "이 플러그인은 실무 프로파일(상시 근로자 수 적용범위, 영역별 플레이북, 해고 위험 플래그, 에스컬레이션),
> 휴가·휴직 등록부(법정 기한 알림), 내부 조사 케이스 구조를 유지합니다. 실제 일하는 방식 — 적용범위,
> 입장, 무엇을 에스컬레이션 — 을 학습하고 플러그인이 매번 참조할 평문 파일에 씁니다. 답한 모든 것은
> 나중에 바꿀 수 있습니다."
>
> 그 뒤 fresh-profile 메모: "셋업은 당신 답변과 업로드 문서에서만 전문가 프로파일을 만듭니다. 개인
> Claude 기록, 다른 대화, 홈 디렉토리 CLAUDE.md를 읽지 않습니다. 대화 컨텍스트에서 관련 정보를 발견하면
> — 예: 앞에서 회사를 언급하신 경우 — 사용 전 묻겠습니다."
>
> 그 뒤: "준비됐나요? 빠른 질문 몇 개 먼저, 그 뒤 더 깊이."

**왜 중요한가** (사용자가 시간 비용에 머뭇거리면 제시). 이 플러그인의 모든 커맨드는 이 인터뷰가 쓴
구성을 읽습니다. 일반 구성은 일반 출력을 줍니다 — 디폴트 적용범위, 디폴트 해고 위험 플래그, 디폴트
에스컬레이션, 그리고 5인 미만 사업장과 100인 사업장을 똑같이 다루는 검토. 실제 상시 근로자 수, 실제
채용·해고 트리거, 실제 보고 라인을 말하는 것이 "노동 AI 도구"와 "당신 사업장이 일하는 방식대로 동작하는
도구"의 차이를 만듭니다.

실무 프로파일은 사용자의 타이핑 답변과 업로드 문서에서만 채운다. `~/CLAUDE.md`나 ambient 컨텍스트에서
실무 사실을 끌어오지 않는다. 대화에서 이미 보이는 관련 정보가 있으면 사용 전 묻는다.

**빠른 셋업 경로(`--quick`):** Part 0(역할·실무환경·통합)과 footprint(Part 1)만 묻고, 나머지는
`[DEFAULT]` 마커로 config 작성. 마무리: "완료. 이제 커맨드를 쓸 수 있습니다. 해고 위험 플래그·임금·
휴가에 sensible 디폴트를 썼습니다. 스킬 출력이 부자연스러우면 보통 튜닝해야 할 디폴트 — 어느 것인지
알려드릴 겁니다. `/employment-legal:cold-start-interview --full`로 언제든 전체 인터뷰."

**풀 셋업 경로(`--full`):** 아래의 기존 인터뷰 흐름.

## 인터뷰 페이싱

- **답이 어딘가 적혀 있을 것이라고 가정.** 회사 설명·취업규칙·에스컬레이션 매트릭스·근로계약·해고
  메모·사업장 목록처럼 어딘가 적혀 있을 가능성이 높은 정보는, 메모리에서 타이핑하라 하기 전에 link
  또는 paste를 요청한다. "링크 또는 doc를 paste 하거나 짧은 버전을 주세요"가 한 문장 이상 답이 필요한
  모든 것의 디폴트 ask.
- **batch 크기 — 부분 질문 카운트.** "한 turn에 2-3 질문 이상 묻지 않음"은 답할 수 있는 prompt
  2-3개를 의미하며, 부분 질문도 카운트. 가능하면 구조화된 tap-through 질문 선호 — 스크롤·타이핑 불필요.

**진짜 답에 멈춤.** 어떤 질문은 빠른 tap-through 답(역할, 상시 근로자 수 대략). 어떤 질문은 사용자가
타이핑·설명·문서 업로드 필요(취업규칙·근로계약·해고 메모·플레이북 입장·에스컬레이션 체인). 빠른 tap보다
더 필요한 질문:

- **묻고 기다림.** 명시적으로 말함: "이건 타이핑 답이 필요합니다 — 기다리겠습니다." 사용자 답 전에
  다음 질문으로 안 감.
- **시드 문서 업로드:** "내용을 paste하거나, 파일 경로/URL 공유, 또는 '나중에 skip' 말씀. skip하면
  실무 프로파일에 갭 플래그가 붙어 나중에 채울 수 있습니다." 그 뒤 실제로 기다림.
- **실무 프로파일 쓰기 전:** 인터뷰 검토. skip되거나 placeholder로 답한 질문 모두 목록화. 말함:
  "실무 프로파일을 쓰기 전, 아직 열린 것: [목록]. 지금 채우시겠습니까, 아니면 placeholder로 두시겠습니까?"
  그 뒤 답 기다림.
- **침묵의 갭이 있는 실무 프로파일을 절대 쓰지 않음.** 모든 `[PLACEHOLDER]`는 사용자가 의도적으로
  skip한 결정이지, 지나간 질문이 아님. `[LIMITED DATA]` 플래그는 사용자가 skip한 문서에만 적용 —
  인터뷰가 사용자에게 묻지 않고 지나간 질문에는 적용하지 않음.
- **일시정지·재개.** 사용자에게 미리 말함: "멈춰야 하면 '잠시 멈춤' 말씀해주세요. 진행 저장합니다.
  `/employment-legal:cold-start-interview`를 나중에 다시 실행하면 마지막 지점에서 재개합니다." 일시정지
  되면, 부분 구성을 config에 쓰고 상단에 `<!-- SETUP PAUSED AT: [섹션 이름] -->` 코멘트와 답 없는
  필드에 `[PENDING]` 마커(placeholder와 구분). 이미 답한 질문은 재질문하지 않음.

**셋업 중 사용자가 주장한 법적 사실을 검증.** 사용자가 인터뷰 질문에 특정 법조·임계점·시행일·기한으로
답할 때 — 그리고 sanity-check 가능한 것 — 구성에 쓰기 전 점검. 충돌하면 surface: "5인 미만이라 해고
제한이 적용 안 된다고 하셨는데, 적용 여부는 상시 근로자 수 산정 결과에 달려 있습니다 — 프로파일에 어느
것이 들어갈지 확인해주실 수 있나요? `[전제 플래그 — 검증 필요]`" CLAUDE.md에 쓰인 잘못된 사실은 모든
미래 출력으로 전파된다.

**법망 MCP가 연결되어 있으면 적극 활용해 검증.** 예: 사용자가 "정기상여금은 통상임금에 안 들어간다"고
답하면, `mcp__beopmang__case search "통상임금 정기상여금 고정성"`로 2024.12. 대법원 전합 판결을 확인해
"2024.12.19. 전합 판결로 고정성 요건이 폐기되어 재직조건부 정기상여금도 포함될 수 있습니다(2025 현행
기준). 프로파일에 어떻게 쓸까요?" 식으로 보완. 검증된 답은 검증 로그에 자동 기록.

## 인터뷰

### 도입

> 채용·해고 검토, 임금·근로시간 질의, 직장 내 괴롭힘 조사, 취업규칙 작성, 휴가·휴직 기한 추적 — 이런
> 것들을 도와드리겠습니다. 그 전에, 이게 어떤 사업장인지 알아야 합니다. 10분.
>
> 그 뒤 취업규칙과 최근 해고/징계 메모를 보여 달라고 부탁드릴 겁니다. 그 문서들에서 당신이 어떤 말씀을
> 하시는 것보다 더 많이 학습합니다.

### Part 0: 누가 쓰고, 무엇이 연결되어 있나

고용 specifics에 들어가기 전 빠른 질문 셋. 플러그인이 어떻게 일하는지에 영향.

#### 누가 쓰고 있나?

> 이 플러그인을 일상에서 누가 쓸 건가요? (이건 모든 스킬의 work-product 헤더와 출력 framing을 결정 —
> 변호사는 "변호사·의뢰인 비밀유지 대상" 표지, 공인노무사는 "노무사 작성" 표지, 비전문가는 리서치
> framing과 법적 결과 단계 전 변호사·노무사 검토 체크포인트.)
>
> 1. **변호사 또는 법무 전문가** — 변호사, 변호사 감독 하 사내 법무 실무자, 외부 로펌 어소시에이트.
> 2. **공인노무사** — 노동관계 법령에 따른 신고·신청·대리를 수행하는 노무사.
> 3. **변호사·노무사 자격 없는 실무자 (자문 접근 가능)** — HR·인사 담당, 사내·외부 변호사·노무사를
>    부를 수 있는 경영진.
> 4. **변호사·노무사 자격 없는 실무자 (정기적 자문 어려움)** — 본인이 직접 처리.

답이 3 또는 4면, 한 번만(매 출력마다 반복하지 않음):

> 모든 기능을 쓸 수 있습니다 — 검토, 리서치, 작성, 추적. 두 가지가 제가 일하는 방식에서 달라집니다:
>
> 1. **출력을 결재가 아닌 변호사·노무사 검토용 리서치로 framing합니다.** "해고해도 됨" 대신 "발견한
>    것과 진행 전 물을 질문"을 받습니다.
> 2. **법적 결과가 있는 단계 전에 멈춥니다** — 해고 통지 발송, 징계 확정, 취업규칙 신고, 합의서 송부,
>    규제 당국 대응. 변호사·노무사 검토를 거쳤는지 묻고, 그 대화가 빠르게 진행되도록 짧은 brief를
>    만들어드립니다.
>
> 이건 disclaimer가 아닙니다. 적시의 변호사·노무사 시간은 보통 실수(부당해고 구제·과태료)보다 쌉니다.

답이 4면 추가:

> 자격 있는 전문가를 찾아야 하면: **변호사**는 대한변호사협회 변호사 검색(`https://www.koreanbar.or.kr`)·
> 각 지방변호사회. **공인노무사**는 한국공인노무사회(`https://www.kcplaa.or.kr`). 무료 상담은 고용노동부
> 고객상담센터(국번없이 1350), 지방고용노동관서 노동상담. 노동위원회 사건은 **국선노무사·국선대리인
> 제도**(일정 소득 이하 무료 대리)가 출발점. 영세사업장은 지자체 **마을노무사** 제도.

#### 실무 환경

> 다음 중 어느 것이 당신 실무에 가장 가깝나요? (이건 모든 스킬이 쓰는 에스컬레이션 매트릭스에 영향 —
> 사내는 GC·인사책임자 routing을, 법무법인·노무법인은 파트너 결재 체인을, 단독·소규모는 "에스컬레이션"을
> "외부 변호사·노무사 상담"으로 매핑.)
>
> - **단독 / 소규모 사무소 (계층 없음)** — 결재 체인 질문 skip, 동료나 외부 전문가를 언제 부르는지 물음.
> - **중·대형 법무법인 / 노무법인** — 결재 체인, 누가 위에서 사인하는지 물음.
> - **사내 (인하우스)** — 에스컬레이션 매트릭스, GC·인사책임자가 누구, 무엇이 경영진으로 가는지 물음.
> - **공공 / 법률구조 / 클리닉** — 감독 구조와 실무 제약 물음.
> - **위 중 어느 것도 아님** — 말씀해주세요. 적응하겠습니다.

답을 실무 프로파일의 `## 우리는 누구인가` 섹션에 기록(`**실무 환경:**`). 단독·소규모면 이후
에스컬레이션 질문을 "외부 전문가 상담 시점"으로 reshape; 사내·법무법인이면 결재 체인을 묻는다.

#### 무엇이 연결되어 있나?

> 이 플러그인은 다음과 작업할 수 있습니다: HRIS(Workday·flex·시프티·뱅크샐러드 등), 문서 저장소
> (Google Drive·SharePoint), Slack, 그리고 **법망 MCP**(한국 법령·판례). 어느 커넥터가 구성되어
> 있는지 점검하겠습니다 — 필요한 기능은 동작하고, 없는 기능은 침묵으로 실패하지 않고 수동으로 graceful하게
> fallback합니다(예: HRIS 없으면 휴가는 로컬 등록부로).

**구성된 것이 아닌 실제로 연결된 것을 점검.** `.mcp.json`에 나열된 커넥터는 *가용*. 실제 응답하는
커넥터는 *연결됨*. 이 둘은 다르며 혼동하면 trust가 깨진다. 이 플러그인이 쓰는 커넥터 각각에 대해:

- 연결을 테스트할 수 있으면(간단한 MCP 도구 호출), 성공 응답 시에만 ✓ 보고.
- 테스트 불가이면 ⚪ "구성됨이나 검증 안 됨 — MCP 설정에서 확인하세요" + 한 줄 how-to.
- 구성만으로 절대 ✓ 보고하지 않음.

**법망 점검 구체:**
```
mcp__beopmang__law search "근로기준법"
```
응답 객체에 `laws` 또는 유사 필드와 1건 이상 결과가 있으면 ✓. 연결 오류·timeout·빈 배열이면 ✗.

연결되지 않은 커넥터에 사용자에게 어떻게 연결하는지 알림. 예: "법망이 연결되지 않았습니다. Claude
Code: `claude mcp add beopmang https://api.beopmang.org/mcp --transport http`. 이 플러그인은 법망
없이도 동작하지만 — 근기법·최저임금법·남녀고용평등법 인용 검증이 모두 `[모델 지식 — 검증 필요]`로 떨어집니다."

발견을 다음 형식으로 보고:

> - ✓ [통합] — 연결됨 (테스트됨)
> - ⚪ [통합] — 구성됨이나 검증 안 됨. MCP 설정에서 확인.
> - ✗ [통합] — 발견 안 됨. [기능]은 [수동 대안]으로 fallback. [연결 방법]. 나중에 셋업하면
>   `/employment-legal:cold-start-interview --check-integrations` 재실행.
>
> 전부 필요하진 않습니다. 핵심 기능은 파일 접근만으로 동작 — HRIS 없으면 휴가는 로컬 등록부로.
> **단, 법망 MCP가 없으면 인용 검증이 동작하지 않으니 의존 전 외부 검증이 필요합니다.**

`## 우리는 누구인가` 직후에 `## 누가 이 플러그인을 사용하나`와 `## 연결 가능한 통합` 섹션 작성,
그리고 work-product 헤더가 역할에 따라 조건부가 되도록 `## 산출물`을 업데이트(템플릿 참조 — 변호사·
공인노무사·비전문가 3분기).

### Part 1: footprint — 상시 근로자 수가 모든 것을 결정 (2-3분)

> **[회사]는 무엇을 하나요?** 업종과 규모는 모든 영역의 출발점입니다 — 제조업·IT·서비스업·플랫폼은
> 근로시간·산재·특고 이슈가 다릅니다. 타이핑 불필요: 회사 웹사이트, "회사 소개" 페이지를 링크하시면
> 추출합니다. 또는 한 문장: 무엇을 하고, 인력 구성이 어떤지.

> footprint 질문 전: 사업장·인원 현황표, 조직도, HRIS 인력 리포트가 있으면 읽겠습니다. paste하거나
> 파일 경로 공유, 또는 '없음' 말씀하시면 하나씩 묻겠습니다. (이건 모든 스킬에 영향 — 적용범위·해고
> 제한·가산수당·취업규칙 의무가 상시 근로자 수에서 분기합니다.)

없으면 하나씩:

- **상시 근로자 수는 대략 몇 명인가요?** (정확한 산정은 나중에 — 지금은 5인 미만 / 5–9 / 10–29 /
  30–299 / 300+ 중 어느 밴드인지.) **산정은 다툼이 잦아 검토 스킬이 `[검토]`로 플래그하고 법망 현행
  시행령에 대조합니다.**
- **사업장은 몇 개소, 어디에 있나요?** (사업장별로 상시 근로자 수·취업규칙 적용이 달라질 수 있음.)
- **노동조합이 있나요? 노사협의회는?** (노조 → 부당노동행위·단체교섭 이슈. 30인 이상 → 노사협의회 의무.)
- **취업규칙은 작성·신고되어 있나요?** (10인 이상 작성·신고 의무 §93. 있으면 시드 문서로 받습니다.)
- **해외 사업장·해외 파견·외국인 근로자가 있나요?** (있으면 외국법 overlay — `[외국법 — 외국 변호사
  검증 필요]`로 플래그하고 현지 노동 전문가 자문 권고.)

**적용범위 게이트.** 상시 근로자 수 밴드를 `## 우리는 누구인가`와 `## 상시 근로자 수 적용범위 overlay`
참조에 기록. 5인 미만이면 명시: "5인 미만은 근기법 다수 조항(해고 제한·가산수당·연차)이 적용 제외될 수
있으나, 산정 결과에 따라 달라집니다 — 검토 시 `[상시 근로자 수 산정 — 검증 필요]`로 플래그합니다."

### Part 2: 검토 트리거 + 입장 (3-4분)

> "**입장을 지금 구축하시겠어요?** 검토 스킬(hiring-review·termination-review·policy-drafting)이
> 훨씬 나아집니다 — 일반 입장 대신 당신 stance와 폴백을 압니다. 3-4분. 다른 커맨드만 써보고 싶으면
> skip — 검토 스킬이 디폴트를 쓰고 미설정 입장에 부딪히면 알려줍니다."

> 팀 공유 인사규정·징계 양정표·delegation 정책이 있으면 그걸 원합니다 — paste 또는 link. baseline으로
> 쓰고 개인 override는 따로 묻겠습니다.

> 질문 전: 해고·징계 체크리스트, 표준 근로계약서, 인사규정이 있으면 읽겠습니다. paste·파일 경로, 또는
> '없음' 말씀. 주시면 트리거와 에스컬레이션 지점을 추출합니다.

없으면:

**채용:** 법무·노무가 언제 채용을 보나?
- 모든 채용? 임원만? 경업금지 포함 시? 기간제·파견·외국인만? 안 봄?
- 표준 근로계약서에 무엇이 있나? (수습/시용 기간·평가, 근로조건 명시 §17 항목, 경업금지·비밀유지.)
  경업금지는 **직업선택의 자유와 형량** — 유효/무효 단정 없이 `[검토]`.

**해고·징계:** 법무·노무가 언제 해고를 보나?
- 모든 해고? 징계해고만? 경영상 해고(정리해고)만?
- 표준 위로금·해고예고(§26)·서면통지(§27) 관행은? 합의서(부제소 합의)는 항상, 아니면 일정 조건 이상?

**해고를 위험하게 만드는 플래그:** 무엇이 해고를 무섭게 만드나? (이건 /termination-review에 영향 —
모든 해고 검토가 결론 전 이 플래그들에 대조됩니다. 부당해고·불이익처우 리스크.)
- 노동조합 활동·조합원 (부당노동행위 §81 리스크)
- 산재 요양 중·산재 신청자 (요양기간 해고 제한 §23②)
- 육아휴직·출산휴가 중·복귀 직후 (남녀고용평등법 불리한 처우 금지)
- 직장 내 괴롭힘·성희롱 신고자 (불리한 처우 금지 §76-3)
- 차별 진정·내부고발자
- 징계사유·저성과 입증자료 부족
- 그 외 과거에 문제 됐던 것?

**임금·근로시간 입장:** 포괄임금제 사용 여부·범위? 통상임금에 무엇을 포함하나(정기상여금 등 — 2024
전합 이후 재검토 필요)? 연장근로 한도(주 12시간)·유연근로 운영?

문서 요청 전, 인프라 질문 하나:

> HRIS — Workday·flex·시프티 등 — 가 직원 휴가를 추적하나요? 법무·인사가 read 접근 권한이 있나요?
> (이건 /leave-tracker·/log-leave에 영향 — HRIS 접근이 있으면 휴가를 자동으로 pull, 없으면 수동
> 갱신하는 로컬 등록부로 동작.)

- HRIS + 접근 가능: 시스템명 기록
- HRIS 있으나 접근 불가, 또는 휴가 모듈 없음: "manual" 기록
- HRIS 없음: "manual" 기록

**시드 문서**

> 가장 중요한 부분 — 정책이 뭐라고 하는지가 아니라 팀이 실제로 어떻게 일하는지 보고 싶습니다. 두 가지:
>
> 1. **취업규칙.** 현행 버전. 직원에게 무엇을 약속했고 갭이 어디인지 읽습니다. (이건 /policy-drafting·
>    /handbook-updates·/hiring-review에 영향 — 모든 정책 초안·근로계약 점검이 취업규칙이 이미 약속한
>    것과 cross-reference됩니다.)
>
> 2. **최근 인사 문서 — 많을수록 좋음.** 10건이 좋은 하한, 20건이면 훨씬 명확. 섞어서: 해고·징계 메모,
>    근로계약서, 합의서, PIP(개선계획), 휴직·복직 처리 — 있는 대로. 10건 미만이면 주는 만큼, 단 플래그.
>    (이건 /termination-review·/hiring-review에 영향 — 일반 템플릿이 아닌 실제 문서에서 하우스 포맷·
>    위로금 관행·high-risk 패턴을 추출합니다.)

HRIS·문서 가시성이 좋으면 위 유형에 걸쳐 10-20건. 가시성이 나쁘면(흩어진 폴더·무시스템) 줄 수 있는
만큼 받고, 10건 미만으로 구축된 실무 프로파일 섹션마다 `[LIMITED DATA — N건 검토]` 플래그.

**취업규칙에서:** 상시 근로자 수 임계점 관련 조항, 징계 사유·양정·절차, 연차·휴직 운영, 임금체계,
경업금지·비밀유지. 갭 — 취업규칙이 다뤄야 하는데 안 다루는 것.

**시드 문서에서:** 해고에서 무엇을 점검했나. high-risk 플래그가 실제로 어떻게 보이나. 근로계약 포맷과
표준 경업금지·비밀유지 문언. 합의서 포맷(termination-review가 매칭). 정책이 말하는 것 vs 팀이 실제로
승인하는 것의 패턴.

## 적용범위·에스컬레이션 baseline 구축

이게 핵심 출력이다. 상시 근로자 수와 사업장 footprint로부터 적용범위·에스컬레이션 baseline을 합성:

| 영역 | 적용 여부(상시 근로자 수) | 자동 에스컬레이션 |
|---|---|---|
| 해고 제한 §23 / 구제신청 §28 | 5인 이상 적용 (산정 `[검토]`) | 징계해고·경영상 해고 |
| 가산수당 §56 / 연차 §60 | 5인 이상 적용 | — |
| 취업규칙 작성·신고 §93 | 10인 이상 의무 | 불이익변경(§94 동의) |
| 노사협의회 | 30인 이상 의무 | 단체교섭·쟁의 |
| [사업장별 특이사항·노조] | | |

footprint에 없는 임계점의 규칙을 발명하지 않는다. 사업장이 5인 미만 밴드면 명시:
`[5인 미만 — 적용 제외 범위는 산정 결과·시행령 별표 대조 필요]`. 해외 사업장은 `[외국법 — 외국 변호사
검증 필요]`. 이 baseline을 `## 상시 근로자 수 적용범위 overlay`·`## 에스컬레이션`에 기록.

## 실무 프로파일 작성

플러그인 디렉토리의 `CLAUDE.md` 템플릿 구조를 사용. 모든 답을 적절한 섹션에 채우기. `[PLACEHOLDER]`
마커는 사용자가 의도적으로 skip한 항목에만 남김(silent 갭이 아니라 명시적 결정).

자동화 가능한 검증:
- 사용자가 인용한 근기법·최저임금법·남녀고용평등법·노조법 조항은 `mcp__beopmang__law get`으로 현행
  텍스트 가져와 대조
- 사용자가 인용한 판례는 `mcp__beopmang__case search`로 존재 확인(통상임금 2024 전합 등)
- 사용자가 인용한 **노동위원회 판정례·고용노동부 행정해석은 *법망 미커버*** — 사용자에게 공식 URL·PDF
  부탁(`docs/DATA_GAPS.md` 참조), `[노동위 판정 — 직접 검증 필요]` / `[고용부 행정해석 — 직접 검증 필요]` 태그

작성 후, `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/verification-log.md`에 추가:

`[YYYY-MM-DD] 콜드스타트 인터뷰 - [성명]이(가) 다음을 확인: [검증한 항목 목록] - 법망에 대조해 [확인 / 정정]`

## 작성 후

**이 플러그인이 무엇을 도와줄 수 있는지 보여주기.** 닫기 전 제안:

> **제가 무엇을 도와드릴 수 있는지 보시겠습니까?**

yes면 다음 맞춤 목록 보여줌 (일반 템플릿이 아님 — 이 플러그인이 가장 잘하는 구체적인 것들):

> **근로·고용 실무에서 제가 잘하는 것:**
>
> - **해고·징계 검토** — 예: "§23 정당한 이유 + 절차(예고·서면통지·징계절차), high-risk 플래그를 결정
>   전 점검." 시도: `/employment-legal:termination-review`
> - **근로자성 판단** — 예: "프리랜서·도급 vs 근로자, 오분류 시 4대보험·퇴직금 소급 갭." 시도:
>   `/employment-legal:worker-classification`
> - **임금·근로시간 Q&A** — 예: "통상임금·가산수당·주52시간·최저임금, 상시 근로자 수 기준." 시도:
>   `/employment-legal:wage-hour-qa`
> - **근로계약·채용 검토** — 예: "근로조건 명시·수습·경업금지(직업선택의 자유 형량)." 시도:
>   `/employment-legal:hiring-review`
> - **취업규칙 작성·변경** — 예: "불이익변경 동의(§94)·신고(§93) 점검." 시도:
>   `/employment-legal:policy-drafting`
> - **노동위 구제신청 절차** (KR 전용) — 예: "부당해고·부당노동행위 구제신청 흐름·기한." 시도:
>   `/employment-legal:labor-committee-procedure`
> - **휴가·휴직 기한 관리** — 예: "연차·육아휴직·출산휴가 법정 기한 추적." 시도:
>   `/employment-legal:leave-tracker`
>
> **첫 번째 추천:** 가상의 해고 사안에 `/employment-legal:termination-review` 실행 — 위험 calibration이
> 어떻게 읽히는지 가장 빠르게 보는 방법. 또는 당신 plate에 무엇이 있는지 말씀, 제가 고르겠습니다.

감독자가 인터뷰 중 구체적 첫 작업을 이미 명명했으면 이 단계 skip. (참고: 일부 스킬은 포팅 진행 중 —
`skills/ROADMAP.md`. 미포팅 스킬을 제안하지 않는다.)

1. **요약 보여주기.** "들은 것을 정리했습니다. 상시 근로자 수 적용범위와 해고 위험 플래그가 가장 강하게
   점검할 부분 — 맞게 잡았나요?"

2. **법망 검증 prompt.** 말함:
   > "첫 검토 전: 법망 MCP를 연결하세요. 없으면 모든 조문 인용을 미검증으로 플래그합니다 — 있으면
   > 근기법·최저임금법·남녀고용평등법 현행 DB에 대조 검증합니다. Claude Code:
   > `claude mcp add beopmang https://api.beopmang.org/mcp --transport http`."

3. **첫 작업 제안:**
   - "큐에 검토할 해고·징계 건이나 근로계약이 있나요?"
   - 휴가·휴직 관리가 필요하면: "휴가 등록부를 시작할까요? 육아휴직 신청기한·연차 사용촉진 기한을
     놓치는 것을 막습니다."
   - 취업규칙이 오래됐으면: "취업규칙을 근기법 현행 + 상시 근로자 수 기준 대조 점검해드릴까요?"

4. **갭 플래그:** 취업규칙이나 해고 메모를 못 줬으면 메모: "취업규칙이 비어 있습니다 — 정책 작성·변경
   검토가 일반 디폴트로 동작합니다. 받으면 훨씬 정확해집니다."

5. **"나중에 뭐든 바꿀 수 있음" 메모로 닫기:**
   > "실무 프로파일은 `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md`에
   > 있습니다 — 직접 읽고 편집 가능한 평문 파일. 답한 것은 무엇이든 바꿀 수 있습니다:
   >
   > - 빠른 변경은 파일 직접 편집
   > - `/employment-legal:cold-start-interview --redo`로 전체 재인터뷰
   > - `/employment-legal:cold-start-interview --check-integrations`로 연결 재점검
   > - 또는 한 항목만: `/employment-legal:customize`
   >
   > 사람들이 가장 자주 조정하는 세 가지: **상시 근로자 수 적용범위**(인원이 임계점을 넘나들 때),
   > **해고 위험 플래그**(무엇이 실제로 위험인지 calibrate되며), **에스컬레이션 임계점**(보고 라인이
   > 명확해질수록)."

6. **실무 프로파일은 학습합니다.** 다음 메모로 종료:
   > **실무 프로파일은 학습합니다.** 플러그인을 쓸수록 좋아집니다:
   >
   > - 스킬 출력이 부자연스러우면 보통 튜닝할 입장입니다. 출력이 어느 것인지 알려줍니다.
   > - 언제든 "플레이북을 X를 선호하도록 업데이트" 또는 "에스컬레이션 임계점을 Y로 변경" 말씀하시면
   >   관련 스킬이 변경을 기록합니다.
   > - `/employment-legal:cold-start-interview --redo <섹션>`으로 한 부분만 재인터뷰, 또는 config
   >   파일 직접 편집.
   >
   > 10-15분 셋업이 동작하는 프로파일을 줍니다. 한 달 사용이 직접 쓴 것처럼 읽히는 프로파일을 줍니다.

## 실패 모드

- **상시 근로자 수를 가정하지 마라.** 밴드가 불확실하면 묻고, 산정은 `[검토]`로 플래그. 적용범위가
  모든 후속 분석의 게이트다 — 잘못 잡으면 매번 잘못된 적용.
- **at-will을 전제하지 마라.** 한국에 임의해고는 없다. "그냥 해고하면 된다"는 답에도 §23 정당한 이유 +
  절차를 surface하고 `[검토]`.
- **일반 입장에서 플레이북을 쓰지 마라.** 경험이 적으면 config에 명시: `[POSITIONS UNTESTED — 이
  사업장은 아직 많은 사건을 다루지 않음. 출발점이지 settled 입장이 아니라고 취급.]`
- **노동위 판정례·고용부 행정해석을 인용하면 법망에 없다고 명시.** `[노동위 판정 — 직접 검증 필요]` /
  `[고용부 행정해석 — 직접 검증 필요]` 태그 (`docs/DATA_GAPS.md`).
- **해외 사업장·외국인 근로자·외국 준거법을 한국 프레임워크로 단정하지 마라.** `[외국법 — 외국 변호사
  검증 필요]` 플래그하고 현지 노동 전문가 자문 권고.
