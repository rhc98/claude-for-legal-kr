# /investigation-query

## 매터 컨텍스트 (활성 매터 필수)

실무 수준 CLAUDE.md `## 매터 워크스페이스` 확인. 활성 매터가 없으면:

> "어느 사건입니까? `/employment-legal:matter-workspace switch <slug>`로 전환하세요."

`incident-log.md`가 없으면: "이 매터의 incident-log가 없습니다.
`/employment-legal:investigation-open`으로 사건을 먼저 개시하세요."

매터 간 컨텍스트가 `on`이 아니면 다른 매터 파일 절대 안 읽음.

---

## 목적지 점검

현황 요약도 조사 자료의 일부다. 비밀누설 금지 의무(근기법 §76-3⑧·남녀고용평등법 §14⑥) 적용.
`[모델 지식 — 검증 필요]`

수신자:
- **허용:** 조사자·법무담당·HR 책임자·경영진(Need-to-know)
- **불가:** 당사자 직접 전달, 공개 채널, 전사 배포, 외부

현황 요약을 경영진 보고용으로 sanitize해야 하면 별도 버전 제공. `[검토]`

---

## 목적

`incident-log.md`를 읽고 조사 진행 상황을 구조화된 현황 요약으로 제공한다.
법정 의무 기한 경과 여부와 미완료 항목을 명시해 조사자가 다음 행동을 결정하게 한다.

---

## 현황 요약 템플릿

`incident-log.md` 로드 후 아래 항목을 채운다.

```markdown
## 조사 현황 요약 — [매터-slug]

**기준 일자:** [YYYY-MM-DD]

---

### 기본 정보

| 항목 | 내용 |
|------|------|
| 사건 번호 | [매터-slug] |
| 신고 유형 | [괴롭힘 / 성희롱 / 복합] |
| 신고 접수일 | [YYYY-MM-DD] |
| 경과 일수 | [N]일 |
| 조사 단계 | [개시 / 면담 진행 중 / 증거 검토 / 보고서 작성 / 완료] |
| 담당 조사자 | [성명·직책] |

---

### 완료 항목

- [x] [완료된 항목 — 날짜와 함께]
- [x] ...

### 미완료 항목

- [ ] [미완료 항목 — 이유 또는 예정 일자]
- [ ] ...

---

### 기한 경과 여부

| 의무 | 기준 | 상태 |
|------|------|------|
| 조사 개시 | 신고 즉시 (실무상) `[검토]` | ✓ 완료 / ⚠️ 지체 [N일] |
| 피해자 보호조치 | 신고 즉시 `[검토]` | ✓ 완료 / ⚠️ 미이행 |
| 비밀준수 서약 | 조사 개시 전 | ✓ 완료 / ⚠️ 미완료 |
| 조사 결과 조치 | 조사 완료 후 지체 없이 `[검토]` | - 진행 중 |

> 법정 기한이 명시된 경우 법망 대조 필요. `[모델 지식 — 검증 필요]`
> ```

> mcp__beopmang__law get "남녀고용평등법 제14조"
> ```

## 기한 알림

조사가 지체 또는 의무 미이행 상태이면 다음을 명시한다:

**조사 지체 경고:**
> ⚠️ 신고 접수 후 [N]일이 경과했습니다. "지체 없이" 조사 의무(근기법 §76-3·남녀고용평등법 §14)
> 위반 위험이 있습니다. `[모델 지식 — 검증 필요]` 지체 사유와 향후 일정을 기록해 두세요.

**보호조치 갱신 필요:**
> ⚠️ 보호조치 이행 후 [N]일이 경과했습니다. 피해자 상황 변화 여부·보호조치 유지 필요성을
> 재확인하는 것을 권고합니다. `[검토]`

**비밀준수 미완료:**
> ⚠️ 조사 관여자 중 비밀준수 서약을 받지 않은 인원이 있습니다. 즉시 완료하세요.
> 위반 시 근기법 §76-3⑧ 과태료 대상. `[모델 지식 — 검증 필요]`
