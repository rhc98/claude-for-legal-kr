# /leave-tracker-agent

가드레일·헤더·태그 어휘·결정 자세·산출물 포맷은 모두 이 플러그인의
`~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md`가 정본이다 — 충돌 시 그 CLAUDE.md가 우선한다.

## 목적

`leave-register.yaml`을 주기적으로 스캔하여 임박한 법정 기한을 설정된 목적지로 알린다.

**추적 대상 기한 (한국 법 기준):**
- 연차사용촉진 1차 촉구 기한 (§61 — 소멸 6개월 전)
- 연차사용촉진 2차 지정 기한 (§61 — 1차 촉구 후 10일, 소멸 2개월 전)
- 출산전후휴가 종료 예정일·복직 예정일 (§74)
- 배우자 출산휴가 사용 마감일 (남녀고평법 §18-2 — 출산일 + 90일)
- 육아휴직 신청 기한 (남녀고평법 §19 — 시작 희망일 - 30일)
- 육아휴직 종료일·복직 예정일 (§19)
- 육아기 근로시간 단축 종료일·복직 예정일 (§19-2)
- 해고 금지 기간 종료 예정일

> **한국 법체계 포인트.** FMLA·CFRA·USERRA·ADA는 이 에이전트의 추적 대상이 아니다.
> 한국은 전국 단일 법 — 주별(state) 보충 규정이 없다. `[모델 지식 — 검증 필요]`

## 설정 확인 (`--config`)

`~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md`의
`## 휴가·휴직 등록부 설정` 블록을 읽는다:

```yaml
# CLAUDE.md ## 휴가·휴직 등록부 설정 블록 예시
register_path: "~/.claude/plugins/config/claude-for-legal-kr/employment-legal/leave-register.yaml"
alert_destination: "slack:#hr-alerts"   # slack:#채널명 | email:주소 | inline
lead_times_days: [30, 14, 7]            # 알림 발송 리드타임 (일)
agent_schedule: "weekly-monday-09:00"   # 에이전트 실행 스케줄 (참고용)
```

**`--config` 실행 시 출력:**

```markdown
## leave-tracker-agent 설정 현황

| 항목 | 값 | 상태 |
|---|---|---|
| 등록부 경로 | [경로] | [파일 존재 ✓ / 없음 ✗] |
| 알림 목적지 | [목적지] | [설정됨 ✓ / 미설정 — 인라인 출력] |
| 리드타임 | [N]일 | [설정됨 ✓ / 디폴트 [30,14,7] 사용] |
| 등록부 항목 수 | [N]건 | |
| 임박 기한 항목 | [N]건 (오늘 기준 [리드타임]일 이내) | |

**추천 행동:**
- [미설정 항목이 있으면: "CLAUDE.md ## 휴가·휴직 등록부 설정 블록을 작성하세요."]
- [등록부가 비어 있으면: "/employment-legal:log-leave로 첫 휴가를 등록하세요."]
- [모두 설정됨이면: "/employment-legal:leave-tracker-agent --dry-run으로 알림 미리보기"]
```

**목적지가 미설정(`alert_destination` 없음)이면:** 알림은 인라인 출력. 에이전트는 동작하나
자동 발송은 하지 않는다.

## 에이전트 실행 모드

`$ARGUMENTS`의 첫 토큰으로 디스패치:

### `--run` — 지금 즉시 실행

1. 설정 확인 (등록부 경로 + 알림 목적지)
2. leave-register.yaml 읽기
3. 오늘 날짜 기준 각 항목의 다음 법정 기한 계산
4. `lead_times_days` 이내 기한 → 알림 생성
5. 설정된 목적지로 발송 (또는 인라인 출력)
6. 발송 결과 보고

### `--dry-run` — 알림 미리보기

`--run`과 동일하게 스캔·계산·알림 생성하되 **발송하지 않고** 인라인으로 미리보기만 보여준다.
발송 전 내용을 확인할 때 사용.

### `--config` — 설정 확인

위 "설정 확인" 섹션 실행.

### 인수 없음

`--dry-run`으로 동작 (안전한 디폴트 — 발송 없이 미리보기).

## 스캔 로직

```
1. leave-register.yaml 읽기
   └── 파일 없음 → "등록부가 없습니다. /employment-legal:log-leave로 첫 항목을 등록하세요."
   └── 항목 없음 → "등록부가 비어 있습니다."

2. 오늘 날짜(KST) 기준으로 각 항목의 다음 법정 기한 계산:
   - status가 "종료"이면 skip
   - 각 항목당 계산 대상 기한 목록 (아래 "알림 항목별 내용" 참조)
   - 기한별 D-day = 기한일 - 오늘

3. D-day ≤ max(lead_times_days)인 항목 → 알림 목록에 포함
   - 🔴 D-7 이하 (가장 가까운 리드타임 이내)
   - 🟠 D-8~14
   - 🟡 D-15~30
   (리드타임은 CLAUDE.md lead_times_days 설정 우선)

4. 알림 목록이 비어 있으면:
   → "임박한 기한이 없습니다. (조회 기준: 오늘 + [N]일)"

5. 알림 목록이 있으면 → 목적지별 포맷으로 출력·발송
```

**기한 계산은 `[모델 계산 — 법조항 대조 검증 필요]` 태그.** 계산 오류가 의심되면
`/employment-legal:leave-tracker --list`로 상세 확인.

## 에이전트 자동 설정 안내

이 스킬 자체는 수동 실행(`--run`)과 미리보기(`--dry-run`)를 지원한다.
**정기 자동 실행**을 위해서는 외부 스케줄러가 필요하다:

**Claude Code 예약 실행:**
```bash
# 매주 월요일 오전 9시에 실행 (cron 또는 /schedule 스킬 활용)
# /schedule 스킬: /employment-legal:leave-tracker-agent --run
```

또는 `/schedule` 스킬로 주간 루틴을 등록:
> `/schedule` → "매주 월요일 09:00에 `/employment-legal:leave-tracker-agent --run` 실행"

자동 실행 스케줄이 없어도 이 스킬을 수동으로 돌리면 언제든 기한 점검이 가능하다.

## 의사결정 트리

> **다음 단계? 하나 골라주세요:**
> 1. **즉시 대응 필요 항목 처리** — 🔴 항목 중 필요 서면 초안 작성 (연차촉진 1차·2차).
> 2. **leave-tracker 상세 조회** — `/employment-legal:leave-tracker --list`로 등록부 전체 확인.
> 3. **새 항목 등록** — `/employment-legal:log-leave`로 휴가·휴직 추가.
> 4. **설정 변경** — CLAUDE.md `## 휴가·휴직 등록부 설정` 블록 수정 (알림 목적지·리드타임 조정).
> 5. **에스컬레이션** — 기한 초과·절차 미이행 위험 항목을 CLAUDE.md 에스컬레이션 담당자에게 보낼
>    메모 초안.
> 6. **다른 것** — 무엇을 하고 싶은지 알려주세요.

## 이 스킬이 하지 않는 것

- 연차사용촉진 서면을 자동으로 발송하지 않는다 — 알림만; 초안은 사용자가 요청.
- 육아휴직급여 금액을 계산하지 않는다 — 고용보험 사이트 안내.
- leave-register.yaml을 자동으로 수정하지 않는다 — 읽기 전용 스캔.
- 해고 금지 위반 여부를 검토하지 않는다 — `/employment-legal:termination-review`로 핸드오프.
- 법적 요건 충족 여부를 단정하지 않는다 — `[검토]` 플래그 후 사용자 판단.

## 실패 모드

- **등록부 없음:** "leave-register.yaml이 없습니다. CLAUDE.md `register_path`를 확인하거나
  `/employment-legal:log-leave`로 첫 항목을 등록하세요."
- **목적지 미설정:** 인라인 출력으로 fallback. 에러 처리 안 함.
- **날짜 파싱 실패:** 해당 항목을 skip하고 플래그: "[항목 ID] — 날짜 형식 오류, 확인 필요."
- **Slack·이메일 발송 실패:** `--dry-run`으로 내용을 인라인 출력하고 수동 발송 안내.
