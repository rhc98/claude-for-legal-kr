# /log-leave

이 스킬은 `/employment-legal:leave-tracker --add` 또는 직접 호출로 진입한다.

가드레일·헤더·검토자 메모·태그 어휘·결정 자세·산출물 포맷은 모두 이 플러그인의
`~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md`가 정본이다 — 충돌 시 그 CLAUDE.md가 우선한다.

## 매터 컨텍스트

`~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md` 읽기.
- 매터 워크스페이스가 활성이면: 활성 매터의 `matter.md`를 읽고 등록부 경로를 확인.
- 사내 사용자: 실무 수준 CLAUDE.md에서 `## 휴가·휴직 등록부 설정` 블록을 읽어
  `register_path`·`lead_times_days`를 가져온다.
- CLAUDE.md가 없거나 `[PLACEHOLDER]`이면: 멈추고 `/employment-legal:cold-start-interview`로 안내.

## 목적지 점검

등록할 근로자명·휴직 사유는 개인정보다.
- 등록부 저장 경로(`register_path`)가 공유 파일시스템·클라우드 드라이브에 있는 경우 접근 권한 확인.
- 익명 코드로 관리하는 사업장은 실명 대신 코드 입력 권장 — 사용자에게 한 번 안내.
- 산출물(YAML 블록)을 복사해 공유할 때 목적지가 비밀유지 서클 내부인지 확인.

## 등록 정보 수집

한 번에 2-3개 이상 묻지 않는다. 아래 순서로 수집:

### Step 1: 기본 정보

> 다음 정보를 알려주세요 (2-3가지씩 묻겠습니다):
>
> 1. **근로자 이름 또는 익명 코드** — 개인정보 보호를 위해 코드 사용도 가능합니다.
> 2. **휴가 종류** — 아래 중 하나:
>    - 연차유급휴가 (근기법 §60)
>    - 출산전후휴가 (§74)
>    - 배우자 출산휴가 (남녀고평법 §18-2)
>    - 육아휴직 (남녀고평법 §19)
>    - 육아기 근로시간 단축 (§19-2)
>    - 기타 법정휴가 (종류 명시)
> 3. **시작일** (YYYY-MM-DD)

### Step 2: 기간·특이사항

휴가 종류에 따라 다르게 묻는다:

**연차유급휴가:**
> 4. **연도 및 총 발생 일수** (예: "2025년, 15일") — 근속연수·출근율은 아래에서 자동 계산 안내.
> 5. **이미 사용한 일수** (없으면 0)
> 6. **소멸일** — 취업규칙상 정함이 있으면 그 날짜; 없으면 발생일로부터 1년 후.

**출산전후휴가:**
> 4. **예정 출산일 또는 실제 출산일** (YYYY-MM-DD)
> 5. **다태아 여부** (예/아니오) — 다태아이면 120일, 단태아이면 90일.
> 6. **휴가 시작일** — 출산 전·후 중 언제부터?

**배우자 출산휴가:**
> 4. **배우자(근로자)의 배우자 출산일** (YYYY-MM-DD)
> 5. **사용 예정일 범위** — 분할 사용 예정이면 각 기간.

**육아휴직:**
> 4. **자녀 생년월일** — 만 8세 이하·초등 2학년 이하 요건 확인용.
> 5. **신청 희망 시작일·종료일** (YYYY-MM-DD)
> 6. **신청서 제출일** — 신청 30일 전 기한 충족 여부 확인용. (아직 미제출이면 "미제출"로)

**육아기 근로시간 단축:**
> 4. **자녀 생년월일**
> 5. **단축 시작일·종료일** 및 **주 소정근로시간** (15~35시간 범위)

**기타:**
> 4. **시작일·종료일**
> 5. **법적 근거** (알고 있는 경우)

### Step 3: 담당·특이사항

> 마지막으로:
> - **담당자** (인사팀·법무팀 등, 없으면 공백)
> - **특이사항** (예: 육아휴직 분할 사용, 단체협약상 가중 절차, 익명 처리 필요 등)

## 법정 사항 자동 계산

수집된 정보를 바탕으로 종류별 법정 기한을 계산한다. 모든 계산 결과에
`[모델 계산 — 해당 법조항 대조 검증 필요]` 태그.

### 연차유급휴가

| 항목 | 계산 | 값 |
|---|---|---|
| 소멸일 | 입력값 또는 발생일 + 1년 | [날짜] |
| 연차사용촉진 1차 촉구 기한 | 소멸일 - 6개월 | [날짜] |
| 연차사용촉진 2차 지정 기한 | 1차 촉구일 + 10일 (단, 소멸일 - 2개월 이전) | [날짜] |
| 잔여 일수 | 총 발생 - 사용 | [N]일 |

> **연차사용촉진 절차 미이행 리스크.** 1차·2차 기한을 모두 충족해야 미사용수당 지급 의무가 면제됩니다.
> 어느 하나라도 누락하거나 기한이 지나면 의무가 살아납니다. `[모델 지식 — 국가법령정보 또는 고용부 행정해석 대조]`

### 출산전후휴가

| 항목 | 계산 | 값 |
|---|---|---|
| 법정 기간 | 단태아 90일 / 다태아 120일 | [N]일 |
| 종료 예정일 | 시작일 + 법정 기간 | [날짜] |
| 출산 후 최소 부여 | 단태아 45일 / 다태아 60일 이상 | 요건 [충족/미충족] `[검토]` |
| 복직 예정일 | 종료 예정일 + 1일 | [날짜] |
| 해고 금지 기간 | 휴가 기간 + 종료 후 30일 (§74⑦) `[검증 필요]` | ~[날짜]까지 |

> **유급 처리.** 최초 60일(다태아 75일)은 사용자 부담 유급 원칙. 우선지원대상기업은 고용보험에서
> 지원 — 구체 금액·요건은 고용보험 사이트(https://www.ei.go.kr) 확인. **금액 하드코딩 금지.**
> `[모델 지식 — 검증 필요]`

### 배우자 출산휴가

| 항목 | 계산 | 값 |
|---|---|---|
| 사용 마감일 | 출산일 + 90일 | [날짜] |
| 잔여 사용 가능 기간 | 마감일 - 오늘 | D-[N] |
| 부여 일수 | 10일 유급 `[모델 지식 — 2024년 기준, 검증 필요]` | 10일 |

### 육아휴직

| 항목 | 계산 | 값 |
|---|---|---|
| 신청 30일 전 기한 | 시작 희망일 - 30일 | [날짜] |
| 신청 기한 충족 여부 | 제출일이 기한 이전인지 | [충족 / 미충족 / 미제출] `[검토]` |
| 자녀 나이 요건 | 만 8세 이하·초등 2학년 이하 | [충족 / 미충족] `[검토]` |
| 종료일 | 입력값 | [날짜] |
| 복직 예정일 | 종료일 + 1일 | [날짜] |
| 해고 금지 기간 | 신청 시부터 복직 후 일정 기간 (§19④) `[검증 필요]` | ~[날짜]까지 |

> **육아휴직급여.** 고용보험에서 지급. 구체 금액·지급률은 수시로 변경됩니다 — 반드시
> 고용보험 사이트(https://www.ei.go.kr) 또는 고용노동부 고객상담센터(1350)에서 확인.
> **이 스킬은 금액을 하드코딩하지 않습니다.**

> **해고 금지.** 육아휴직 중 해고는 원칙적으로 금지(남녀고평법 §19④). 해고 검토가 필요한 경우
> `/employment-legal:termination-review`로 핸드오프. `[모델 지식 — 검증 필요]`

## leave-register.yaml 업데이트 안내

계산 결과를 바탕으로 등록부에 추가할 YAML 블록을 생성한다. 사용자가 직접 복사해 붙여넣는다.
(자동 쓰기가 아님 — 사용자 확인 후 붙여넣기.)

```yaml
# 아래 블록을 leave-register.yaml의 leaves: 목록 하단에 추가하세요.
- id: "[EMP코드]-[종류]-[YYYY]"
  employee: "[근로자명 또는 익명 코드]"
  leave_type: "[연차 | 출산전후휴가 | 배우자출산휴가 | 육아휴직 | 육아기근로시간단축 | 기타]"
  start_date: [YYYY-MM-DD]
  end_date: [YYYY-MM-DD]
  status: "진행중"
  multiple_birth: [true | false]       # 출산전후휴가 시
  annual_days_total: [N]               # 연차 시
  annual_days_used: [N]
  annual_days_remaining: [N]
  expiry_date: [YYYY-MM-DD]           # 연차: 소멸일
  promotion_1st_deadline: [YYYY-MM-DD]  # 연차: 1차 촉구 기한
  promotion_1st_sent: false
  promotion_2nd_deadline: [YYYY-MM-DD]  # 연차: 2차 지정 기한
  promotion_2nd_sent: false
  reinstatement_date: [YYYY-MM-DD]    # 복직 예정일 (출산휴가·육아휴직)
  dismissal_prohibition_end: [YYYY-MM-DD]  # 해고 금지 종료 예정일
  application_deadline: [YYYY-MM-DD]  # 육아휴직: 신청 마감
  owner: "[담당자]"
  notes: "[특이사항]"
  provenance: "[모델 계산 — 법조항 대조 검증 필요]"
```

> **이 블록을 붙여넣은 후** `/employment-legal:leave-tracker --list`로 등록 결과를 확인하세요.

## 알림 등록 안내

leave-tracker-agent가 설정되어 있으면, 이 등록 항목의 기한을 자동 알림에 포함시키기 위해
별도 설정이 필요하지 않다 — 에이전트는 등록부 전체를 스캔한다.

에이전트가 미설정이면:
> `/employment-legal:leave-tracker-agent --config`로 주간 알림 에이전트를 설정하면
> 이 기한들을 자동으로 추적합니다.

추적이 필요한 기한 목록:
[계산된 기한들을 bullet로 나열 — 예:
- 연차사용촉진 1차 촉구 기한: [날짜] (D-[N])
- 복직 예정일: [날짜] (D-[N])]

## 이 스킬이 하지 않는 것

- leave-register.yaml을 자동으로 덮어쓰지 않는다 — YAML 블록 제공 후 사용자가 붙여넣기.
- 육아휴직급여 금액을 계산하지 않는다 — 고용보험 사이트 안내.
- 법적 요건 충족 여부를 단정하지 않는다 — `[검토]` 플래그 후 사용자 판단.
- 해고 금지 위반 여부를 검토하지 않는다 — `/employment-legal:termination-review`로 핸드오프.
