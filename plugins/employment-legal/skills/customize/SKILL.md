# /customize

## 실행 조건

사용자가 `/employment-legal:customize` 입력. 콜드스타트 인터뷰 전체 재실행 없이, 파일 수동 편집
없이 실무 프로파일의 무엇 — 상시 근로자 수 적용범위·에스컬레이션·플레이북 입장·해고 위험 플래그·
하우스 스타일·산출물 형식 — 을 변경하고 싶음.

## 할 일

1. **config 읽기.** `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/CLAUDE.md`
   (와 한 단계 위 `~/.claude/plugins/config/claude-for-legal-kr/company-profile.md`) 읽기.
   플러그인 config 없거나 `[PLACEHOLDER]` 값 남아 있으면:

   > 셋업을 아직 안 하셨습니다. 먼저 `/employment-legal:cold-start-interview` 실행 —
   > customize는 이미 가진 프로파일을 조정하기 위한 것.

2. **맞춤화 가능 맵 보여주기.** 프로파일에 있는 것을 그룹화, 현재 값의 한 줄 요약과 함께:

   - **회사 / 우리는 누구인가** — 회사명·업종·상시 근로자 수·사업장·노조·취업규칙·실무 환경
     *(회사명·업종·규모·위험 성향은 모든 플러그인 공유 — `company-profile.md` 통해 변경 전파)*
   - **누가 이 플러그인을 사용하나** — 변호사 / 공인노무사 / 비전문가 (work-product 헤더·출력 framing 영향)
   - **플레이북 — 해고·징계** — 정당성 기준·절차 원칙·자동 에스컬레이션·절대 안 됨
   - **플레이북 — 채용·근로계약** — 표준 근로계약 입장·경업금지/비밀유지
   - **플레이북 — 임금·근로시간** — 임금체계·포괄임금제·근로시간 입장
   - **플레이북 — 휴가·휴직 / 괴롭힘·성희롱 / 근로자성** — 각 영역 하우스 입장
   - **해고 위험 플래그** — termination-review가 결론 전 점검하는 부당해고·불이익처우 리스크 목록
   - **상시 근로자 수 적용범위 overlay** — 적용 게이트가 되는 임계점·사업장 특이사항
   - **에스컬레이션** — 승인 매트릭스·임계점·자동 에스컬레이션 트리거
   - **하우스 스타일** — 검토 톤·이해관계자 요약·산출물 보관 위치·알림 목적지
   - **검토 선호** — `confirm_routing` 등 라우팅 동작
   - **휴가·휴직 등록부 설정** — `register_path`·`alert_destination`·`lead_times_days`
   - **내부 조사 설정** — `log_path`·`confidentiality`
   - **통합** — HRIS·문서 저장소·Slack·**국가법령정보 MCP** 상태·폴백

3. **무엇을 바꿀지 묻기.**

   > 무엇을 조정할까요? 섹션을 고르거나 자기 말로 변경 설명.

4. **변경 수집.** 사용자가 섹션을 고르면 그 섹션의 현재 값을 보여주고 새 값 묻기. 사용자가
   자유 형식으로 설명하면 어느 섹션이 영향인지 식별하고 mapping 확인.

5. **변경 전 검증.** 변경에 법적 인용·임계점·기한이 있으면 국가법령정보 대조 검증:
   ```
   mcp__korean-law__get_law_text "[조항]"
   ```
   인용이 안 맞으면 사용자에게 확인 ("당신은 X라 하셨는데, 국가법령정보의 현행 텍스트는 Y. 어느
   것으로 갈까요?"). 노동위 판정례·고용부 행정해석은 국가법령정보 미커버 — `[노동위 판정 — 직접 검증 필요]`.

6. **변경 적용.** CLAUDE.md의 해당 섹션을 갱신. 변경 전후 diff 보여주기.

   예시:
   - *해고 위험 플래그에 "복직 직후" 추가:* "이제 termination-review가 육아휴직·산재 복직
     직후 해고를 자동으로 high-risk로 플래그합니다."
   - *새 에스컬레이션 승인권자:* "당신 권한을 초과하는 해고·징계는 이제 이 승인권자로 라우팅."
   - *포괄임금제 입장 변경:* "wage-hour-qa가 포괄임금 적정성을 이 기준으로 점검합니다."
   - *상시 근로자 수 밴드 변경 (예: 9명 → 12명):* "이제 취업규칙 작성·신고 의무(§93)가 적용
     범위에 들어옵니다 — 적용범위 overlay를 갱신했습니다. `[상시 근로자 수 산정 — 검증 필요]`"

7. **확인 + 검증 로그 기록.**
   ```
   [YYYY-MM-DD] customize - [성명]이(가) [섹션]에서 [구체적 변경]
   ```
   `~/.claude/plugins/config/claude-for-legal-kr/employment-legal/verification-log.md`에 추가.

## 흔한 맞춤화 패턴

**상시 근로자 수 변경:** 인원이 임계점을 넘나들면(5인·10인·30인) 영향 받는 적용범위 다시 보여주고
사용자 확인. 산정은 `[검토]`로 플래그.

**에스컬레이션 담당자 변경:** 옛 담당자에서 새 담당자로 — 단순 이름 교체. 매트릭스 전체 보여주기.

**플레이북 입장 단 한 줄 변경:** "해고예고 관행을 30일 전 통지로" — 영향: 모든 미래 해고 검토.

**해고 위험 플래그 추가·제거:** termination-review의 결론 전 점검 목록 갱신.

**하우스 스타일 갱신:** "휴가 알림을 이메일에서 Slack 채널로" → 영향 받는 스킬·에이전트(leave-tracker 등) 식별.

**국가법령정보 MCP 추가·제거:** `.mcp.json` 직접 편집 안내 또는 자동화.

## 가드레일

- **섹션을 삭제하지 않는다.** 사용자가 "제거"를 원하면 `[미구성]`으로 설정하고 그것이 플러그인
  동작에 무엇을 의미하는지 설명.
- **내부 모순 플래그.** 변경이 프로파일을 모순되게 만들면(상시 근로자 수 5인 미만 + "해고 제한
  적용"; 또는 위험 성향 적극 + "모든 해고 GC 승인 필요"), 긴장을 플래그하고 어느 것을 원하는지 묻기.
- **가드레일 약화 플래그.** 사용자가 가드레일을 끄려 하면(`[검토]` 플래그 제거, 문서 헤더 생략,
  `[검증 필요]` 태그 제거), 그 가드레일이 무엇을 막는지 설명하고 trade-off를 이해하는지 확인.
  `[검토]` 플래그·출처 attribution 태그·인용 조문의 `[검증 필요]` 태그는 load-bearing이며 제거하지 않는다.
- **at-will을 도입하지 않는다.** 사용자가 "정당한 이유 없이 해고 가능하게" 같은 설정을 요청해도
  한국법상 §23 정당성 요건은 config로 끌 수 없다 — 설명하고 거부.
- **한 번에 하나.** 전체 인터뷰를 다시 묻지 않는다.
