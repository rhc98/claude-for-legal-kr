# 규제-정책 갭 분석

## 매터 컨텍스트

CLAUDE.md `## 매터 워크스페이스` 확인. 비활성이면 skip. 활성이면 묻기.

## 목적

PIPA가 개정된다. PIPC가 새 고시·가이드라인을 발표한다. 행정안전부가 공공기관용 새 지침을
낸다. 무엇이 움직이고 — 무엇을 바꿔야 하는지 알아야 한다.

이 스킬은 새 요건을 현재 실무(CLAUDE.md → 처리방침 약속 + PIA documented practice) 대조
diff하고 시정 계획 포함 갭 목록 산출.

## 현재 상태 로드

`~/.claude/plugins/config/claude-for-legal-kr/privacy-legal/CLAUDE.md` 읽기:
- `## 개인정보처리방침 약속` — 공개 약속한 것
- `## 우리는 누구인가` → `## 규제 풋프린트` — 이미 적용되는 것
- `## 정보주체 권리행사 처리 절차` → 시스템 목록 — 실제 운영적 처리

규제가 적용 안 되면(잘못된 관할·임계점 미달·다른 산업) 갭 분석은 한 줄: "적용 안 됨. 이유:
[이유]. 행동 불필요."

## 워크플로우

### Step 1: 규제 범위

diff 전 답:

- **적용?** 관할 (한국 법체계인가, 외국 거주자 대상 한국법인가), 임계점 (매출·이용자 수·
  데이터 볼륨), 산업 carve-out (금융·의료·교육 등)
- **언제?** 시행일, 집행일(종종 더 늦음), phase-in
- **실제로 새로운 것?** 많은 "새" 한국 규제는 90% 기존 PIPA + 미세조정. 이미 컴플라이언스한
  것에서의 delta 식별, 전체 텍스트 아님.

**법망에서 규제 본문 가져오기:**
```
mcp__beopmang__law search "AI 기본법" 또는 mcp__beopmang__law get [법령명]
```

PIPC 고시·KISA 안내서는 법망 미커버 가능 — 사용자에게 URL 또는 PDF 부탁.

### Step 2: 요건 추출

규제(또는 요약·가이드라인) 읽기. 모든 실질 요건을 별개 항목으로 목록:

| # | 요건 | 인용 | 카테고리 |
|---|---|---|---|
| 1 | [기재된 요건] | [조항·고시 번호] | [고지 / 권리 / 안전조치 / 수탁자 / 동의 / 거버넌스] |

**카테고리:**
- **고지** — 이용자에게 말해야 하는 것 (처리방침 내용)
- **권리** — 이용자가 요구할 수 있는 것 (권리행사 인접)
- **안전조치** — 기술적·관리적 조치
- **수탁자** — 수탁자에 flow down해야 하는 것
- **동의** — 옵트인·옵트아웃 메커니즘
- **거버넌스** — CPO, 영향평가, 기록 보유

### Step 3: 현재 상태 대조 diff

각 요건에 대해:

```markdown
### [요건 #N]: [짧은 이름]

**규제 내용:** [요건, 인용 또는 paraphrase]

**현재 우리:** [config CLAUDE.md / 처리방침 / 실무가 보여주는 것]

**갭:** [없음 | 부분 | 전체]

**부분·전체 갭이면 — 빠진 것:** [구체적]

**해소 노력:** [처리방침 갱신만 | 제품 변경 | 수탁자 재협상 | 신규 절차]

**미컴플라이언스 위험:** [규제 벌금 범위 (PIPA §74 — 최대 매출 3%), 집행 가능성, 평판]
```

### Step 4: 우선순위화

모든 갭이 같지 않음. 정렬:

1. **이가 있는 hard 기한** — 시행일 + 활성 집행 + 진짜 벌금 (PIPA는 매출 3% 또는 위반행위
   관련 매출 3% 과징금)
2. **연쇄 효과** — 한 갭이 다른 컴플라이언스를 깸 (처리방침 갱신이 동의 메커니즘 갱신 트리거 등)
3. **검출 가능성** — PIPC가 실제로 잡을 가능성. 처리방침 검토는 자주 일어남. 안전조치 미준수는
   사고 후 사후 평가에서 surface.
4. **시정 비용** — 헐값 fix는 우선 (vs 분기 단위 엔지니어링).

### Step 5: 시정 계획

```markdown
[WORK-PRODUCT 헤더]

# 규제 갭 분석: [규제명]

**분석일:** [날짜] | **시행일:** [날짜] | **집행일:** [날짜]

## 핵심 결론

[2 문장. 적용? 어디 갭? 비차단 또는 차단 작업?]

**갭 요약:** [N] 항목 — [N] 차단 / [N] 우선 / [N] 권장 / [N] 갭 없음

## 적용성

[적용 또는 미적용 + 이유. 미적용이면 여기서 끝.]

## 갭 표

| # | 요건 | 갭 | 우선순위 | 노력 | 담당 | 기한 |
|---|---|---|---|---|---|---|
| 1 | [요건] | [None/부분/전체] | 🔴/🟠/🟡/🟢 | [노력] | [담당] | [날짜] |

## 시정 계획 (우선순위순)

### [요건 #1] — [짧은 이름]

**갭:** [구체적]
**시정:** [구체적 행동 — 처리방침 §X 갱신 / 동의 화면 wireframe 변경 / 위·수탁계약 재협상]
**담당:** [성명·역할]
**기한:** [날짜 — 시행일 전 buffer]
**검증:** [완료 어떻게 확인]

[반복]

## 차단 갭 (있으면)

[시행일까지 닫을 수 없는 갭. 위험 evaluate, 단계적 컴플라이언스 계획 또는 활동 중단 결정.]

## 검토자 메모

⚠️ 사용 가능한 신뢰 가능 인용을 위해 법망 검증. 모든 인용은 PIPA·시행령·고시 1차 소스 대조.
```

## 외국 규제 (GDPR, CCPA 등)

한국 회사에 외국 정보주체가 있어 GDPR·CCPA 등이 적용되면:
- 법망 사용 불가 (외국법)
- 웹 검색 또는 외국 변호사 자문 필요
- 모든 인용에 `[웹 검색 — 검증 필요]` 또는 `[외국법 — 외국 변호사 검증 필요]` 태그
- 외국법 갭은 별도 표·별도 우선순위 (외국 집행 강도에 따름)

## 흔히 모니터링할 한국 규제 출처

- **법망 MCP** — 본법·시행령·시행규칙 변경
- **PIPC 보도자료** (`https://www.pipc.go.kr`) — 신규 고시·가이드라인·결정례
- **KISA 안내자료** (`https://www.kisa.or.kr`) — 기술적 안전조치 기준
- **법제처 입법예고** — 향후 변경 미리 보기
- **국회 의안정보시스템** (`https://likms.assembly.go.kr/bill`) — 법망에서 의안 검색 가능
