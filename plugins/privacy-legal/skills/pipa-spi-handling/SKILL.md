# 민감정보·고유식별정보·주민번호 처리 적법성

## 매터 컨텍스트

실무 수준 CLAUDE.md `## 매터 워크스페이스` 확인. 비활성이면 skip. 활성이면 묻기.

## 목적

한국 PIPA는 일반 개인정보 외에 추가 보호가 필요한 카테고리를 별도로 규정. 이 항목들의
처리는 별도 동의 또는 법령 특별 규정만 가능하고, 추가 안전조치 의무가 따른다. 이 스킬은
처리하려는 데이터 항목이 이 카테고리에 해당하는지, 해당하면 적법 처리 요건이 무엇인지 점검.

**왜 별도 스킬인가:** 미국·EU는 sensitive data 정의가 다르고(GDPR Art. 9 special categories는
범위 다름) US는 federal level에서 통일된 sensitive data 정의가 없음. 한국 PIPA의 §23·§24·§24-2는
한국-specific 카테고리와 요건 — 일반 PIA 워크플로우와 별도 처리할 가치 있음.

## 한국 PIPA 카테고리 정의

### 민감정보 (PIPA §23, 시행령 §18)

법망에서 현행 텍스트 가져오기:
```
mcp__beopmang__law get "개인정보보호법 제23조"
mcp__beopmang__law get "개인정보보호법 시행령 제18조"
```

**§23 본법 카테고리:**
- 사상·신념
- 노동조합·정당의 가입·탈퇴
- 정치적 견해
- 건강
- 성생활
- 그 밖에 정보주체의 사생활을 현저히 침해할 우려가 있는 개인정보로서 시행령으로 정함

**시행령 §18 추가 카테고리:**
- **유전정보** (유전자검사 결과)
- **범죄경력** (범죄에 관한 형의 선고, 면제, 선고유예, 집행유예, 사면, 복권 등)
- **개인의 신체적·생리적·행동적 특징에 관한 정보로서 특정 개인을 알아볼 목적으로 일정한 기술적
  수단을 통해 생성한 정보** (= 생체인식정보, **2024.3.15. 시행** PIPA 개정으로 명문화)
- **인종·민족에 관한 정보**

### 고유식별정보 (PIPA §24, 시행령 §19)

```
mcp__beopmang__law get "개인정보보호법 제24조"
mcp__beopmang__law get "개인정보보호법 시행령 제19조"
```

**§24 + 시행령 §19 카테고리:**
- **주민등록번호** (별도 §24-2 강한 규정 — 아래)
- **여권번호**
- **운전면허번호**
- **외국인등록번호**

### 주민등록번호 (PIPA §24-2)

```
mcp__beopmang__law get "개인정보보호법 제24조의2"
```

**가장 강한 규정. 별도 법조가 있는 이유:** 한국 사회에서 주민번호의 광범위한 사용 + 유출
시 회복 불가능한 피해 + 정부 정책상 사용 제한 강화 흐름.

## 워크플로우

### Step 1: 데이터 항목을 카테고리 매핑

사용자가 처리하려는 데이터 항목을 듣고 분류:

| 데이터 항목 | 카테고리 | 적용 조항 |
|---|---|---|
| 혈액형·DNA·유전자검사 결과 | 민감정보 | §23 + 시행령 §18 |
| 건강검진 결과·진료기록·복약내역 | 민감정보 (건강) | §23 |
| 종교 | 민감정보 (사상·신념) | §23 |
| 정당 가입 여부 | 민감정보 | §23 |
| 노조 가입 여부 | 민감정보 | §23 |
| 성적 지향 | 민감정보 (성생활) | §23 |
| 지문·홍채·정맥·얼굴 (특정 개인 식별 목적, 기술적 수단) | 민감정보 (생체인식정보) | §23 + 시행령 §18, 2024.3.15. 시행 |
| 인종·민족 | 민감정보 | 시행령 §18 |
| 범죄경력 | 민감정보 | 시행령 §18 |
| 주민등록번호 | 고유식별정보 (최강) | §24-2 |
| 여권번호 | 고유식별정보 | §24, 시행령 §19 |
| 운전면허번호 | 고유식별정보 | §24, 시행령 §19 |
| 외국인등록번호 | 고유식별정보 | §24, 시행령 §19 |
| 신용카드번호·계좌번호 | 일반 개인정보 + 신용정보법 | §15·§17, 신용정보법 §32 |
| 휴대폰 번호 | 일반 개인정보 | §15·§17 |

**판단 모호한 경우:**
- "얼굴 사진"은 단순 식별·기록 목적이면 일반 개인정보. 특정 개인을 알아볼 목적으로 기술적
  수단(딥러닝 얼굴인식 등)으로 처리하면 시행령 §18 생체인식정보로 민감정보.
- 사용자 입력한 "내 건강 상태" 자가 진술은 PIPA §23 건강 카테고리에 포함 가능 — 모호 시
  보수적으로 민감정보 처리.
- IP 주소·쿠키 ID·디바이스 ID: 일반 개인정보(단독으로 식별 불가하나 결합 시 식별 가능 →
  PIPA §2 정의의 "개인정보" 해당).

### Step 2: 적법 처리 근거 점검

#### 민감정보 (§23)

**§23① 두 가지 적법 처리 근거:**
1. 정보주체로부터 **다른 개인정보 처리에 대한 동의와 별개로 동의**를 받은 경우
2. **법령에서 민감정보 처리를 요구하거나 허용**하는 경우

별도 동의 요건 구체:
- 다른 동의(서비스 약관, 마케팅 동의 등)와 **명백히 구분**
- 처리 목적·항목·보유기간·거부 시 불이익 명시
- 거부할 수 있는 권리 안내

서비스 약관에 묻혀 받은 "동의"는 별도 동의로 인정 안 됨 (PIPC 시정조치 사례 다수).

#### 고유식별정보 (§24)

**§24① 적법 처리 근거:**
1. 정보주체의 별도 동의
2. **법령에서 구체적으로 고유식별정보 처리를 요구하거나 허용**

법령상 근거 예: 부동산 등기, 금융실명거래, 통신비밀보호법상 통신자료 제공 등.

#### 주민등록번호 (§24-2)

**가장 엄격. §24-2①의 매우 제한적 근거:**
1. 법률·대통령령·국회규칙·대법원규칙·헌법재판소규칙·중앙선거관리위원회규칙·감사원규칙에서
   구체적으로 주민등록번호 처리를 요구·허용한 경우
2. 정보주체·제3자의 급박한 생명·신체·재산 이익을 위하여 명백히 필요한 경우
3. 그 외 주민등록번호 처리가 불가피한 경우로서 **개인정보보호위원회가 고시로 정하는 경우**

**중요: §24-2②** — 법령상 근거가 있어도 처리 시 인터넷 등 정보통신서비스 제공 시 **대체수단
제공 의무** (아이핀, 휴대폰 본인확인, 공동인증서 등).

**§24-2③** — 위 근거 없이 주민번호 처리 시 과징금 (최대 5억 원, PIPA §74-2).

### Step 3: 안전조치 요건

#### 민감정보·고유식별정보 공통 (시행령 §30 + PIPC 안전성 확보조치 기준 고시)

```
mcp__beopmang__law get "개인정보보호법 시행령 제30조"
```

**기본 안전조치 (모든 개인정보):**
- 접근통제 (IP·계정·기간)
- 접근권한 관리·기록
- 암호화 (전송 + 저장)
- 접속기록 보관 (1년 이상)
- 보안프로그램 (백신 등)
- 물리적 안전조치

**민감정보·고유식별정보 추가:**
- 저장 시 암호화 의무 (시행령 §30③) — 일반 개인정보는 암호화가 권장이지만 이건 의무
- 외부 송신 시 암호화 의무
- 접근권한 부여·변경·말소 기록 3년 이상 보관
- 비밀번호 1년에 1회 이상 변경 (관리자 등)

#### 주민등록번호 추가

- **2014년 이후 신규 수집 금지** (당시 개정) — 법령상 근거 없으면 수집 자체 불가
- 기존 수집분도 법령상 근거 사라지면 파기 의무
- **인터넷 본인확인 시 대체수단 제공** (§24-2②) — 아이핀·휴대폰 본인확인·공동인증서 등 사용
- 별도 분리 저장·암호화

### Step 4: 보유 기간 제한

민감정보·고유식별정보는 일반 개인정보 대비 더 엄격한 보유 기간 관리:
- 처리 목적 달성 즉시 파기 (PIPA §21)
- 다른 법령에 따른 보존 의무 있어도 분리 보관 + 접근 제한
- 보유 기간 종료 후 파기 시 복구 불가능한 방법

### Step 5: 처리방침·개인정보처리방침 게재 의무

민감정보·고유식별정보 처리 시 처리방침에 **별도 항목 게재 의무** (PIPA §30):
- 민감정보의 처리에 관한 사항 (수집 항목, 처리 목적, 보유 기간)
- 고유식별정보 처리에 관한 사항 (법령상 근거 명시)

## 흔한 함정

**"가명처리하면 민감정보 아님" — 틀림.**
가명정보(PIPA §28-2 이하)도 추가정보 결합 시 식별 가능. 가명처리 자체로 §23 적용 면제 안 됨.
가명정보의 결합·제3자 제공은 별도 규정 (§28-2~§28-7).

**"동의 안 받고 법령 근거로 처리" — 법령이 명시적이어야.**
"개인정보 보호와 무관한 다른 법령이 일반적으로 이용을 허용"하는 것은 §23의 "법령에서 처리를
요구·허용"에 해당 안 함. PIPC 결정례 다수.

**"외국 사용자라서 GDPR만 적용" — 한국 회사면 PIPA 적용.**
한국 회사가 외국 사용자 민감정보 처리 시 PIPA + GDPR 등 양쪽 적용. PIPA §23이 GDPR Art. 9
보다 엄격한 부분 (예: 인종·민족이 PIPA 시행령 §18에 포함) — 양쪽 만족 필요.

**"신용정보는 일반 개인정보로 처리" — 신용정보법 별도 적용.**
금융 신용정보는 신용정보법 §32, §33 등 별도 규정. 본 스킬은 PIPA 카테고리만 다룸 — 신용정보는
금융 분야 전용 처리 필요.

**"AI 학습에 민감정보 사용 가능?" — 매우 제한적.**
AI 학습 목적 처리는 PIPA §15(수집·이용) 또는 §28-2(가명정보 처리 특례) 평가 필요. 별도 동의 또는
법령 근거 + 안전조치 + PIA. AI 기본법 2026 시행 시 추가 규제 가능.
