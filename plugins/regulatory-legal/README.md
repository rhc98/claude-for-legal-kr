# 규제 모니터링·입법예고 대응 플러그인 (regulatory-legal)

한국 규제 모니터링·입법예고/행정예고 대응·정책 컴플라이언스 실무를 위한 Claude Code 플러그인.
규제기관 피드 감시, 공포 법령·시행령·시행규칙 개정 추적(시행일 관리), 입법예고·행정예고 의견제출
마감 관리, 국회 계류 법률안 워치, 정책 라이브러리 대조 diff, 재료성(materiality) 분류, 갭 트리아지,
주간 규제 다이제스트 작성을 다룬다.

원본 [`anthropics/claude-for-legal`](https://github.com/anthropics/claude-for-legal)의
상류 `regulatory` 플러그인(Apache 2.0)을 한국 규제·입법 체계 — **행정절차법·법령공포법·국회법·
행정규제기본법**과 개별 소관 법령(개인정보보호법·자본시장법·공정거래법·근로기준법 등) — 로 포팅한
결과물이다. 국가법령정보 MCP(`korean-law`)로 공포 법령·판례·헌재결정·행정규칙 위에서 동작한다.

**모든 산출물은 변호사 검토 전제의 초안이다** — 인용된 조문은 국가법령정보 MCP로 검증해 표시하고,
판단해야 할 지점은 `[검토]` 플래그를 단다. 결정적 행위(의견제출서 제출, 정책 변경 확정, 규제기관
대응 문서 발송, 에스컬레이션)는 명시적 확인 게이트를 통과해야 한다.

> **한국에는 미국식 Federal Register·NPRM/ANPR·attorney work-product 제도가 없다.** 상류는
> **미국 연방 관보(Federal Register), 규칙제정예고(NPRM)·사전규칙제정예고(ANPR)**를 전제로
> 설계됐다. 한국 포팅의 핵심 재맥락화는 **대륙법계 입법·행정예고 절차로의 재설계**다 — 미국의
> "어느 연방기관 Federal Register 피드냐"라는 축이 한국에서는 **소관 부처·규제 위원회 × 법령
> 단계(공포 법령 / 입법예고·행정예고 / 국회 계류 법률안)**의 조합으로 바뀐다. 특히 **국회에
> 계류 중인 법률안은 아직 법이 아니다** — 미국의 ANPR과 달리 컴플라이언스 의무를 만들지 않으며
> 이 플러그인은 계류 법안을 절대 material로 분류하지 않는다(아래 데이터 갭 참조).

## 무엇을 하나

- **규제 피드 감시** — 공포 법령(국가법령정보)·입법예고·행정예고(국민참여입법센터)·국회 계류
  법률안(의안정보시스템)·규제기관 의결·보도자료를 item-type별로 분기 분류하고 재료성 임계값으로
  필터링. `reg-change-monitor` 에이전트가 주기적으로 다이제스트를 게시한다.
- **입법예고·행정예고 대응** — 열린 의견제출 기간을 훑고 마감을 추적하며, 제출·미제출·보류
  결정을 기록한다(행정절차법 §44).
- **정책 대조 diff** — 특정 규제 변경을 색인된 사내 정책 라이브러리와 대조해 걸리는 정책과 갭을
  찾아내고, 갭 트래커로 라우팅한다.
- **정책 레드라인 초안** — 발견된 갭을 닫는 정책 마크업 수정안을 제안(내부 검토용 1차 초안).
- **국회 계류 법률안 워치(KR 신규)** — 의안정보시스템 직접 검색으로 심사단계(접수→위원회 심사→
  체계자구→본회의→정부이송→공포)를 추적. 공포 확인 전에는 절대 material로 승격하지 않는다.
- **갭 트리아지·추적** — 규제 변경이 정책 라이브러리와 어긋나는 항목(갭)을 상태별로 추적·종결·
  위험수용 처리.

**재료성(materiality) 분류가 이 플러그인의 핵심 변수다.** 공포된 법률·시행령·시행규칙 개정은
항상 material(즉시 대응·시행일 필수 기록), 입법예고·행정예고는 review-worthy(의견제출 마감
필수 기록), **국회 계류 법률안은 절대 material이 아니고 watch에 고정**된다. 계류 법안을
material로 오분류하는 것은 이 플러그인의 가장 위험한 실패 모드다 — 존재하지 않는 컴플라이언스
의무를 만들어낸다.

## 누구를 위한 것인가

| 역할 | 주 사용 영역 |
|---|---|
| **변호사·법률 전문가** | 규제 변경 분석·정책 diff·갭 시정·의견제출서 초안, 에스컬레이션 판단 |
| **사내 법무·컴플라이언스(변호사)** | 규제기관 watchlist 운영, 정책 라이브러리 관리, 다이제스트 검토 |
| **비변호사 컴플라이언스 담당(변호사 접근 가능)** | 피드 트리아지·갭 발견(변호사 검토 전제), 의견제출 여부 에스컬레이션 |
| **비변호사(변호사 접근 없음)** | 규제 동향 인지·정책 갭 식별(리서치 노트 — 의존 전 변호사 검토 필수) |

> **규제 대응은 오분류·기한도과의 downside가 크다.** 비변호사 역할의 산출물은
> `리서치 노트 — 법률자문이 아님. 의존하기 전 변호사 검토 필수` 헤더로 처리되며, 의견제출서
> 대외 제출·정책 변경 확정·규제기관 대응 문서 발송 같은 결정적 행위는 변호사(또는 본인 책임)의
> 몫으로 게이트를 건다.

## 설치

`.mcp.json`에 포함:

- **국가법령정보(korean-law)** — 한국 법령·판례·헌재결정·행정규칙·위원회 결정례 조회·검증.
  **1순위.** 없으면 모든 조문 인용이 `[모델 지식 — 검증 필요]`로 떨어진다. 환각방지 인용검증
  (`legal_analysis` verify_citations)과 행정규칙·위원회 결정례 부분 통합검색(`search_decisions`)
  포함. **무료 OC 인증키 필요** — [open.law.go.kr](https://open.law.go.kr)에서 발급, 환경변수
  `LAW_OC`로 설정.
- **Slack** — 규제 변경 알림, 입법예고·의견제출 마감 공지, 재료성 판단·에스컬레이션 논의, 주간
  다이제스트 배포.
- **Google Drive** — 정책 라이브러리(사내 정책·컴플라이언스 매뉴얼·표준 서식), 규제 공고문·
  의견제출서 선례 로드.

```
export LAW_OC=발급받은_OC_인증키
claude plugin install regulatory-legal@claude-for-legal-kr
/regulatory-legal:cold-start-interview
```

콜드스타트 인터뷰가 규제기관 watchlist(소관 부처·위원회), 재료성 임계값, 정책 라이브러리 인덱스,
소스·피드 구성, 의견제출 결정 owner, 역할·에스컬레이션 매트릭스를 학습한다. 결과는
`~/.claude/plugins/config/claude-for-legal-kr/regulatory-legal/CLAUDE.md`에 저장되며 플러그인
업데이트와 무관하게 유지된다. 회사 공통 정보는 상위 `company-profile.md`에 모여 모든 플러그인이
공유한다.

⚠️ **국회 의안(법률안)·입법예고·행정예고 피드는 국가법령정보 MCP 범위 밖이다** — 국회 의안정보
시스템·국민참여입법센터를 직접 검색한다(아래 데이터 갭 참조).

## 스킬 목록 (10개) + 에이전트

| 스킬 | 하는 일 |
|---|---|
| `cold-start-interview` | 콜드스타트 인터뷰 — 규제기관 watchlist·재료성 임계값·정책 라이브러리·소스·피드 구성 학습 |

| `matter-workspace` | 매터 워크스페이스 관리(법무법인·단독·소규모 사무소 — 규제 매터 격리) |
| `reg-feed-watcher` | 규제 소스 점검 — KR 소스 매트릭스(공포 법령/입법예고·행정예고/국회 계류 법률안/규제기관 의결)를 재료성 임계값으로 필터해 보고 |
| `comments` | 입법예고·행정예고 의견제출 기간 추적 — 제출·미제출·보류 결정 기록, 마감 추적 |
| `policy-diff` | 특정 규제 변경을 색인된 정책 라이브러리와 대조 diff, 갭 산출 |
| `gap-surfacer` | (내부 전용) `gaps`·`comments`를 뒷받침하는 공유 gap-tracker·comment-tracker 프레임워크 |
| `gaps` | 열린 갭 트래커 — 종결·위험수용 처리 |
| `policy-redraft` | 갭을 닫는 정책 마크업 레드라인 초안(내부 검토용 1차 초안) |
| `bill-watch` **(KR 신규)** | 국회 계류 법률안을 의안정보시스템에서 직접 검색·추적 — 심사단계 추적, 관심 법안 워치리스트, 공포 시 material 경로로 승격 |

| 에이전트 | 하는 일 |
|---|---|
| `reg-change-monitor` | 스케줄드 에이전트(디폴트 주 1회) — 규제 소스를 점검해 재료성 임계값으로 필터한 규제 다이제스트를 게시. 공포 법령에는 policy-diff를 돌려 갭 요약 포함. 계류 의안은 절대 material이 아니므로 watch로 고정하고 `bill-watch`에 위임 |

## End-to-End 데모

**시나리오 — 개인정보보호법 시행령 개정 입법예고 대응**

```
1. 개인정보보호법 시행령 개정 입법예고가 떴다
   → /regulatory-legal:reg-feed-watcher
     KR 소스 매트릭스 점검 · 입법예고로 분류(review-worthy) · 의견제출 마감 추출
     (행정절차법 §43, 40일 원칙 — 공고문 실제 마감일 확인) [검증 필요]

2. 우리 내부 정책 어디가 영향받고, 의견제출 마감이 언제인가
   → /regulatory-legal:policy-diff  (개정 시행령 vs 정책 라이브러리)
     걸리는 정책 식별 · 갭 산출 · gap-tracker 등록

3. 의견을 낼지 결정, 마감 추적
   → /regulatory-legal:comments
     의견제출 마감 D-day 추적 · 제출/미제출/보류 결정 기록(--decide CMT-ID)

4. 갭을 닫을 정책 레드라인 초안
   → /regulatory-legal:policy-redraft  (GAP-ID)
     내부 검토용 1차 마크업 — 승인된 정책 문서에 직접 적용 아님

5. 시행일까지 갭 추적 → reg-change-monitor가 주간 다이제스트로 상태 보고
   → /regulatory-legal:gaps
```

**시나리오 — 국회 계류 법률안 워치**

```
1. 관심 법률안이 국회에 발의됐다는 소식
   → /regulatory-legal:bill-watch
     의안정보시스템 직접 검색 · 심사단계 확인(접수→위원회→체계자구→본회의)
     [국회 의안시스템 — 직접 검증 필요] · status_verified: false · watch 고정

2. 정책 변경은 하지 않는다 — 계류 법안은 아직 법이 아니다
   (컴플라이언스 의무 없음, material 승격 금지)

3. 공포(관보 게재) 확인 후에만 material 경로로 승격
   → /regulatory-legal:reg-feed-watcher 가 공포 법령으로 재분류
     → /regulatory-legal:policy-diff 로 정식 갭 분석
```

## 데이터 갭

국가법령정보로 **가져올 수 없거나 부분만 가져오는** 정보가 있다. 지어내지 않고, 사용자에게 1차
소스를 요청하며 태그로 플래그한다(`docs/DATA_GAPS.md` §1-B와 일관):

- **국회 의안(법률안)은 커넥터 범위 밖이다.** 국가법령정보 MCP에는 국회 의안 검색 도구(`/bill`
  등가)가 없다. 국회 의안정보시스템(`https://likms.assembly.go.kr/bill`)을 직접 검색하고
  `[국회 의안시스템 — 직접 검증 필요]` 태그. **계류 법안은 절대 material로 분류하지 않고 watch에
  고정**한다(`bill-watch`). 공포 확인 후에만 material 경로로 승격.
- **입법예고·행정예고는 커넥터 범위 밖이다.** 국민참여입법센터(`https://opinion.lawmaking.go.kr`)·
  각 부처 공고를 직접 검색(WebFetch)하고 `[법제처]` 태그. 의견제출 마감은 공고문의 실제 날짜로
  기록한다.
- **행정규칙(고시·훈령·예규)은 부분 커버다.** 국가법령정보가 전체 일부만 색인, 새 고시는 반영이
  지연될 수 있다. 미검색·불완전 시 법제처 행정규칙정보시스템·소관 부처 사이트를 직접 확인하고
  `[검증 필요]`.
- **규제기관 의결례는 부분 커버다.** PIPC·공정위·노동위 등은 `search_decisions`로 부분 커버되나,
  최신 의결·전문은 각 기관 사이트가 우위다. `[규제 당국 결정 — 직접 검증 필요]`.
- ✓ 잘 커버: 공포된 법률·시행령·시행규칙, 대법원·헌재 판례, 행정절차법·법령공포법 등 절차법 조문.

## 면책

⚠️ **한국법에는 미국식 Federal Register·NPRM/ANPR 제도가 없다.** 상류 미국 규제 모니터링
플러그인을 **대륙법계 입법·행정예고 절차로 재설계**했다 — 규칙제정예고는 입법예고·행정예고
(행정절차법 §41~§47)로, 사전규칙제정예고는 국회 계류 법률안 워치(공포 전 절대 material 아님)로
재설계했다.

⚠️ **데이터 갭 — 국회 의안·입법예고·행정예고·행정규칙 일부·규제기관 의결례 일부는 직접 검증이
