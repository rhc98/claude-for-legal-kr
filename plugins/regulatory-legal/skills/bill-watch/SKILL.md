# /bill-watch

1. `~/.claude/plugins/config/claude-for-legal-kr/regulatory-legal/CLAUDE.md` 로드 → watch 기관·분야,
   재료성 임계값(`## 중요도(재료성) 임계값`), 소스·피드 구성.
2. 아래 워크플로우 실행 — 의안정보시스템 직접 검색.
3. 심사단계를 확인하고 관심 법안 등록부(watch)를 갱신.
4. 산출: 계류 법안의 현재 심사단계 + 다음 재검토 시점. **어떤 것도 material로 분류하지 않는다.**

## 매터 컨텍스트

`~/.claude/plugins/config/claude-for-legal-kr/regulatory-legal/CLAUDE.md`의 `## 매터 워크스페이스`
확인. `활성화`가 `✗`(사내 사용자 디폴트)면 이 문단 skip — 스킬은 실무 수준 컨텍스트를 자동으로
쓴다. 활성인데 활성 매터가 없으면 묻는다: "어느 매터입니까? `/regulatory-legal:matter-workspace
switch <slug>` 실행하거나 `practice-level`이라고 말씀해주세요." 활성 매터가 있으면 그 매터의
`matter.md`를 로드하고, 트래커·출력을 매터 폴더
`~/.claude/plugins/config/claude-for-legal-kr/regulatory-legal/matters/<매터-slug>/`에 둔다.
`매터 간 컨텍스트`가 `on`이 아니면 다른 매터 파일을 절대 읽지 않는다.

## 목적

국회에 계류 중인 법률안은 **아직 법이 아니다.** 컴플라이언스 의무를 만들지 않고, 통과 여부·통과
시점·최종 문언이 모두 불확실하다. 그래도 watch할 가치가 있다 — 우리 사업에 걸리는 개정이 국회에
올라와 있으면, 통과 전에 방향을 읽고(사전 포지셔닝), 국회입법예고 의견제출 기회를 놓치지 않고,
공포되는 순간 컴플라이언스 시계를 즉시 켜야 하기 때문이다.

이 스킬의 핵심 규율은 하나다: **계류 법안을 추적하되 절대 material로 승격하지 않는다.** 심사단계만
따라가고, 공포(관보 게재)가 확인되는 그 순간에만 reg-feed-watcher의 "항상 material"(공포 법령)
경로로 넘긴다. 그 전까지 모든 항목은 `watch` 고정이다(아래 재료성 규율).

## 데이터 갭 — 국회 의안은 커넥터 범위 밖

**국가법령정보 MCP(`korean-law`)에는 국회 의안 검색 도구(`/bill` 등가)가 없다.** 계류 법률안·심사
경과·의안번호 조회는 이 커넥터의 범위 밖이다(CLAUDE.md `## 데이터 갭`, `docs/DATA_GAPS.md` §1-B).

- **소스: 국회 의안정보시스템** — `https://likms.assembly.go.kr/bill` `(verify)`. 의안번호·법안명·
  발의자·소관 위원회·심사경과를 여기서 직접 검색(WebFetch)한다.
- **출처 태그: `[국회 의안시스템 — 직접 검증 필요]`** — 이 스킬이 산출하는 모든 의안 인용·심사단계·
  의안번호에 **예외 없이** 붙인다. 공개 API가 없어 직접 검색·직접 검증이 유일한 경로다.
- **지어내지 않는다(침묵의 보완 금지 — CLAUDE.md `## 공유 가드레일`).** 의안정보시스템에 접근하지
  못하거나 검색이 빈 결과를 내면, 웹 검색·모델 지식으로 의안번호·심사단계를 조용히 메우지 않는다.
  사용자에게 의안번호·의안정보시스템 URL·법안 원문 paste를 부탁하고, 줄 때까지 진행하지 않는다.
  웹 검색으로 방향을 잡되 의존 전 의안정보시스템 대조가 필요하면 `[웹 검색 — 검증 필요]`를 함께 단다.

## 심사단계 추적

법률안은 아래 단계를 거친다. 각 단계는 통과 확률·최종 문언 확정도가 다르므로, 심사단계 전이는
watch 항목의 **재분류 트리거**다.

```
접수 → 위원회 심사(소관 상임위) → 체계·자구 심사(법사위) → 본회의 의결 → 정부이송 → 공포
```

- **접수 / 위원회 심사:** 계류 초기. 문언이 크게 바뀔 수 있고 폐기·철회·대안반영폐기 가능성도 높다.
  방향성 신호로만 본다. `status_verified: false`.
- **체계·자구 심사(법사위):** 본회의 직전. 통과 가능성이 실질적으로 올라간 신호 — 재검토 주기를
  좁힌다. 그래도 **아직 법이 아니다.** `status_verified: false`.
- **본회의 의결(가결):** 국회를 통과했으나 **아직 공포 전.** 정부이송·공포 절차가 남았다. 이 단계에서
  재검토 주기를 최단으로 당기고, 공포를 능동적으로 감시한다. 그래도 `status_verified: false` 유지 —
  공포·시행 확인 전에는 컴플라이언스 시계를 켜지 않는다.
- **공포:** 관보 게재로 확정. **이 순간에만** material 경로로 승격한다(아래 "공포 시 승격").

> **단계 전이 시 재분류.** watch 항목의 심사단계가 바뀌면 등록부의 `stage`와 `due`(다음 재검토일)를
> 갱신한다. 단계가 올라갔다고 재료성이 올라가는 것은 아니다 — **본회의 가결도 여전히 material이
> 아니다.** 바뀌는 것은 재검토 주기(감시 빈도)이지 재료성 tier가 아니다.

## 관심 법안 등록부 (gap-tracker watch)

관심 법안은 gap-surfacer가 소유하는
`~/.claude/plugins/config/claude-for-legal-kr/regulatory-legal/gap-tracker.yaml`의 `watch` 항목으로
관리한다 — 별도 트래커 파일을 만들지 않는다(크로스 스킬 계약: gap-surfacer 스키마 정합).

**`--register [법안명]`으로 등록:**

```yaml
gaps:
  - id: GAP-0NN
    requirement: "[이 법안이 통과되면 생길 요건 — 예비적 서술, 미확정]"
    regulation: "[법안명] (의안번호 [번호]) [국회 의안시스템 — 직접 검증 필요]"
    policy_affected: "[영향 예상 정책 또는 '확정 후 평가']"
    gap_type: "watch"          # 항상 watch — 계류 의안은 컴플라이언스 갭이 아니다
    owner: "[정책 인덱스의 성명 또는 미배정]"
    opened: [오늘]
    due: [다음 재검토일 — 예상 표결일·다음 회기, 컴플라이언스 기한 아님]
    status_verified: false      # 공포 확인 전까지 항상 false — Overdue 진입 금지
    status: "open"
    stage: "위원회 심사"        # 접수 | 위원회 심사 | 체계자구 | 본회의 의결 | 정부이송 | 공포
    resolution: ""
```

> **`status_verified: false` 고정.** gap-surfacer의 KR 규칙과 정합한다 — 국회 계류 법률안 유래
> 항목은 트래커 진입 시 **항상** `status_verified: false`로 생성하고, 국가법령정보 `get_law_text`로
> 공포·시행을 확인하기 전에는 절대 🔴 Overdue에 진입시키지 않는다. `due:`는 컴플라이언스 기한이
> 아니라 재검토일(예상 표결일·시행 예정일)이다. `gap_type`은 항상 `watch` — comment-decision·partial·
> full·new-policy로 승격하지 않는다.

**`--advance [의안번호]`로 심사단계 갱신:** 의안정보시스템에서 현재 심사경과를 확인하고 등록부
항목의 `stage`·`due`를 갱신한다. 저장된 단계를 그대로 믿지 않고 매 실행 시 의안정보시스템에서
재확인한다 — 심사경과는 이 세션 밖에서 바뀐다.

## 공포 시 승격 (핸드오프 → reg-feed-watcher)

본회의 통과·공포가 확인되면, watch 항목을 material 경로로 승격한다:

1. **국가법령정보로 공포·시행 확인.** 공포되면 해당 법령이 국가법령정보에 올라온다:
   ```
   mcp__korean-law__search_law {"query": "[공포된 법령명]"}
   mcp__korean-law__get_law_text {"query": "[법령명 제N조]"}
   ```
   현행/시행예정 상태와 **부칙 시행일·경과조치**를 대조한다. **공포일 ≠ 시행일**이다.
2. **확인되면** 등록부 항목을 `status_verified: true`로 전환하고, `regulation` 필드의 인용을
   `[국회 의안시스템 — 직접 검증 필요]`에서 국가법령정보 `[국가법령정보]` 태그로 갱신한다. 이 승격만이
   항목을 Overdue 판정 대상으로 만든다(gap-surfacer KR 규칙과 정합).
3. **reg-feed-watcher의 "항상 material"(공포 법령) 경로로 넘긴다.** 시행일을 기록하고, 영향 예상
   정책에 대해 `/regulatory-legal:policy-diff` 실행을 제안한다.
4. **국가법령정보에서 아직 확인되지 않으면**(공포 직후 반영 지연 가능) — 승격하지 않고 관보
   (gwanbo.go.kr) 게재를 직접 확인하거나 사용자에게 관보 게재 확인을 부탁한다. 반영 전까지
   `status_verified: false` 유지. **의안정보시스템의 "공포" 표시만으로 컴플라이언스 시계를 켜지 않는다.**

## 국회입법예고 의견제출 마감 (핸드오프 → comments)

법안이 국회 소관 위원회의 **국회입법예고**(국회법 §82-2 `[검증 필요]`) 대상이고 의견제출 마감이
있으면, 그 마감은 `/regulatory-legal:comments`가 추적한다 — 이 스킬이 아니다.

- comment-tracker.yaml에 `item_type: 의안`으로 append한다(필드 키는 comments 스킬이 소유하는
  comment-tracker.yaml 스키마를 따른다). `due:`(마감일)은 **국회입법예고 기간의 마감**
  (의안정보시스템·국회입법예고에서 확인)이지 법안 통과·공포일이 아니다. `[국회 의안시스템 — 직접
  검증 필요]` 태그를 붙인다.
- 국회법 §82-2의 위원회 입법예고 기간(10일 이상 `[검증 필요]`)은 참고치일 뿐 — **공고의 실제 마감이
  우선**하고 사안별로 단축될 수 있다. `verify_citations`로 조문을 확정하되 실패분은 태그를 유지한
  채 출고한다.
  ```
  mcp__korean-law__legal_analysis {"mode": "verify_citations", "citation": "국회법 제82조의2"}
  ```
- 이 스킬은 법안의 **심사단계**를 추적하고, comments는 그 법안의 **국회입법예고 의견제출 마감**을
  추적한다 — 두 축을 혼동하지 않는다.

## 재료성 규율 — 계류 의안은 컴플라이언스 의무가 없다

**이 플러그인에서 가장 위험한 오분류는 계류 법률안을 material로 승격하는 것이다** — 존재하지 않는
컴플라이언스 의무를 만들어낸다(CLAUDE.md `## 중요도(재료성) 임계값` 핵심 규율).

- 계류 법안은 **절대 material로 분류하지 않는다.** 공포(관보 게재) 확인 전까지 컴플라이언스 영향은 0.
- 심사단계가 아무리 진행돼도(본회의 가결 포함) 재료성은 `watch` 고정이다.
- policy-diff로 넘길 때는 **사전 포지셔닝 분석**으로만 — 갭 종결 diff로 넘기지 않는다. 그 분석 결과도
  gap-tracker에 `watch`로만 남고 Overdue에 진입하지 않는다.
- 다이제스트·출력의 모든 의안 항목에 명시한다: "계류 법률안 — 아직 법 아님. 심사단계 [단계].
  `status_verified: false`. 공포 확인 전 material 아님."

## 구성 의존 폴백

이 스킬은 watch 기관·분야·재료성 임계값을
`~/.claude/plugins/config/claude-for-legal-kr/regulatory-legal/CLAUDE.md`에서 읽는다. 필요한 값이
아직 `[PLACEHOLDER]`이거나 비어 있으면 출력에 구체적으로 말한다:

- **watch 기관·분야 비어 있음:** 여전히 사용자가 지정한 법안은 추적할 수 있으나 덧붙인다 — "구성에
  watch 분야가 없어 어느 법안이 우리 사업에 걸리는지 자동 판단할 수 없습니다. 사용자가 지정한
  의안번호·법안명만 추적합니다. `/regulatory-legal:cold-start-interview --redo`로 watch 분야를
  추가하세요."

관련 값이 채워져 있으면 구성에 대해 아무 말도 하지 않는다.

## 핸드오프

- **reg-feed-watcher로:** 공포 확인된 법안 → material 경로 승격(위 "공포 시 승격"). 국가법령정보로
  공포·시행 확인 후에만.
- **comments로:** 국회입법예고 의견제출 마감이 있는 법안 → comment-tracker에 `item_type: 의안`으로
  기록. comments가 마감을 추적.
- **policy-diff로:** 사전 포지셔닝 분석이 필요한 법안 → diff 실행 제안(**갭 종결 diff 아님**). 결과는
  gap-tracker에 `watch`로만.
- **gap-surfacer로:** 관심 법안 등록부 항목(`gap_type: watch`, `status_verified: false`) → 상태 보고의
  Watch 버킷에 표면화. **공포 확인 전 Overdue 진입 금지.**

## 다음 단계 의사결정 트리로 마무리

CLAUDE.md `## 산출물`의 다음 단계 의사결정 트리로 마무리한다. 이 스킬이 방금 산출한 것에 옵션을
맞춤화한다 — 다섯 디폴트 분기(X 초안 / 에스컬레이션 / 사실 추가 확보 / 관찰·대기 / 다른 것)는
출발점이지 고정이 아니다. 계류 법안 워치의 자연스러운 기본 옵션은 "관찰·대기"(다음 재검토 시점
메모와 함께)다. 데이터가 무거우면(추적 법안 10건 이상) 대시보드 옵션도 제안한다(CLAUDE.md `## 산출물`).
**신뢰 못 할 입력**(법안명·발의자·의안정보시스템 추출 텍스트)은 HTML 렌더 전 escape, 셀은
`textContent`로만. 트리가 산출물 — 변호사가 고른다.

## 이 스킬이 하지 않는 것

- **계류 법안을 material로 분류하지 않는다.** 공포(관보 게재) 확인 없이는 어떤 법률안도 material이
  아니다 — 이 플러그인의 가장 위험한 오분류.
- **국가법령정보로 계류 의안을 조회하려 하지 않는다.** 커넥터 범위 밖이다 — 의안정보시스템 직접 검색.
- **의안번호·심사단계를 지어내지 않는다.** 의안정보시스템에서 확인하거나 사용자에게 요청한다.
- **의견제출서를 작성하지 않는다.** 국회입법예고 의견제출 마감 추적은 comments, 의견서 초안은
  별도 변호사 작업(policy-redraft 또는 변호사).
- **통과 여부·통과 확률을 단정하지 않는다.** 심사단계를 surface하지, 정치적 전망을 예측하지 않는다.
  전망성 판단이 필요한 라인은 `[검토]`로 플래그한다.
- **공포 전 컴플라이언스 시계를 켜지 않는다.** 등록부 `due:`는 재검토일이지 컴플라이언스 기한이 아니다.
