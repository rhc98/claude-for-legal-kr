# /public-disclosure

> **참고:** 상류(Anthropic) corporate-legal의 Public Company 모듈은 스킬 미출시 상태다. 이 스킬은
> **KR 포팅이 선도**하는 한국 자본시장법·거래소 공시 스킬이다.

1. `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/CLAUDE.md` 로드 — `## 상장사` 모듈
   (시장·규모·공시 체계·소유상황보고·내부자거래·공정공시), `## 우리는 누구인가`(상장 여부·시장).
2. 아래 워크플로우 — 공시 트리거 판정 → 유형·기한 → 작성 체크리스트 → 공정공시·내부자거래 점검.
3. M&A 주요사항보고는 closing-checklist·merger-control-filing과, 이사회 결의 공시는 board-minutes와 연계.

---

## 매터 컨텍스트

실무 수준 CLAUDE.md `## 매터 워크스페이스` 점검. `활성화`가 `✗`(사내 디폴트)이면 skip. 활성인데 활성
매터 없으면 묻기. 다른 매터 파일 절대 안 읽음. **상장사 공시는 미공개중요정보(자본시장법 §174)와 직결 —
정보장벽·목적지 점검을 가장 엄격히 적용.**

---

## 목적

상장사는 사건·거래마다 **공시 의무**가 트리거되고, 기한·서식·정정·불성실공시 제재가 따른다. 이 스킬은
어떤 사건이 어느 공시를 발생시키는지 판정하고, 유형·기한을 정리하며, 작성 체크리스트를 산출한다.
**공시 대상·기한을 단정하지 않는다 — 거래소 공시규정·시행령에 달려 있고 갱신이 잦다(`[검토]`).**

## 결정 자세

- **공시 대상·기한·임계점을 하드코딩하지 않는다.** 자본시장법·시행령은 법망(커버)에서, **거래소 공시규정은
  법망 미커버**라 사용자 자료·거래소 공시가이드 대조. 법망 현행 조문:
  ```
  mcp__beopmang__law get "자본시장과 금융투자업에 관한 법률 제161조"   # 주요사항보고서
  mcp__beopmang__law get "자본시장과 금융투자업에 관한 법률 제147조"   # 대량보유보고(5%룰)
  ```
  미연결이면 `[모델 지식 — 검증 필요]`.
- **공시 위반·제재 여부를 결론짓지 않는다.** 의무 트리거·기한을 `[검토]`로 surface. 불성실공시 지정·제재는 거래소·증선위 몫.
- **거래소 공시규정·금융위/증선위 의결은 법망 미커버** → `[거래소 규정 — 직접 검증 필요]` / `[금융당국 의결 — 직접 검증 필요]`.

## 워크플로우

### Step 1: 공시 트리거 판정 (사건 → 공시)

사건·거래를 공시 유형에 매핑. 법망(자본시장법) + 거래소 규정 대조 — 아래는 가이드(`[검토]`):

| 공시 유형 | 근거 | 트리거 예시 | 기한(현행 대조) |
|---|---|---|---|
| **정기공시** | 자본시장법 §159·§160 | 사업보고서·반기·분기보고서 | 결산·반기·분기 후 법정기한 |
| **주요사항보고서** | 자본시장법 §161 | 합병·분할·영업양수도·자기주식 취득/처분·감자·해산·부도·대규모 결손 등 | 사유발생 익영업일 등 |
| **수시공시** | 거래소 공시규정(자율) | 주요 경영사항(투자·계약·소송·증자 등) — 시장별(유가/코스닥) 항목 상이 | 사유발생 당일/익일 `[거래소 규정 — 직접 검증 필요]` |
| **공정공시** | 거래소 공시규정 | 장래 실적·경영계획 등을 선별 제공 시 동시·사전 공시 | 제공 전/동시 |
| **자기주식** | 자본시장법·시행령 | 자기주식 취득·처분·신탁 | 이사회 결의·취득 전후 |

> 한 사건이 **복수 공시**를 동시에 트리거할 수 있다(예: 합병 = 주요사항보고 + 수시공시 + 주총 + 기업결합신고).
> 각각을 별도 항목으로 `[검토]`.

### Step 2: 소유·지분 변동 보고

- **임원·주요주주 소유상황보고(§173):** 임원·주요주주의 소유 주식 변동 보고. 보고 기한(변동일로부터 법정
  영업일 내 — 현행 대조). 미보고·지연 제재. `[검토]`.
- **단기매매차익 반환(§172):** 임원·주요주주가 6개월 내 매수·매도(또는 그 반대)로 얻은 이익 반환. 점검·고지.
- **주식등 대량보유보고(5%룰, §147):** 본인·특별관계자 합산 5% 이상 보유·이후 1% 이상 변동 시 보고
  (보유목적 — 경영참가/단순투자 구분). 기한·약식/일반 보고. M&A 지분취득 시 핵심. `[검토]`.

### Step 3: 미공개중요정보·내부자거래 점검(§174)

- 거래·공시 전 **미공개중요정보** 해당 여부, 내부자·준내부자·정보수령자 범위.
- **거래 windows·사전승인**(회사 내부자거래 정책 — config) 점검. 블랙아웃 기간 거래 금지.
- 공시 전 정보가 외부로 새지 않도록 정보장벽·목적지 점검(이 스킬의 모든 산출물에 적용).

## 데이터 갭

**한국거래소 공시규정(유가증권·코스닥)·공시서식·불성실공시 기준은 자율규정으로 법망 미커버**. 금융위·
증선위 의결, DART 공시 선례도 미커버. 인용 시 거래소 공시가이드·DART(`dart.fss.or.kr`) 자료를 사용자에게
요청하고 `[거래소 규정 — 직접 검증 필요]` / `[금융당국 의결 — 직접 검증 필요]`. 자본시장법·시행령 조문은
법망 커버이나 갱신 잦으니 시점 조회.

## 핸드오프

- **closing-checklist로:** M&A 주요사항보고(합병·분할·영업양수도)·자기주식이 종결·공시 게이트.
- **merger-control-filing로:** 기업결합 관련 공시·신고 병행.
- **board-minutes로:** 공시 대상 이사회 결의(자기주식·증자 등)의 의사록.
- **shareholder-meeting로:** 주총 결의 공시(합병·정관변경 등).

## 이 스킬이 하지 않는 것

- 공시 위반·제재를 결론짓지 않음 — 의무 트리거·기한을 점검하고 `[검토]` 플래그.
- DART에 공시를 대신 제출하지 않음 — 작성·기한을 준비, 제출은 공시담당·대리인.
- 거래소 수시공시 세부 항목을 단정하지 않음 — 자율규정이라 거래소 가이드 대조 필요.

## 마무리

CLAUDE.md `## 산출물`의 의사결정 트리로 종료. 공시 항목이 많으면 대시보드 제안(공시 유형·기한·담당 상태).
옵션을 맞춤(예: 주요사항보고서 초안, 5%룰 보고 타임라인, 내부자거래 사전승인 메모, 공정공시 대응).
