# /customize

## 실행 조건

사용자가 `/corporate-legal:customize` 입력. 콜드스타트 인터뷰 전체 재실행 없이, YAML 수동 편집 없이
실무 프로파일의 무엇 — 위험 성향·에스컬레이션 담당자·모듈 토글·중요성 임계점·산출물 포맷 — 을
변경하고 싶음.

## 할 일

1. **config 읽기.** `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/CLAUDE.md`
   (와 한 단계 위 `~/.claude/plugins/config/claude-for-legal-kr/company-profile.md`) 읽기.
   플러그인 config 없거나 `[PLACEHOLDER]` 값 남아 있으면:

   > 셋업을 아직 안 하셨습니다. 먼저 `/corporate-legal:cold-start-interview` 실행 —
   > customize는 이미 가진 프로파일을 조정하기 위한 것.

2. **맞춤화 가능 맵 보여주기.** 프로파일에 있는 것을 그룹화, 현재 값의 한 줄 요약과 함께:

   - **회사 / 우리는 누구인가** — 회사명·업종·관할·단계·상장/비상장·실무 환경
     *(모든 12 플러그인 공유 — `company-profile.md` 통해 변경 전파)*
   - **활성 모듈** — M&A / 이사회·간사 / 상장사 / 법인관리 중 무엇이 on. 모듈 on/off는 어느 스킬이 셋업을 요구하는지 바꿈.
   - **위험 성향** — 보수·중간·적극, 각각이 실사 중요성·공개목록 범위·`[검토]` 바에 의미하는 것
   - **인력** — 딜팀·이사회 간사·법인관리 담당·에스컬레이션 체인
   - **M&A 모듈** — 중요성 임계점(계약금액·소송 노출), 신뢰하는 VDR 플랫폼, AI 일괄검토 신뢰 수준(Luminance/Kira), 딜팀 브리핑 주기
   - **이사회·간사 모듈** — 하우스 결의문안 포맷·의사록 포맷·서명 선호·위원회 구조·공증 여부
   - **상장사 모듈** — 공시 일정·공시 체계·소유상황보고 추적·내부자거래 정책
   - **법인관리 모듈** — 법인 표·등기 관리·관할·변경등기 기한 추적
   - **워크플로우** — 매터 워크스페이스(딜룸)·종결 체크리스트 위치·VDR 워처 주기
   - **통합** — 데이터룸(Box·Intralinks·Datasite)·Google Drive·Slack·**국가법령정보 MCP** 상태·폴백

3. **무엇을 바꿀지 묻기.**

   > 무엇을 조정할까요? 섹션을 고르거나 자기 말로 변경 설명.

4. **변경 수집.** 사용자가 섹션을 고르면 그 섹션의 현재 값을 보여주고 새 값 묻기. 사용자가 자유
   형식으로 설명하면 어느 섹션이 영향인지 식별하고 mapping 확인.

5. **변경 전 검증.** 변경에 법적 인용·임계점·기한이 있으면 국가법령정보 대조 검증:
   ```
   mcp__korean-law__get_law_text "[조항]"
   ```
   인용이 안 맞으면 사용자에게 확인 ("당신은 X라 하셨는데, 국가법령정보의 현행 텍스트는 Y. 어느 것으로 갈까요?").

6. **변경 적용.** CLAUDE.md의 해당 섹션을 갱신. 변경 전후 diff 보여주기.

   예시:
   - *중요성 임계점 2.5억 → 5억:* "이제 `/diligence-issue-extraction`·`/material-contract-schedule`이
     5억을 cutoff로 다룹니다; 기존 발견은 그대로 유지, 소급 적용 원하면 재실행."
   - *상장사 모듈 켜기:* "해당 영역 스킬 실행 시 공시 일정·공시 체계를 묻겠습니다."
   - *AI 일괄검토 신뢰 "전 row 확인" → "10% 스팟체크":* "`/ai-tool-handoff`가 전체 대신 10% 샘플 QA."
   - *위험 성향 중간 → 적극:* "더 낮은 중요성 임계점을 올리고 `[검토]` 바를 높입니다."

7. **공유 프로파일 변경**(회사명·업종·관할·실무 환경·상장 여부): `company-profile.md`에 쓰고 안내:

   > 이 변경은 모든 12 플러그인에 영향 — 관할 풋프린트를 읽는 모든 플러그인이 이제 [새 값]을 봅니다.

8. **확인 + 검증 로그 기록.**
   ```
   [YYYY-MM-DD] customize - [성명]이(가) [섹션]에서 [구체적 변경]
   ```
   `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/verification-log.md`에 추가.

## 가드레일

- **섹션을 삭제하지 않는다.** 사용자가 "제거"를 원하면 `[미구성]`으로 설정하고 그것이 플러그인 동작에
  무엇을 의미하는지 설명. *(모듈 끄기는 예외 — 비활성 모듈 섹션은 통째로 생략되는 것이 정상.)*
- **내부 모순 플래그.** 변경이 프로파일을 모순되게 만들면(상장사 모듈 off + 에스컬레이션에 "공시 담당";
  적극 위험 성향 + 2,500만 원 중요성 임계점), 긴장을 플래그하고 어느 것을 원하는지 묻기.
- **가드레일 약화 플래그.** 사용자가 가드레일을 끄려 하면(`[검토]` 플래그 제거, 문서 헤더 생략,
  `[검증 필요]` 태그 제거), 그 가드레일이 무엇을 막는지 설명하고 trade-off를 이해하는지 확인.
  `[검토]` 플래그·출처 attribution 태그·인용 조문의 `[검증 필요]` 태그는 load-bearing이며 제거하지 않는다.
- **한 번에 하나.** 전체 인터뷰를 다시 묻지 않는다.
