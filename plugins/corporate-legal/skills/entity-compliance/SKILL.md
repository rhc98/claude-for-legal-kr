# /entity-compliance

1. `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/CLAUDE.md` → `## 법인관리 (Entity Management)` 로드(법인 표, 관할, 등기 관리 방식). **한국엔 registered agent 개념이 없다 — 등기는 본점소재지 관할 등기소.**
2. 플래그에 따라 모드 라우팅:
   - 플래그 없음 또는 `--init`: Mode 1 — 법인 표에서 트래커 초기화
   - `--report`: Mode 2 — 임박 기한·도과 항목 surface
   - `--update`: Mode 3a(수기) 또는 3b(--from-report 업로드) — 상태 업데이트
   - `--sweep`: Mode 3c — 미확인·도과 항목을 하나씩 점검
   - `--audit`: Mode 4 — 전체 헬스 감사
   - `--export`: Mode 5 — CSV 또는 표 내보내기
3. `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/entities/compliance-tracker.yaml` 읽기/쓰기.
4. 업데이트 후에는 변경 요약과 다음 행동을 보여준다.

## 매터 컨텍스트

실무 수준 CLAUDE.md `## 매터 워크스페이스` 점검. `활성화`가 `✗`(사내 사용자 디폴트)이면 이 단락 skip —
스킬은 실무 수준 컨텍스트를 쓰고 매터 기구는 보이지 않음. 법인관리는 통상 **한 회사군(모회사 + 자회사)**
단위이므로 사내 단일 그룹에서는 실무 수준 컨텍스트로 충분하다. 법무법인·소규모 사무소에서 여러 의뢰
회사군을 다루면 활성 매터를 확인하고, 활성인데 활성 매터가 없으면 묻기: "어느 회사군인가요?
`/corporate-legal:matter-workspace switch <slug>` 또는 `practice-level`." 활성 매터의 `matter.md`를
로드. **다른 매터 파일은 절대 안 읽음(매터 간 컨텍스트 `off`).** 출력은 매터 폴더
`~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/matters/<매터-slug>/`.

## 목적

변경등기, 정기주주총회, 재무제표 승인·공고, 사업보고서, 법인세 신고, 외부감사인 선임 — 모든 법인은
각자의 일정과 해태 시 각자의 결과(과태료·등기 반려·제재)를 가진다. 이 스킬은 무엇이 언제 어느 법인에
대해 도래하는지 아는 단일 YAML 트래커를 유지한다. 설계상 가볍다: 트래커는 사용자가 소유하는 파일이고,
Claude가 명령 시 갱신하며, 공유가 필요할 때 내보낸다.

> **미국 원본과의 핵심 차이.** 미국 원본은 "주(state)별 연차보고서·franchise tax + registered agent
> (CT Corp)" 모델이다. 한국엔 **registered agent 개념이 없다** — 상업등기는 **본점소재지 관할 등기소**에서
> 한다. 그래서 이 포트는 (a) 캘린더 항목을 한국 법정 의무(변경등기·정기주총·사업보고서·법인세·
> 외부감사) 유형으로, (b) "registered agent 보고서 ingest"를 **법무사·등기 위탁처(법무법인) 보고서
> ingest**로, (c) "good standing certificate"를 **법인등기부등본·증명서 확인**으로 재맥락화한다.

## 중요: 기한 참조 주의

> 이 스킬은 변경등기 기한(본점 2주·지점 3주), 과태료(상법 §635), 정기주총·재무제표·사업보고서·법인세·
> 외부감사 관련 기한을 **기억으로 하드코딩하지 않는다.** 법정 기한·임계점·과태료 금액은 법령·시행령
> 개정으로 바뀌고, 산정 기준(자산·매출·결산기·연결/별도)에 따라 적용 여부가 달라진다.
> **의존 전 현행 조문(상법·외감법·자본시장법·법인세법 등)을 국가법령정보에서 대조하라.** 등기를 법무사·
> 법무법인에 위탁한 경우, 그 위탁처의 컴플라이언스 캘린더가 해당 법인에 대해 권위 있는 1차 자료다 —
> 이 트래커는 그 데이터를 조직·surface하는 도구이지 대체물이 아니다. 모든 기한 인용은 `[검토]`
> 또는 `[모델 지식 — 검증 필요]`로 플래그한다.

## 관할 가정

> 이 트래커는 각 법인에 기록된 **설립 준거법·관할(국내/해외)**과 **법인 형태**를 기준으로 의무를
> 계산한다. 등기·신고 규칙·기한 산정·수수료 구조는 관할과 형태에 따라 크게 다르다. 법인의 실제
> footprint가 `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/CLAUDE.md` 법인 표와 다르면
> (미공개 해외 자회사, 휴면·해산 예정 법인, 본점 이전, 현지 대리인이 관리하는 외국 신고),
> 출력이 그대로 적용되지 않을 수 있다 — 해당 관할의 법무사·현지 전문가에게 확인.
> **외국 법인·외국 상장·외국 준거법 사안은 `[외국법 — 외국 변호사 검증 필요]`로 플래그하고 현지
> 회사법 전문가 자문 권고.** 한국 등기 doctrine을 외국 법인에 침묵으로 적용하지 않는다.

## 법인 형태 disambiguation (핵심)

> 캘린더는 **관할만이 아니라 법인 형태**에 따라 달라진다. "한국 법인"을 단일 버킷으로 다루는 것은
> 흔하고 결과가 큰 오류다 — **주식회사 / 유한회사 / 유한책임회사 / 외국회사(국내영업소·지점)**는
> 적용되는 의무·기한·해태 결과가 다르다. 기한을 계산·리포트하기 전 법인 표에서 형태를 확인하고,
> 한 형태의 기한을 같은 관할의 다른 형태에 복사하지 않는다.
>
> **형태별로 갈리는 지점(현행 조문 대조 필요 — 단정 금지):**
>
> - **주식회사:** 변경등기(임원변경·본점이전·자본금변경 등)는 본점 **2주 내**(주식회사는 상법 **§317④**이
>   §181~§183을 준용; §183 자체는 합명회사 조항이므로 주식회사엔 §317④ 경유), 지점 **3주 내**. 변경등기
>   해태 시 **과태료(§635)**. **정기주주총회**(결산 후 일정 기간 내)·**재무제표 승인·공고**·외감 대상이면
>   **외부감사인 선임·내부회계관리제도**. 상장·외감 등 요건 충족 시 **사업보고서**. `[검토]`
> - **유한회사:** 사원총회 구조가 주식회사 주총과 다르고, 재무제표·공고 의무·외감 적용 범위가 다를 수
>   있다(외감법 대상 판단은 형태 불문 자산·매출 기준으로 별도 검토). 변경등기 의무는 존재하나 준용 관계·
>   기재사항이 주식회사와 다르다 — `[검토]`.
> - **유한책임회사(LLC형, 상법 2011 개정 도입):** 업무집행자 구조·등기사항이 주식회사·유한회사와 또 다르다.
>   주총·이사회 개념이 그대로 적용되지 않는다 — 정관·현행 조문 대조 `[검토]`.
> - **외국회사 지점·국내영업소:** 상법 §614 이하 등기·대표자 선임·영업소 설치 등기. 한국 회사법 그대로
>   적용되지 않으며 본국법과 한국 등기·세무·외감 규율이 교차한다 — `[외국법 — 외국 변호사 검증 필요]`
>   + 한국 등기·세무 `[검토]`.
>
> 법인 표에 형태 없이 기록된 법인은 `type_unknown`으로 플래그하고 어느 기한도 계산하기 전 사용자에게
> 형태를 확인한다. **외감 대상 여부**는 형태와 별개로 자산·매출·부채·종업원 기준(외감법 시행령)으로
> 판정되며 **매년 기준이 갱신**되므로 `[회사 규모 산정 — 검증 필요]`로 플래그하고 현행 시행령 대조.

## 트래커 파일

`~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/entities/compliance-tracker.yaml`에 산다. 구조:

```yaml
# 법인 컴플라이언스 트래커 (법정 기한 캘린더)
# 생성: [날짜]
# 최근 업데이트: [날짜]
# 주의: 기한은 참조용 — 의존 전 현행 조문(상법·외감법·자본시장법·법인세법) 및 등기 위탁처 확인

metadata:
  company_group: "[회사군 / 모회사명]"
  generated: "[날짜]"
  last_updated: "[날짜]"
  last_audit: "[날짜 또는 null]"

custom_jurisdictions:   # 수기 추가 — 빌트인 참조에 없는 관할(해외 등)
  []                    # 새 관할 만나면 채워짐

entities:
  - name: "[법인명]"
    type: "[주식회사 / 유한회사 / 유한책임회사 / 외국회사 지점 / type_unknown]"
    home_jurisdiction: "[국내(본점소재지) / 해외 관할명]"
    registry_office: "[본점소재지 관할 등기소 — 예: 서울중앙지방법원 등기국]"   # registered agent 아님
    filing_handler: "[법무사 / 법무법인 위탁처 / 사내]"   # 미국 registered agent의 KR 대응 = 등기 위탁처
    formation_date: "[설립등기일 또는 null]"
    fiscal_year_end: "[결산기 — 예: 12-31]"   # 정기주총·재무제표·법인세 기한의 기준점
    parent: "[모회사 또는 null]"
    ownership_pct: "[지분% 또는 null]"
    external_audit: "[대상 / 비대상 / unknown]"   # 외감법 대상 여부 — [회사 규모 산정 — 검증 필요]
    listed: "[상장(유가/코스닥/코넥스) / 비상장]"   # 사업보고서·소유상황보고 트리거
    status: "[활성 / 휴면 / 해산예정]"
    agent_managed: false   # 외국 법인 등 현지 대리인이 컴플라이언스를 관리하면 true
    local_agent: "[현지 대리인명 또는 null]"   # 외국 법인용 — 본점 등기소 대체 아님
    notes: ""

    filings:
      - type: "[변경등기 / 정기주총 / 재무제표승인·공고 / 사업보고서 / 법인세 / 외부감사인선임 / 기타]"
        trigger_basis: "[고정일 / 결산기 기준 / 변동사유 발생일 기준 / 설립일 기준]"
        # 변경등기: due_basis가 "사유발생일 + 2주(본점) / 3주(지점)" — 사유 발생 시 동적 산정
        due_date: "[YYYY-MM-DD 또는 null]"
        last_filed: "[날짜 또는 null]"
        last_fee: "[등록면허세·수수료 등 또는 null]"
        status: "[current / due_soon / overdue / unknown]"
        last_verified_registry: "[등기부등본 최종 확인일 또는 null]"   # good standing의 KR 대응
        statute: "[근거 조문 — 예: 상법 §317④ (§181~§183 준용) [검토]]"
        notes: ""
```

상태 값:
- `current` — 현재 기간분 완료, 90일 내 도래 없음
- `due_soon` — 90일 내 도래
- `overdue` — 기한 도과인데 이행 기록 없음 (**변경등기 도과는 과태료 §635 노출 — 헬스 감사에서 강조**)
- `unknown` — 정보 없음, 수기 확인 필요

> **변경등기는 사유 기반(event-driven)이다.** 미국 연차보고서는 고정일·설립기념월 기반이지만, 한국
> 변경등기는 **변동사유(임원변경·본점이전·자본금변경·상호변경 등) 발생일로부터 2주(본점)/3주(지점)**
> 기산이다. 따라서 트래커에는 미리 박힌 고정 기한이 아니라 **변동사유가 발생하면 그때 동적으로 due_date를
> 산정**하는 항목이 들어간다. 정기 도래분(정기주총·법인세·사업보고서)과 사유 도래분(변경등기)을 모두
> 추적한다. 기한·준용 관계·과태료는 현행 조문 대조 `[검토]`.

## Mode 1: 초기화

트래커가 없을 때, 또는 `--rebuild`로 처음부터 재생성할 때 실행.

### Step 1: 법인 표 로드

`~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/CLAUDE.md` → `## 법인관리 (Entity Management)` →
법인 표를 읽는다(콜드스타트의 조직도 업로드에서 채워진). 채워져 있으면 그대로 사용. 없으면 사용자에게
콜드스타트 법인관리 모듈 실행 또는 법인 목록 제공을 요청.

### Step 2: 각 법인 × 관할에 대해 의무를 확인

각 법인에 대해 현행 신고·등기 일정을 등기 위탁처(법무사·법무법인) 또는 현행 조문으로 확인한다. 법정
기한·임계점은 개정되고(외감 대상 자산·매출 기준은 **매년 갱신**, 변경등기 기한·과태료는 상법 개정 대상),
**캐시된 일정에 의존하지 않는다.** 트래커는 확인한 사실을 기록한다.

법인 형태·관할(국내/해외)·결산기·상장 여부·외감 대상 여부에 따라 다음 의무를 식별한다:

1. 사용자에게 **등기 위탁처(법무사·법무법인)의 현행 컴플라이언스 캘린더·등기부등본**이 있는지 묻는다 —
   가장 권위 있는 1차 자료.
2. 없으면 사용자가 아는 것을 묻는다(신고·등기 유형, 결산기, 최근 정기주총일, 최근 변경등기 사항·일자,
   외감 대상 여부, 상장 여부). 제공한 것을 기록.
3. 사용자가 모르는 것은 해당 법인 × 의무 항목을 `unknown`으로 플래그 — 캐시 참조에서 기한을 채우지
   않는다. 다음 단계는 등기 위탁처·현행 조문 대조.

**참조 표가 아니라 트래커에 세부를 캡처:**

> [관할/형태]에 대한 의무 일정이 참조에 없습니다. 추적을 위해 캡처하겠습니다.
>
> [법인] ([형태], 결산기 [MM-DD]):
> 1. 어떤 정기 의무가 적용되나요? (정기주총·재무제표 승인·공고, 사업보고서, 법인세, 외부감사인 선임 중)
> 2. 변경등기가 필요한 변동사유가 최근 있었나요? (임원변경·본점이전·자본금변경 등 — 있으면 발생일)
> 3. 외감법 대상인가요? (자산·매출·부채·종업원 기준 — 모르면 "unknown", 산정은 `[회사 규모 산정 — 검증 필요]`)
> 4. 등기는 누가 처리하나요? (법무사·법무법인 위탁처 / 사내)

답을 트래커의 `custom_jurisdictions` 블록에 저장:

```yaml
custom_jurisdictions:
  - jurisdiction: "[국내 / 국가명]"
    jurisdiction_type: "[국내 본점소재지 / 해외 관할]"
    filings:
      - type: "[의무 유형]"
        trigger_basis: "[고정일 / 결산기 기준 / 사유발생일 기준]"
        statute: "[근거 조문 또는 현지법 — [검토] / [외국법 — 외국 변호사 검증 필요]]"
        notes: "[기타 — 예: 현지 대리인 필요, 현지어 신고]"
    added_by: "manual"
    added_date: "[날짜]"
```

이 정의는 그 관할의 모든 법인에 적용된다. 이후 `--init`·법인 추가 시 자동 사용.

**해외 관할 특히:**

해외 신고는 관할별로 크게 다르다. 위 custom 정의 플로우를 반드시 거친다 — 현지 등기·신고 대리인과
유형·주기·수수료를 확인한 뒤 트래커에 채운다. 외국 법인은 추가로:

- 현지에 컴플라이언스를 처리하는 등기·신고 대리인이 있나? 있으면 대리인명을 기록 — 트래커는 독립적으로
  기한을 계산하기보다 그 대리인에게 후속 확인할 시점을 플래그한다.
- 그 관할에서 그룹 차원 보고(실질적 소유자 등록, 경제적 실질 신고, 국가별 보고 등)가 필요한가?

현지 대리인이 있는 외국 법인은 `agent_managed: true`로 플래그. 리포트 모드는 이들을 별도로 나열하고
계산된 기한 대신 "현지 대리인에게 상태 확인" 메모를 단다. **외국 법인은 `[외국법 — 외국 변호사 검증 필요]`.**

결산기 기준 의무(정기주총·법인세·사업보고서)는 트래커의 `fiscal_year_end`에서 계산. 결산기가 null이면
상태를 `unknown`으로 두고 확인 플래그. **사유 기반 변경등기는 미리 기한을 박지 않고, 변동사유가 들어올 때
산정**(Mode 3 / Mode 2 참조).

### Step 3: 트래커 쓰기

`~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/entities/compliance-tracker.yaml`을 모든 법인·
의무와 함께 생성. 초기 상태:
- 현재 기간분이 이행되었으면 `current`
- 90일 내 도래·현재 기간분 미이행이면 `due_soon`
- 기한 도과·미이행이면 `overdue`
- 결산기·형태가 없거나 관할이 참조에 없으면 `unknown`

생성 후 요약:

```
법인 컴플라이언스 트래커 초기화 완료.

법인: [N]개
관할(국내/해외): [N]
추적 의무: [N]건

상태 요약:
  ✅ 정상:      [N]
  ⏰ 임박:      [N] (향후 90일)
  🔴 도과:      [N]  (변경등기 도과는 과태료 §635 노출)
  ❓ 미확인:    [N] (등기 위탁처·현행 조문 확인 필요)

/corporate-legal:entity-compliance --report 로 도래분을 확인하세요.
```

## Mode 2: 리포트

임박 기한·도과 항목을 surface. 디폴트: 향후 90일.

```
/corporate-legal:entity-compliance --report [--days 30|60|90|180]
```

출력 포맷:

```
법인 컴플라이언스 리포트 — [날짜]
[회사군 / 모회사명]

🔴 도과 ([N]):
  [법인] / [관할] / [의무 유형] — [날짜] 도래분 미이행
    ↳ 변경등기 항목이면: 과태료(상법 §635) 노출 — [검토]

⏰ [N]일 내 도래 ([N]):
  [법인] / [관할] / [의무 유형] — [날짜] 도래  [등기 위탁처]
  [법인] / [관할] / 정기주총 — 결산기([MM-DD]) 기준 산정일  [검토]

✅ 최근 이행 (최근 90일 [N]건):
  [법인] / [관할] / [의무 유형] — [날짜] 이행

❓ 미확인 상태 ([N]):
  [법인] / [관할] / [의무 유형] — 정보 없음; 등기 위탁처·현행 조문 확인

🌐 현지 대리인 관리 (외국 법인 [N]):
  [법인] / [국가] / [의무 유형] — [현지 대리인] 관리; 직접 상태 확인 [외국법 — 외국 변호사 검증 필요]
  [법인] / [국가] — 현지 대리인 미기록; --update 로 추가

등기부 확인 상태 (good standing 대응):
  최종 확인: [날짜]
  등기부등본 확인된 법인: 전체 [N]개 중 [N]개
  최근 12개월 미확인: [목록]
```

> **변경등기 항목 표시.** 변경등기는 사유 기반이라 평시엔 도래분이 없을 수 있다. 그러나 **알려진 미반영
> 변동사유**(예: 지난달 임원 변경, 본점 이전 결정)가 있으면 "사유발생일 + 2주(본점)/3주(지점)"로 due_date를
> 산정해 ⏰/🔴에 표시한다. 사유가 없으면 변경등기 항목은 리포트에 나타나지 않는다 — 정상.

트래커가 ~10개 법인을 초과하거나 사용자가 요청하면: 대시보드 제안(CLAUDE.md `## 산출물 → 데이터 무거운
산출물의 대시보드 옵션`). 이 출력에 맞춰 — 의무 상태별 카운트(도과/임박/이행/미확인), 등기부 확인 상태별
카운트, 관할·의무 유형·차기 기한 컬럼의 정렬 가능 법인 표.

## Mode 3: 업데이트

트래커의 하나 이상 법인을 갱신. 세 서브모드:

### 결과적 행위 게이트 (등기 신청·신고 지시)

**등기·신고를 지시·확정하기 전:** `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/CLAUDE.md`의
`## 누가 이 플러그인을 사용하나` 읽기. 역할이 **변호사 아님**이면:

> 변경등기 신청, 정기주총 결의·재무제표 승인, 사업보고서·법인세 신고는 법적 결과를 낳습니다 — 법인의
> 공식 표시이고, 등록면허세·수수료가 들며, 누락·오류는 등기 반려·과태료(상법 §635)·외감 위반·공시
> 위반으로 이어질 수 있습니다. 변호사·법무사와 검토하셨나요? 예라면 이행을 기록하겠습니다. 아니라면
> 가져갈 브리프:
>
> - 법인·관할·의무 유형·기한(변경등기면 변동사유 발생일 + 2주/3주 기산)
> - 트래커가 보여주는 직전 이행(일자·수수료·반영된 임원/자본 정보)
> - 미해결 질문(임원·본점·자본 정보가 여전히 정확한가, 첨부 의사록 공증이 필요한가, 결산기·외감
>   대상 여부에 변동이 있는가)
> - 무엇이 잘못될 수 있나(변경등기 기한 도과 → 과태료, 외감 대상인데 외부감사인 선임 누락, 사업보고서
>   기한 도과, 정기주총 미개최)
> - 변호사·법무사에게 물을 것(올해 실제로 필요한 신고·등기는 무엇인가, 반영해야 할 정관변경·임원변경이
>   있나, 누가 서명·날인하나)
>
> 변호사·법무사를 찾아야 하면: 대한변호사협회·지방변호사회의 변호사 소개, 대한법무사협회, 또는 사내라면
> 자문 로펌·법무팀·등기 위탁처에 연결.

이 게이트를 명시적 "예" 없이 넘어 새 `last_filed` 일자를 기록하지 않는다. 트래커 읽기·기한 리포트·"무엇이
도래" 출력은 게이트 불필요.

### 3a: 수기 업데이트

```
/corporate-legal:entity-compliance --update
```

변호사·간사가 무엇을 했는지 말한다:
> "[법인]의 본점이전 변경등기를 3월 1일에 완료. 등록면허세 [N]원."
> "[법인] 정기주주총회 3월 20일 개최, 재무제표 승인·이사 선임 완료."

Claude 갱신:
- `last_filed` → 일자
- `last_fee` → 수수료·등록면허세
- `status` → `current` (변경등기면 그 사유 항목 closure)
- metadata `last_updated`

### 3b: 등기 위탁처 보고서 업로드

```
/corporate-legal:entity-compliance --update --from-report
```

사용자가 **법무사·법무법인 등기 위탁처의 컴플라이언스 보고서·등기부등본·결산 일정표**(PDF·CSV·Excel)를
업로드. (미국 원본의 "CT Corp / registered agent 보고서 ingest"의 KR 대응.) Claude가 읽고 매칭되는
법인을 갱신:

보고서에서 각 법인에 대해 추출:
- 의무 유형·기한(변경등기 사유·일자, 정기주총·재무제표·사업보고서·법인세·외부감사인 일정)
- 최근 이행 일자(있으면)
- 등기부등본 확인 상태·일자 (good standing 대응)
- 위탁처의 플래그·경고

보고서 법인을 트래커 법인에 이름으로 매칭(근사 매칭은 확인 플래그 — "(주)에이스" vs "에이스 주식회사"는
같은 법인일 가능성).

처리 후:
```
보고서에서 [N]개 법인 갱신.

매칭: [N]
미매칭 (보고서엔 있으나 트래커에 없음): [목록 — 법인 표에 추가 필요할 수 있음]
보고서에 없음 (트래커엔 있으나 갱신 없음): [목록 — 상태 유지]
```

### 3c: 일괄 상태 점검

```
/corporate-legal:entity-compliance --sweep
```

`unknown`·`overdue` 상태인 각 법인을 하나씩 돌며 현재 정보를 묻는다:

> [법인] / [관할] / [의무 유형] — 현재 [상태]로 표시됨.
> 이행되었나요? 예라면 언제, 수수료는? (변경등기면 변동사유와 발생일도 — 기한 기산점)

각 확인 후 트래커 갱신. 완료 요약 산출.

## Mode 4: 헬스 감사

```
/corporate-legal:entity-compliance --audit
```

단순 이행 상태를 넘는 광범위 검토. surface:

**이행 컴플라이언스:**
- 도과 항목(리포트 모드에서)
- 미확인 상태 항목

**등기·과태료 리스크 (KR 핵심):**
- **변경등기 기한 도과 — 과태료(상법 §635) 노출.** 알려진 변동사유(임원변경·본점이전·자본금변경 등)가
  발생일 + 2주(본점)/3주(지점)를 지났는데 미이행이면 강조. 과태료 금액·부과 메커니즘은 단정하지 않고
  현행 조문 대조 `[검토]`.
- **정기주총 미개최** — 결산기 기준 개최 기한을 지났는데 개최 기록 없음. 재무제표 승인 연쇄 지연.
- **외감 대상인데 외부감사인 선임·내부회계관리제도 누락** — `external_audit: 대상`인데 선임 기록 없음.
  외감 대상 여부 산정 자체가 게이트 `[회사 규모 산정 — 검증 필요]`.
- **사업보고서·법인세 도과** — 상장·외감 대상이거나 법인세 신고기한 경과.

**법인 헬스:**
- `휴면`으로 표시된 법인 — 검토 플래그: 청산·해산해야 하나? 휴면 법인 유지는 비용(정기주총·등기·외감
  대상이면 감사 비용, 위탁 수수료)과 지속 컴플라이언스 의무를 발생시킨다.
- 설립등기일 5년 초과 + `휴면` 법인 — 청산·해산 후보 플래그.
- 설립등기일·결산기 없는 법인 — 데이터 갭 플래그.

**등기부 확인 갭 (good standing 대응):**
- `last_verified_registry` 일자 없는 법인 — 등기부 상태 미확인; M&A·금융 거래에서 단기간에 등기부등본·
  증명서가 필요할 때 리스크.

## 이 스킬이 하지 않는 것

- 아무것도 신청·신고·등기하지 않는다. 산출은 트래커와 할 일 목록이며, 등기·신고는 변호사·법무사·
  등기 위탁처가 한다.
- 등기부등본·증명서를 떼지 않는다. 마지막 확인 시점을 추적하되, 발급은 수기 또는 등기 위탁처를 통한다.
- 지점 설치·외국회사 영업소·해외 자회사 등기가 필요한지 독립 판정하지 않는다. 그 분석은 영업 실재
  사실에 달려 있고 변호사가 확인해야 한다.
- 변경등기 기한·과태료 금액·외감 대상 임계점·정기주총 기한을 기억으로 단정하지 않는다. 모든 법정
  기한·임계점은 **현행 조문(상법·외감법·자본시장법·법인세법) 대조** — `[검토]` / `[모델 지식 — 검증 필요]`.
- 복잡한 다법인 구조에서 등기 위탁처(법무사·법무법인)를 대체하지 않는다. 이 스킬은 위탁처가 없는
  소규모 조직, 또는 위탁처 데이터 위의 가벼운 레이어로 가장 적합하다.
- 기한 참조는 법률 자문이 아니며 현행 요건을 반영하지 않을 수 있다. 의존 전 모두 확인.

> **등기선례·등기예규 주의.** 변경등기 실무의 세부(첨부서류·기재방식·준용 관계 해석)는 **등기선례·
> 등기예규**에 의존하는 경우가 많은데, 이는 국가법령정보가 부분 커버다. 인용해야 하면 `[등기선례 — 검증 필요]`로
> 플래그하고 대법원 인터넷등기소·등기예규를 사용자에게 요청.

## Formula injection 방어

Excel·시트·CSV 출력의 모든 셀을 쓰기 전 formula injection을 무력화한다. 외부 출처 텍스트(법인명, 등기
위탁처 보고서 텍스트, 업로드 일정표 추출 문자열)는 공격자 통제 가능. `=`, `+`, `-`, `@`, 탭, 캐리지리턴,
개행으로 시작하는 셀은 수식으로 해석되거나 행 구조를 깨뜨린다.

- **단일 인용부호 prefix:** `'=SUM(A1:A10)` (수식 실행 대신 텍스트 표시)
- **문서·도구 결과·사용자 paste에서 온 텍스트를 담는 모든 셀에 적용.** 사용자가 통제하는 컬럼 헤더와
  직접 산출한 계산값은 안전.
- **CSV: 내장 쉼표·큰따옴표·개행도 escape**(RFC 4180 quoting).
- 선택이 아니다. 사용자가 Excel에서 열었을 때 매크로를 트리거하거나 DDE로 데이터를 유출하는 스프레드시트는
  사용자에 대한 공급망 공격이다.
