# /shareholder-meeting

1. `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/CLAUDE.md` 로드 — `## 이사회·간사` 모듈,
   `## 적용범위 overlay`(상장/비상장·소규모회사), `## 우리는 누구인가`(상장 여부·자본금).
2. 아래 워크플로우 사용 — 안건 분류 → 소집절차 → 정족수·결의요건 → 통지·공시 → 운영 체크리스트.
3. 의사록은 board-minutes로, 영업양도·합병 주총은 closing-checklist로, 서면결의 가능 여부는 written-consent로 연계.

## 목적

주주총회는 **안건마다 결의요건이 다르고**(보통/특별/특수), **소집절차·통지 기한·정족수**가 어긋나면
결의가 취소·무효 위험에 놓인다. 이 스킬은 안건을 분류하고, 소집·통지·의결권·정족수를 점검하며, 운영
체크리스트를 산출한다. **결의 성립·하자 여부는 단정하지 않는다 — 정관·주식 종류·사안에 달려 있다(`[검토]`).**

## 결정 자세

- **정족수·결의요건·소집 기한·임계점을 하드코딩하지 않는다.** 정관과 현행 상법·상장사 특례·시행령에 달려
  있고 갱신된다. 국가법령정보 현행 조문을 가져와 적용:
  ```
  mcp__korean-law__get_law_text "상법 제434조"   # 특별결의
  mcp__korean-law__get_law_text "상법 제363조"   # 소집통지
  ```
  미연결이면 `[모델 지식 — 검증 필요]`.
- **결의 하자(취소·무효·부존재)를 결론짓지 않는다.** 절차 점검 결과를 `[검토]`로 surface. 하자 판단은 법원 몫.
- 상장사는 **상법 상장회사 특례(§542-2 이하)**가 추가 적용 — 소집통지·전자투표·집중투표 등에서 비상장과 다름. `[검토]`.

## 워크플로우

### Step 1: 안건 분류 (결의요건)

각 안건이 보통/특별/특수 중 무엇인지 분류. 국가법령정보 현행 조문 대조 — 아래는 분류 가이드(`[검토]`):

| 결의 종류 | 요건(현행 대조) | 대표 안건 |
|---|---|---|
| **보통결의** (상법 §368) | 출석 의결권 과반수 + 발행주식총수 1/4 이상 | 재무제표 승인, 이사·감사 선임, 이사 보수 한도, 배당 |
| **특별결의** (상법 §434) | 출석 의결권 2/3 이상 + 발행주식총수 1/3 이상 | 정관변경, 영업양도(§374), 합병(§522)·분할(§530-3), 감자, 해산, 이사 해임 |
| **특수결의** | 총주주 동의 등 | 이사 책임면제(§400①), 주식회사→유한회사 조직변경 등 |

> 정관이 가중(예: 더 높은 정족수)·종류주주 보호를 둘 수 있다 — 정관 우선 확인. 의결권 없는 주식·자기주식
> (의결권 제한)·특별이해관계인(§368③ 의결권 제한)을 정족수 산정에서 제외. `[검토]`.

### Step 2: 소집절차 점검

- **소집 결정:** 이사회 결의(§362). 소수주주 소집청구(§366)·법원 허가 소집 경로도 인식.
- **소집통지 기한·방법:** 주총일 **2주 전** 통지(§363) — 상장사는 특례(전자공고·통지 갈음 등). 통지에
  **회의 목적사항** 기재(§363②); 특별결의 안건은 요령 기재. 자본금 10억 미만 소규모회사는 절차 간소화·
  전원동의 서면결의(§363④, → written-consent) 가능.
- **주주제안(§363-2):** 주주제안 안건이 있으면 통지 반영 여부 점검.

### Step 3: 의결권 행사 방법

- **전자투표(§368-4):** 이사회 결의로 도입. 도입 시 통지·공고·시스템(예탁결제원 K-eVote 등) 점검.
- **서면투표(§368-3):** 정관에 정함이 있어야. 서면투표 용지·기재.
- **의결권 대리행사·위임장(§368②):** 위임장 양식·대리행사 권유 규제(상장사) 점검. 전자위임장 권유 가능.
- **집중투표(§382-2):** 2명 이상 이사 선임 시, 정관 배제 없으면 소수주주 청구 가능(상장 대규모회사 특례 주의). `[검토]`.

### Step 4: 종류주주총회·검사인

- **종류주주총회(§435):** 종류주식이 있고 특정 종류 주주에 손해 미칠 정관변경 등은 종류주총 별도 결의 필요. `[검토]`.
- **검사인(§367):** 총회 소집절차·결의방법의 적법성 조사를 위한 검사인 선임 여부(주주 청구·법원 선임).

### Step 5: 반대주주 주식매수청구 연계

영업양도(§374-2)·합병(§522-3)·분할합병(§530-11) 등 특별결의 안건은 **반대주주 주식매수청구권**이 따른다 —
사전 반대 통지 → 결의 → 매수청구 기간 → 가격 결정 타임라인. closing-checklist·merger-control-filing과 연계. `[검토]`.

### Step 6: 운영 체크리스트·산출

```markdown
[문서 헤더 — config ## 산출물 따라, 역할별 상이]

# 주주총회 운영 체크리스트: [회사] — [정기/임시] ([날짜])

**상장 여부:** [상장/비상장]  ·  **자본금:** [N원]  ·  **소규모회사 특례:** [해당/비해당 — 검토]

### 안건별 결의요건
| 안건 | 결의 종류 | 요건 | 매수청구 | 비고 |
|---|---|---|---|---|
| [안건] | [보통/특별/특수] | [§368/§434 — 검토] | [유/무] | [종류주총 필요? 등] |

### 소집절차
- [ ] 이사회 소집결정(§362) — [날짜]
- [ ] 소집통지 발송(주총 2주 전 §363 / 상장 특례) — [기한]
- [ ] 목적사항·요령 기재(§363②)
- [ ] (전자투표 도입 시) 공고·시스템
- [ ] (상장) 거래소 공시·사업보고서 연계 → public-disclosure `[거래소 규정 — 직접 검증 필요]`

### 정족수 산정(검토)
- 발행주식총수 [N], 의결권 있는 주식 [N](자기주식·무의결권 제외), 특별이해관계인 제외 [N]

### 다음 단계
[의사록 → board-minutes / 영업양도·합병 → closing-checklist / 서면결의 가능 여부 → written-consent]
```

## 데이터 갭

상장사 주총 관련 **거래소 공시규정·기업지배구조 모범규준**은 국가법령정보 미커버·부분커버 → `[거래소 규정 — 직접
검증 필요]`. 전자주주총회 등 **개정·계류 동향**은 `[모델 지식 — 검증 필요]`로 surface하되 시행 여부 확인.

## 핸드오프

- **board-minutes로:** 주총 의사록 작성.
- **closing-checklist로:** 영업양도·합병·분할 주총 특별결의·매수청구가 종결 게이트.
- **merger-control-filing로:** 결합 관련 안건.
- **written-consent로:** 소규모회사 주주 전원동의 서면결의(§363④)가 회의 대안일 때.
- **public-disclosure로:** 상장사 주총 소집·결의 공시.

## 이 스킬이 하지 않는 것

- 결의 성립·하자(취소·무효)를 결론짓지 않음 — 절차를 점검하고 `[검토]`로 플래그.
- 정족수를 단정하지 않음 — 정관·주식 종류·특별이해관계를 확인해 산정 가이드를 제공.
- 의사록을 작성하지 않음 — board-minutes로 핸드오프.

## 마무리

CLAUDE.md `## 산출물`의 의사결정 트리로 종료. 안건·일정이 많으면 대시보드 제안(안건별 결의요건·매수청구·
소집 진행 상태). 옵션을 이 산출물에 맞춤(예: 소집통지문 초안, 위임장 양식, 의사록 골격, 매수청구 타임라인).
