# /diligence-issue-extraction

1. `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/CLAUDE.md` + `~/.claude/plugins/config/claude-for-legal-kr/corporate-legal/deals/[코드]/deal-context.md` 로드.
2. 아래 워크플로우 사용.
3. `ai-tool-handoff` 점검 — 카테고리가 bulk이고 도구가 구성돼 있으면 먼저 핸드오프.
4. 문서 읽기, 중요성 필터 적용, 카테고리별 추출.
5. 발견을 하우스 메모 포맷으로. 종결 액션(consent 등)은 closing-checklist로 핸드오프.

## 매터 컨텍스트

실무 수준 CLAUDE.md `## 매터 워크스페이스` 점검. `활성화`가 `✗`(사내 사용자 디폴트)이면 이 단락 skip —
스킬은 실무 수준 컨텍스트를 쓰고 매터 기구는 보이지 않음. 단, **사내라도 M&A는 딜 단위로 분리**될 수
있으니 deal-context가 있으면 우선. 활성인데 활성 매터가 없으면 묻기: "어느 매터인가요?
`/corporate-legal:matter-workspace switch <slug>` 또는 `practice-level`." 활성 매터의 `matter.md`를
로드. **다른 매터 파일은 절대 안 읽음(매터 간 컨텍스트 `off`).** M&A는 정보장벽이 특히 중요.

중요성 임계점에 대조해 문서를 읽고, 이슈를 추출해 하우스 메모 포맷으로 쓴다.

## 컨텍스트 로드

- 실무 CLAUDE.md `## M&A → 실사 구조`(카테고리·중요성 임계점)

## 워크플로우

### Step 1: VDR 인벤토리

VDR MCP(Box/Intralinks/Datasite)가 연결돼 있으면 인덱스를 가져온다. VDR 폴더를 실사 요청목록
카테고리에 매핑. 갭 메모 — 대응 VDR 콘텐츠가 없는 요청 카테고리.

```markdown
## VDR 인벤토리: [딜 코드]

| 요청 카테고리 | VDR 폴더 | 문서수 | 상태 |
|---|---|---|---|
| 회사 일반·등기 | /01-회사 | 45 | 검토완료 |
| 주요 계약 | /02-계약 | 312 | 진행중 |
| 지식재산 | /03-IP | 89 | 미착수 |
| 인사·노동 | /04-HR | 67 | 미착수 |
| [등] | | | |

**갭:** [대응 VDR 콘텐츠 없는 요청 카테고리 — 추가 요청 필요]
```

### Step 2: 중요성 필터 적용

실무 CLAUDE.md / deal-context 임계점에 따라. 임계점이 "계약 N원 이상"이면 전부 검토하지 않는다.

계약은: 명시 금액(파일명·메타데이터에 있으면) 또는 상대방 중요도로 정렬. 임계점에 닿거나 카테고리가
소진될 때까지 top-down 검토.

### Step 3: 이슈 추출

각 문서를 그 카테고리의 표준 실사 관심사에 대조:

**주요 계약 — 표준 추출 세트:**
- **지배권 변경(COC) 조항**(이 거래로 트리거? 사전동의 필요?)
- **양도제한**(계약이 매수인에게 이전 가능한가? — 영업양도·합병별로 승계 효과 다름)
- 독점·경업금지(매수인 사업 제한?)
- 최혜대우(MFN — 가격 제약)
- 해지권(상대방이 거래를 이유로 이탈 가능?)
- 이상 면책·책임 노출

**회사 일반·등기 — 표준 추출 세트:**
- 주주명부·주식 현황, 미행사 신주인수권·전환사채·스톡옵션
- 거래에 필요한 **이사회·주주총회 결의**(영업양도·합병 §374/§522 특별결의 여부)
- 주주간계약(SHA) 제한(드래그·태그·우선매수권 ROFR)
- 자회사 구조·특수관계자거래·내부거래

**지식재산 — 표준 추출 세트:**
- 권리귀속·승계(창업자·직원으로부터의 양도, **직무발명 승계·보상 — 발명진흥법**)
- 제품 내 오픈소스(카피레프트 위험)
- 핵심 IP 라이선스 vs 소유
- 계류·위협 IP 분쟁(특허·상표)

**인사·노동 — 표준 추출 세트:**
- **근로관계 승계**(영업양도 시 자동승계 원칙; 합병은 포괄승계) + 거부·이의 가능성
- 지배권 변경 보상·퇴직금·미지급 임금·통상임금 리스크
- 핵심인력 retention 리스크
- 계류 노동 소송·**노동위 사건**, 근로자성 오분류(특수형태근로·프리랜서)

**소송·분쟁 — 표준 추출 세트:**
- 계류 사건·충당금, 위협받는 청구
- **규제기관 조사**(공정위·금감원·고용노동부 등)
- 패턴 소송(소비자 집단분쟁·소비자원 분쟁조정)

**인허가·규제 — 표준 추출 세트:**
- 업종 인허가 보유·유효기간, **거래로 인한 변경신고·승계 가능 여부**
- 조건부 인허가·시정명령 이력

**공정거래 — 표준 추출 세트:**
- **기업결합신고 대상 여부**(당사회사 자산·매출 규모 — `merger-control-filing`이 §11 기준 판정)
- 부당내부거래·일감몰아주기·하도급법 위반 이력

### Step 4: 각 발견 진술

> **출처 attribution.** 발견이 법령·판례·규제기관 조치를 참조하면 — COC 조항을 적용법령으로 분석,
> IP 권리귀속 갭을 특정 doctrine에 대조, 계류 소송을 사건번호로 — 인용에 출처 태그: 리서치 커넥터에서
> 가져온 인용은 `[국가법령정보]`·`[법제처]`·`[공정위]` 등; 웹 검색은 `[웹 검색 — 검증 필요]`; 훈련 데이터
> 회상은 `[모델 지식 — 검증 필요]`; VDR·딜팀 메모·외부 로펌 피드백은 `[사용자 제공]`. 문서 출처 인용
> (VDR 경로·파일명)은 native 참조 유지. `검증 필요` 태그 인용은 fabrication 위험이 높아 먼저 점검.
> 태그를 절대 strip·collapse 안 함.
>
> **사용자 인용 조문에 동의 안 할 때, 텍스트를 인용하거나 특성 짓기를 거부.** 사용자(딜팀 메모·매도측
> 공개자료)가 조문을 인용해 명제를 주장하는데 당신은 안 맞다고 보면서 국가법령정보·VDR에 그 조문 텍스트가
> 없다면, 발명하지 않는다: "그 조항은 제가 예상한 [상호속용 책임 / 승계 / 등] 요건과 안 맞습니다 —
> 실제 텍스트를 가져와야 알 수 있습니다. `[조문 미조회 — 검증 필요]`" 그 뒤 (a) 국가법령정보에서 가져와 인용,
> (b) paste 부탁, (c) 변호사 플래그. 실재 조문의 확신에 찬 잘못된 설명은 "모름"보다 나쁘다.
>
> **침묵의 보완 금지.** 리서치 쿼리(국가법령정보)가 발견에 필요한 법적 근거(COC 동의 요건, IP 귀속 doctrine,
> 근로자성 기준)에 결과를 거의/전혀 안 주면, 찾은 것을 보고하고 멈춘다. 웹·모델 지식으로 묻지 않고
> 메우지 않는다: "[도구]에서 [N]건. [규칙]에 커버 얇음. 옵션: (1) 쿼리 확장 (2) 다른 도구 (3) 웹 검색
> — `[웹 검색 — 검증 필요]` 태그, 1차 소스 대조 필요 (4) 미검증 플래그 후 멈춤. 어느 것?"

실무 CLAUDE.md 이슈 메모 포맷에 따라. 시드 메모가 이걸 썼으면:

```
이슈 #N: [제목]
카테고리: [요청목록 카테고리]
심각도: [하우스 체계 등급]
문서: [VDR 경로 + 문서명]
발견: [문서가 말하는 것과 왜 중요한지]
권고: [가격조정 / 면책 / 사전동의 필요 / 진술보장 / 이탈]
```

…그대로 사용. 시드 메모가 bullet이면 bullet.

**심각도 calibration**(하우스 체계가 🔴🟡🟢이면 — 실무 CLAUDE.md의 이중 심각도 적용):
- 🔴 **차단:** 딜 가치·구조에 영향. 주요 고객 동의 필요한 COC. 미공개 중대 소송. IP 권리귀속 갭. 기업결합신고 불승인 리스크.
- 🟡 **중간:** 주의 필요, 해결 가능. 동의 필요하나 획득 가능. 오픈소스 remediation. 근로자성 리스크.
- 🟢 **낮음:** 파일용 메모. 진술과 일치. 진술 외 조치 불필요.

### Step 5: 카테고리별 집계

요청목록 카테고리별 그룹화. 카테고리 내 심각도순 정렬.

```markdown
[문서 헤더 — 플러그인 config ## 산출물 따라, 역할별 상이; `## 누가 이 플러그인을 사용하나` 참조]

> 이 산출물은 비밀·미공개 VDR 자료에서 파생됨. 출처의 비밀유지 지위를 상속 — 비밀유지 서클 밖 배포는
> 보호를 포기시킬 수 있다. 매터의 비밀 파일과 함께 보관하고 배포는 신중히 결정. **M&A 미공개정보는
> 자본시장법 §174(미공개중요정보 이용금지)도 고려.**
```

## 핸드오프

- **ai-tool-handoff로:** 실무 CLAUDE.md에 Luminance/Kira가 있으면 bulk 계약 검토를 거기로. 이 스킬은
  미묘한 문서(side letter·변경계약·도구가 어려워하는 것) 담당.
- **deal-team-summary로:** 집계 발견이 딜팀 브리프로.
- **material-contract-schedule로:** 계약 단위 추출이 공개목록으로.
- **closing-checklist로:** 종결 전 개별 액션을 함의하는 발견은 체크리스트 항목이 된다. 제3자 동의에
  국한되지 않음 — 다음도 포함:
  - **주총·이사회 등 회사 결의** — 영업양도·합병 주총 특별결의(§374/§522), 필요한 이사회 결의, 반대주주
    주식매수청구 통지·기간(§522-3). 액션·결의요건·근거(법령/정관)·기한 제약을 특성화.
  - **규제 신고·승인** — **기업결합신고(공정거래법 §11)**, 외국인투자신고, 산업별 인허가 승인·변경신고.
  - **상대방 동의** — COC·양도금지·MFN 트리거 동의.
  - **해지·면제·변제** — COC 연동 근로 관련 처리, 변제확인서, 담보 말소.
  - **에스크로·홀드백** — 면책 에스크로·진술보장보험(W&I)·특정 이슈 연동 홀드백.
  종결 전 액션 태그가 붙은 모든 발견은 closing-checklist로 — under-handoff는 일방통행, over-handoff는 검토에서 교정.

**승계책임(한국).** 플래그: 계류·위협 불법행위·제조물책임, 환경·정화의무, **상호속용 양수인 책임
(상법 §42 — 영업양수인이 양도인 상호를 계속 사용하면 양도인 영업상 채무 변제책임)**, **근로관계 승계**,
**조세 제2차 납세의무·국세기본법상 양수인 책임**, 사해행위취소 리스크(민법 §406), 매도인 청산 계획.
자산양수도라도 **법인격 부인·실질적 동일성(영업의 실질적 동일성) 법리**로 채무가 이전될 수 있다 —
"자산만 깨끗이 산다"고 믿는 매수측을 놀라게 하는 분석. 단정 말고 `[검토]`.

## Batch 처리

큰 카테고리(계약 300건)는 batch로. 각 batch 후 진행 이슈 목록 갱신, 🔴는 즉시 플래그 — 전체 소진을 기다리지 않는다.

## 이 스킬이 하지 않는 것

- 경계 사례의 중요성 판단을 안 함. 임계점을 적용하지 경계는 사람이 결정.
- 진술보장을 협상하지 않음. 그것에 정보를 주는 발견을 산출.
- bulk AI 검토를 대체하지 않음. 고volume 조항 추출은 Luminance/Kira로 핸드오프. 이 스킬은 판단 레이어.
