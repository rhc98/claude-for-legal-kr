# Google Sheets 출력 사양

Google Workspace 팀 대상. Excel 출력과 구조는 동일하고 구현 방식이 다르다. Excel과 Sheets 경로가
모두 사용 가능하면 어느 것을 선호하는지 묻는다 — 환경으로 추측하지 않는다.

## 작성 방법

세 경로, 선호 순서:

1. **Google Sheets MCP** (`gdrive` 또는 `gsheets` MCP에 쓰기·생성 권한이 연결된 경우). 스프레드시트 생성, 시트 작성, API로 서식 설정.
2. **Google Sheets API via ADC** (사용자가 `gcloud auth application-default login --enable-gdrive-access`와 Python `google-api-python-client`를 설정한 경우). `sheets.spreadsheets().create()`와 서식용 `batchUpdate` 사용.
3. **대체: CSV + 수동 임포트.** CSV 파일을 작성하고 Sheets로 임포트하도록 안내. `format-instructions.md`도 작성해 색상 코딩과 데이터 검증을 수동으로 적용할 수 있게.

쓰기 권한을 확인하지 않고 가정하지 않는다. 먼저 확인하고 실패 시 우아하게 대체.

## 워크북 구조

Excel 사양을 그대로 반영 — 시트·의미는 동일, 구현 방식은 Sheets 네이티브:

**시트: `Review`** (메인 그리드)
- 1행: 산출물 헤더 (병합 셀)
- 2행: 열 레이블
- 3행~: 문서 1건당 1행
- A열: 문서명 / 링크 (소스 문서가 Drive에 있으면 파일로 하이퍼링크 — Excel 대비 Sheets 장점)
- B열~: 스키마 열 순서대로
- **원문 인용은 셀 노트에** (Sheets 노트, 댓글이 아님 — 노트는 영구 주석, 댓글은 협업 스레드). 노트는 hover 시 표시되고 `.xlsx` 내보내기 시 설명으로 변환.
- 상태별 셀 색상: 기본 = `answered`, 연노랑 = `unclear` 또는 `needs_review`, 연회색 = `not_present`. `batchUpdate`에서 `repeatCell`과 `userEnteredFormat.backgroundColor` 사용.
- 그룹별 `Verified` 열: 기본 공백, `setDataValidation`으로 드롭다운 `✓ | ✗ | ?`.

**시트: `Flags`**
- Excel 사양과 동일. 플래그된 셀마다 1행.

**시트: `_schema`**
- `.review-schema.yaml`의 열 정의.

**시트: `_summary`**
- 카운트, 플래그된 열, 검증 알림.

## Sheets 고유 장점 활용

- **소스 문서 하이퍼링크.** 검토 문서가 Drive에 있으면 (VDR 내보내기·내부 저장소에서 흔함) 각 행의 문서명을 파일 하이퍼링크로. 클릭-투-소스 패턴이며 Sheets에서 네이티브로 동작.
- **공유 검토.** Sheets는 동시 검토를 로컬 `.xlsx`보다 잘 처리한다. 딜팀이 검증 작업을 나누려면 이 포맷이 적합.
- **스키마 명명 범위.** 각 열에 명명 범위를 정의해 다운스트림 수식 (피벗 테이블, 조건부 카운트)이 가독성 있게.
- **상태 열 조건부 서식.** 데이터 열마다 숨긴 `_state` 열을 작성하면 조건부 서식 규칙으로 색상 코딩을 구동 — 셀별 서식보다 깔끔하고 정렬 후에도 유지.

## Sheets 고유 주의 사항

- **노트는 셀별이며 인쇄 시 보이지 않는다.** 파트너 회의용 인쇄나 PDF가 필요하면 `Flags` 시트에도 인용을 기재해 인쇄본에서도 살아있게.
- **Sheets는 셀 1,000만 개 한도.** 법률 검토에서는 넘지 않지만, 문서 5만 건 × 30개 열 + 소스 열 그리드를 시도한다면 경고.
- **공유 기본값.** 이 플러그인 실무 프로파일에 따라 이것은 변호사 작업물이다. 스프레드시트를 제한된 공유(소유자만)로 생성하고 사용자에게 의도적으로 공유하라고 알린다. "링크가 있는 모든 사람"을 기본으로 하지 않는다.
- **수식 이스케이프.** 원문 인용이 `=`, `+`, `-`, `@`로 시작하면 단일 따옴표(`'`)를 접두해 Sheets가 수식으로 파싱하지 않게. 실제 발생하는 오류: "- 양 당사자는 합의한다..."로 시작하는 계약 조항이 이스케이프 없이는 수식 오류로 렌더된다.

## 수식 주입 방어

Excel·Sheets·CSV 출력에 셀을 쓰기 전, 수식 주입을 무력화한다. 상대방 출처 텍스트(계약 인용,
당사자명, 등기 데이터, CLM 추출값)는 공격자 통제 입력이다. `=`, `+`, `-`, `@`, 탭(`\t`), 캐리지리턴(`\r`)으로
시작하는 셀은 수식 또는 행 구조 파괴로 해석된다.

- **단일 따옴표 접두:** `'=SUM(A1:A10)` → `=SUM(A1:A10)` (텍스트로 표시, 실행 안 됨)
- **문서 텍스트, 도구 결과, 사용자 붙여넣기에서 온 모든 셀에 적용.** 직접 작성한 열 헤더와 계산된 값은 안전.
- **CSV: 삽입된 쉼표, 큰따옴표, 줄바꿈도 이스케이프** (RFC 4180 인용 방식).
- 이것은 선택이 아니다. 사용자가 Sheets에서 여는 스프레드시트가 매크로를 실행하거나 데이터를 유출하면 공급망 공격이 된다.

**이 세션 외부에서 온 모든 값** — 상대방 문서 텍스트, 회사명, VDR 추출 문자열, 외부 제공값 — **은 이스케이프 대상이다.** 직접 제어하는 열 헤더와 스킬이 계산한 값은 안전하다. 대시보드·HTML 렌더에서는 `textContent`만 사용 — `innerHTML` 금지. URL은 emit 전 scheme 검사 (`http:` / `https:` / `mailto:`만 허용).

## 하지 말 것

- 쓰기 권한 없이 스프레드시트 수정을 시도하지 않는다. 먼저 확인.
- 노트와 댓글을 혼동하지 않는다. 노트가 인용을 담는다.
- 서식 없이 값만 쓰지 않는다 — 상태 색상이 검증 작업을 우선순위화한다.
- `Verified` 열을 미리 채우지 않는다.
