# /customize

## 실행 조건

사용자가 `/ip-legal:customize` 입력. 콜드스타트 인터뷰 전체 재실행 없이, 파일 수동 편집 없이 IP 실무
프로파일의 무엇 — 집행 입장·에스컬레이션 담당자·권리 영역 mix·갱신 리드타임·브랜드 보호·산출물
포맷 — 을 변경하고 싶음.

## 할 일

1. **config 읽기.** `~/.claude/plugins/config/claude-for-legal-kr/ip-legal/CLAUDE.md`
   (와 한 단계 위 `~/.claude/plugins/config/claude-for-legal-kr/company-profile.md`) 읽기.
   플러그인 config 없거나 `[PLACEHOLDER]` 값 남아 있으면:

   > 셋업을 아직 안 하셨습니다. 먼저 `/ip-legal:cold-start-interview` 실행 —
   > customize는 이미 가진 프로파일을 조정하기 위한 것.

2. **맞춤화 가능 맵 보여주기.** 프로파일에 있는 것을 그룹화, 현재 값의 한 줄 요약과 함께:

   - **회사 / 우리는 누구인가** — 회사명·업종·관할·법인 형태·실무 환경
     *(모든 claude-for-legal-kr 플러그인 공유 — `company-profile.md` 통해 변경 전파)*
   - **권리 영역 mix** — 상표·특허·디자인·저작권·영업비밀의 비중. 어느 영역이 활성인지가 어느 스킬
     체크리스트가 켜지는지 바꿈.
   - **권리자/이용자 입장** — 권리자(보호·집행) / 실시권자·이용자(리스크 회피) / 양쪽. 같은 사실도
     이 축에 따라 분석 방향(침해 집행 vs FTO·클리어런스)이 달라짐.
   - **등록 footprint** — 특허청(KIPO) 중심 + PCT·마드리드·헤이그 국제출원, 외국 권리 현지 대리인 경유.
   - **집행 입장** — 적극·신중·보수, 경고장(내용증명)·소프트레터·제소·심판청구 트리거와 각각이
     `[검토]` 바·승인 체인에 의미하는 것
   - **인력** — 권리 영역별 사내 담당·외부 대리인(출원·심판 변리사 / 소송·집행 변호사)·에스컬레이션 체인
   - **포트폴리오 등록부** — `portfolio.yaml` 경로·최근 감사·갱신 알림 리드타임(만료 N개월·M개월·1개월 전)
   - **브랜드 보호** — 감시 상표·감시 관할·모니터링 서비스·주기
   - **직무발명(발명진흥법)** — 보상규정·예약승계 약정·보상 체계
   - **영업비밀(부정경쟁방지법)** — 비밀관리 체계·KIPI 원본증명 이용 여부
   - **IP 관리 시스템** — Anaqua·CPA Global·PatSnap·특허법인 시스템·스프레드시트·없음
   - **워크플로우** — 매터 워크스페이스(권리·분쟁·거래·클리어런스 건)·산출물 폴더 위치
   - **통합** — 국가법령정보 MCP 상태·폴백 · KIPRIS(MCP 없음, 수동) · IP 관리 시스템 · 문서 저장소 · Slack

3. **무엇을 바꿀지 묻기.**

   > 무엇을 조정할까요? 섹션을 고르거나 자기 말로 변경 설명.

4. **변경 수집.** 사용자가 섹션을 고르면 그 섹션의 현재 값을 보여주고 새 값 묻기. 사용자가 자유
   형식으로 설명하면 어느 섹션이 영향인지 식별하고 mapping 확인.

5. **변경 전 검증.** 변경에 법적 인용·임계점·기한·존속기간·수수료가 있으면 국가법령정보 대조 검증(국가법령정보 연결 시):
   ```
   mcp__korean-law__get_law_text "[조항]"
   mcp__korean-law__legal_analysis {"mode": "verify_citations", "citation": "[조항]"}
   ```
   인용 예시는 검증 태그를 함께 단다 — 기본 `[검증 필요]`, 핀포인트 `[검증 필요-핀포인트]`, 모델 지식
   `[모델 지식 — 검증 필요]`. 인용이 안 맞으면 사용자에게 확인 ("당신은 X라 하셨는데, 현행 텍스트는 Y.
   어느 것으로 갈까요?"). **상표·특허·디자인 등록현황·심결은 국가법령정보 미커버** — 사용자에게 KIPRIS
   검색결과·심결문을 요청하고 `[KIPRIS — 직접 검증 필요]` / `[심판원 심결 — 직접 검증 필요]` 태그.

6. **변경 적용.** CLAUDE.md의 해당 섹션을 갱신. 변경 전후 diff 보여주기.

   예시:
   - *갱신 리드타임 만료 6개월 전 → 3개월 전:* "이제 포트폴리오·갱신 트래커가 3개월 전을 알림
     리드타임으로 다룹니다. **상표 갱신은 존속기간 만료 전 1년 이내 신청** 등 기간 제한이 있으니 — 현행
     상표법·시행규칙·특허청 고시 대조 필요 `[검증 필요]`. 기존 항목은 그대로, 소급 원하면 재실행."
   - *집행 입장 신중 → 적극:* "경고장(내용증명) 발송 트리거를 낮추고 `[검토]` 바를 조정합니다. **근거
     없는 경고는 부당경고로 손해배상·업무방해 역공의 빌미가 될 수 있으니** 발송 전 권리 유효성·침해
     성립 점검과 승인 매트릭스를 유지합니다."
   - *권리 영역에 영업비밀 추가:* "영업비밀 관련 스킬 실행 시 비밀관리 체계·KIPI 원본증명을 묻겠습니다.
     비밀관리성이 권리 존속의 요건이므로 산출물 목적지 점검을 가장 엄격히 적용합니다."
   - *감시 상표 추가:* "브랜드 보호 워치리스트에 추가합니다. 모니터링 주기·관할도 확인할까요?"

7. **공유 프로파일 변경**(회사명·업종·관할·법인 형태·실무 환경): `company-profile.md`에 쓰고 안내:

   > 이 변경은 모든 claude-for-legal-kr 플러그인에 영향 — 관할·회사 정보를 읽는 모든 플러그인이 이제
   > [새 값]을 봅니다.

8. **확인 + 검증 로그 기록.**
   ```
   [YYYY-MM-DD] customize - [성명]이(가) [섹션]에서 [구체적 변경]
   ```
   `~/.claude/plugins/config/claude-for-legal-kr/ip-legal/verification-log.md`에 추가.

## 가드레일

- **섹션을 삭제하지 않는다.** 사용자가 "제거"를 원하면 `[미구성]`으로 설정하고 그것이 플러그인 동작에
  무엇을 의미하는지 설명. *(권리 영역 끄기는 예외 — 비활성 영역 섹션이 얇아지는 것은 정상.)*
- **내부 모순 플래그.** 변경이 프로파일을 모순되게 만들면(영업비밀 영역 off + 에스컬레이션에 "영업비밀
  유출 사고"; 보수 집행 입장 + 즉시 경고장 트리거; 이용자 입장 + 적극 침해 집행), 긴장을 플래그하고
  어느 것을 원하는지 묻기.
- **가드레일 약화 플래그.** 사용자가 가드레일을 끄려 하면(`[검토]` 플래그 제거, 문서 헤더 생략,
  `[검증 필요]`·`[KIPRIS — 직접 검증 필요]` 태그 제거), 그 가드레일이 무엇을 막는지 설명하고 trade-off를
  이해하는지 확인. `[검토]` 플래그·출처 attribution 태그·인용 조문의 검증 태그·KIPRIS/심결 데이터 갭
  태그는 load-bearing이며 제거하지 않는다.
- **한 번에 하나.** 전체 인터뷰를 다시 묻지 않는다.
