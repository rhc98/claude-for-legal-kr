# /cold-start-interview

1. `~/.claude/plugins/config/claude-for-legal-kr/ip-legal/CLAUDE.md` 확인.
   `--check-integrations`이면 인터뷰 skip — Part 0 `무엇이 연결되어 있나` 점검만 재실행하고
   `## 연결 가능한 통합` 표만 갱신. 점검 시: MCP 도구 호출이 실제로 성공한 경우에만 ✓. 구성만 되고
   미테스트는 ⚪ + 확인 방법 한 줄. `.mcp.json` 선언만으로 ✓ 보고 안 함 — trust를 깬다. **KIPRIS는
   MCP가 없다 — N/A로 표기하고 사용자가 검색결과를 직접 제공함을 명시.**
2. 아래 인터뷰 실행(Part 0 먼저 — 역할 + 통합 — 그 뒤 실무 specifics).
3. 시드 문서: 경고장(내용증명) 선례, 표준 라이선스·양도 계약, 직무발명 보상규정, 포트폴리오 대장.
4. 추출: 권리 영역 mix, 권리자/이용자 입장, 집행 트리거·승인 체인, 경고장·라이선스 하우스 포맷, 포트폴리오 등록부 구조.
5. 마이그레이션: populate된 CLAUDE.md(placeholder 없음)가 옛 캐시 경로
   `~/.claude/plugins/cache/claude-for-legal-kr/ip-legal/*/CLAUDE.md`에는 있는데 config 경로에 없으면, config 경로로 복사하고 무엇이 마이그레이션됐는지 알림.
6. `~/.claude/plugins/config/claude-for-legal-kr/ip-legal/CLAUDE.md` 작성(상위 디렉토리 생성).

## 목적

IP 실무는 권리 영역과 입장에 따라 모습이 크게 다르다. 어떤 팀은 상표 클리어런스·브랜드 보호가 거의
전부고, 어떤 팀은 특허 출원·FTO 중심, 어떤 팀은 라이선스 거래·OSS 컴플라이언스가 핵심이다. 같은 사실도
**권리자 입장(침해 집행·경고장)**이냐 **이용자 입장(FTO·클리어런스·리스크 회피)**이냐에 따라 분석 방향이
갈린다. 이 인터뷰는 권리 영역 mix·입장·등록 footprint·집행 자세를 학습해 **그들의 실무대로** 동작하는
실무 프로파일을 짓는다.

**ip-legal은 모듈형이 아니다.** corporate-legal과 달리 단일 패스 인터뷰로 `## IP 실무 프로파일`의 모든
섹션을 한 번에 채운다(권리 영역 mix·등록 footprint·IP 관리 시스템·권리 영역별 담당·포트폴리오 등록부·
브랜드 보호·집행 입장·직무발명·영업비밀·시드 문서) + Part 0(역할 + 통합).

## 콜드스타트 점검

`~/.claude/plugins/config/claude-for-legal-kr/ip-legal/CLAUDE.md` 읽기:
- **존재하지 않음** → 인터뷰 시작.
- **`<!-- SETUP PAUSED AT: -->` 포함** → 맞이하고 해당 섹션부터 재개 제안.
- **`[PLACEHOLDER]`는 있는데 pause comment는 없음** → 템플릿 미완료; 새로 시작 또는 placeholder부터 재개 제안.
- **populate됨** → 이미 구성됨; `--redo`가 아니면 skip.

템플릿 구조는 `${CLAUDE_PLUGIN_ROOT}/CLAUDE.md` — 섹션 골격으로 사용. 완성된 프로파일을 config 경로에 쓰고 상위 디렉토리 생성.

옛 캐시 경로에 CLAUDE.md가 있고 config 경로에 없으면 앞으로 복사.

- `--redo` — 전체 재인터뷰, 모든 섹션 덮어씀
- `--check-integrations` — 인터뷰 skip, 통합 표만 재점검·갱신

## 공유 회사 프로필 확인

`~/.claude/plugins/config/claude-for-legal-kr/company-profile.md` 확인.

- **있으면:** 읽기. 한 줄 확인: "당신은 [성명], [실무 환경], [회사], [업종], [관할]에서 운영. 맞나요?
  (또는 '업데이트'를 말해 공유 프로필 변경.)" 확인되면 회사 질문 skip — 바로 플러그인 특화 질문으로.
- **없으면:** 당신이 첫 플러그인. 오리엔테이션·fork 후, 회사 질문을 묻고 공유 프로필에 쓴다(플러그인
  루트 `references/company-profile-template.md` 따름). 그 후 특화 질문. 사용자에게: "회사 프로필 저장
  완료 — 다른 claude-for-legal-kr 플러그인들이 이걸 읽고 이 질문들을 skip합니다."

공유 프로필에 속하는 회사 질문(있으면 재질문 금지): 실무 환경, 회사명, 법인 형태, 업종, 무엇을 파는지,
규모, 관할, 규제 당국, 위험 성향, 에스컬레이션 담당자. 플러그인 특화 질문(권리 영역 mix·집행 입장·갱신
리드타임·경고장 포맷·직무발명·영업비밀 등)은 플러그인별로 유지.

> 설치하세요 — QUICKSTART 참조. 프로젝트 스코프로 계속해도 되지만 파일을 이 폴더로 이동해야 합니다.**

진행 전 사용자 확인. 작업 디렉토리가 홈이면 침묵으로 skip.

>

## 사용자가 빠른/풀을 선택한 후

선택되면, 첫 질문 전 오리엔테이션:

> "이 플러그인은 실무 프로파일(권리 영역 mix·집행 입장·갱신 리드타임), 포트폴리오 등록부, 매터별
> 폴더(클리어런스·FTO·침해 분석·라이선스)를 유지합니다. 상표 클리어런스·특허 FTO·침해 트리아지·경고장·
> 라이선스 검토·OSS 컴플라이언스를 당신 하우스 포맷으로 지원합니다. 이 셋업 인터뷰는 어느 권리 영역이
> live인지, 어떻게 운영하는지 학습해 스킬이 매번 읽을 평문 파일에 씁니다. 답한 모든 것은 나중에 바꿀 수
> 있습니다. 끝나면 일반 템플릿이 아닌 당신이 일하는 방식대로 동작합니다."
>
> 그 뒤: "준비됐나요? 빠른 질문 몇 개 먼저, 그 뒤 실무 specifics를 더 깊이."

**왜 중요한가.** 이 플러그인의 모든 커맨드는 이 인터뷰가 쓴 구성을 읽습니다. 일반 구성은 일반 출력 —
디폴트 집행 임계점, 디폴트 경고장 포맷, 디폴트 클리어런스 기준. 실제 권리 영역·입장·집행 방식을 말하는
것이 "IP AI 도구"와 "당신 팀이 일하는 방식대로 동작하는 도구"의 차이를 만듭니다. 답이 구체적일수록(실제
집행 트리거, 실제 경고장 문언, 실제 하우스 포맷) 출력이 당신 책상에서 나온 것처럼 느껴집니다.

**fresh 전문가 프로파일.** 셋업은 사용자의 타이핑 답변과 명시적으로 공유한 문서에서만 프로파일을 만든다.
개인 Claude 기록·다른 대화·홈 디렉토리 CLAUDE.md를 읽지 않는다. 대화 컨텍스트에서 관련 정보가 보이면
사용 전 묻는다 — 사용자가 타이핑·승인하지 않은 어떤 개인적인 것도 IP 프로파일에 넣지 않는다.

## 인터뷰 페이싱

- **답이 어딘가 적혀 있을 것이라고 가정.** 회사 설명·권한위임규정·에스컬레이션 매트릭스·경고장 선례·
  표준 라이선스·직무발명 보상규정·포트폴리오 대장처럼 적혀 있을 가능성이 높은 정보는, 메모리에서 타이핑
  하라 하기 전에 link 또는 paste 요청. "링크 또는 doc를 paste 하거나 짧은 버전을 주세요"가 한 문장 이상
  답이 필요한 모든 것의 디폴트 ask.
- **batch 크기 — 부분 질문 카운트.** 한 turn에 답 가능한 prompt 2-3개. 5개 부분 질문은 5개 질문. 스크롤
  없이 답할 수 있어야 함. 가능하면 구조화된 tap-through 선호.

**진짜 답에 멈춤.** 어떤 질문은 빠른 tap(권리 영역, 시장, 입장). 어떤 질문은 타이핑·업로드 필요(경고장
선례, 표준 라이선스, 보상규정, 포트폴리오 대장):

- **묻고 기다림.** "이건 타이핑 답이 필요합니다 — 기다리겠습니다." 답 전 다음 질문 안 감.
- **업로드(경고장·라이선스·보상규정·대장):** "내용을 paste하거나, 파일 경로/URL 공유, 또는 '나중에 skip'.
  skip하면 실무 프로파일에 갭 플래그가 붙습니다." 그 뒤 실제로 기다림. 시드 문서는 포맷 추출을 좌우 —
  침묵으로 skip하면 모든 미래 출력이 하우스 포맷 아닌 일반 템플릿이 됨.
- **실무 프로파일 쓰기 전:** 인터뷰 검토, skip·placeholder 답 목록화(특히 시드 문서). "쓰기 전, 아직 열린
  것: [목록]. 지금 채울까요, placeholder로 둘까요?" 그 뒤 기다림.
- **침묵의 갭이 있는 프로파일을 절대 쓰지 않음.** 모든 placeholder는 사용자가 의도적으로 skip한 결정.
- **일시정지·재개.** 미리 말함: "멈춰야 하면 '잠시 멈춤'. 진행 저장합니다. 나중에 다시 실행하면 마지막
  지점에서 재개." 일시정지 시 부분 구성을 config에 쓰고 상단 `<!-- SETUP PAUSED AT: [섹션] -->` +
  답 없는 필드 `[PENDING]`(placeholder와 구분). 이미 답한 질문 재질문 안 함.

**셋업 중 사용자가 주장한 법적 사실 검증.** 사용자가 인터뷰에 특정 법조·존속기간·갱신기한·수수료·등록
번호로 답하고 sanity-check 가능하면, 구성에 쓰기 전 점검. 충돌하면 surface: "갱신기한을 X라 하셨는데 제
이해는 Y입니다 — 프로파일에 어느 것이 들어갈지 확인해주실 수 있나요? `[전제 플래그 — 검증 필요]`"
CLAUDE.md에 쓰인 잘못된 사실은 모든 미래 출력으로 전파된다. **법망 연결 시 적극 활용** — 예: 사용자가
"상표 갱신은 만료 후 아무 때나"라 하면 `mcp__beopmang__law get "상표법 제84조"` 호출로 갱신등록신청
기간(존속기간 만료 전 1년 이내 + 만료 후 추가기간)을 가져와 보완하되 인용에 `[검증 필요-핀포인트]`.
**상표·특허·디자인 등록현황·심결은 법망 미커버** — 사용자에게 KIPRIS 검색결과·심결문을 요청하고
`[KIPRIS — 직접 검증 필요]` / `[심판원 심결 — 직접 검증 필요]`.

### Part 0: 누가 쓰고, 무엇이 연결되어 있나

IP specifics 전 빠른 질문 셋. 플러그인이 어떻게 일하는지에 영향.

#### 누가 쓰고 있나?

> 이 플러그인을 일상에서 누가 쓸 건가요? (이건 모든 메모·분석·검토의 work-product 헤더를 결정 —
> 변호사 출력은 비밀유지 표지, 변리사·비자격자 출력은 "리서치 노트, 변호사·변리사 검토" 헤더.)
>
> 1. **변호사 또는 변리사** — 변호사(침해소송·계약·형사), 변리사(출원·심판), 변호사·변리사 감독 하 사내 실무자, 외부 특허법인·로펌 어소시에이트.
> 2. **자격 없는 실무자 (변호사·변리사 상담 가능)** — 사업개발·IP 지원·발명자, 사내·외부 변호사·변리사 상담 가능.
> 3. **자격 없는 실무자 (정기적 상담 어려움)** — 본인이 직접 처리.

> **한국 특유 — 변리사·변호사 이원화.** 특허·실용신안·상표·디자인의 **출원·심판 대리는 변리사**(변리사법)
> 영역, **침해소송·라이선스 계약·형사는 변호사** 영역, 일부 중첩됩니다. 어느 자격자인지에 따라 산출물
> framing과 에스컬레이션 라우팅이 달라집니다 — 출원·심판 전략은 변리사, 침해소송·계약·형사 고소는 변호사.

답이 2 또는 3이면 한 번만:

> 모든 기능을 쓸 수 있습니다 — 리서치·검토·작성·추적. 두 가지가 제가 일하는 방식에서 달라집니다:
>
> 1. **출력을 결재가 아닌 변호사·변리사 검토용 리서치로 framing합니다.** "발송 OK" 대신 "발견한 것과 발송 전 물을 질문".
> 2. **법적 결과가 있는 단계 전에 멈춥니다** — 경고장·내용증명 대외 발송, 출원·심판 청구, 라이선스 계약
>    서명, 형사 고소, 권리 포기·기한 도과 결정. 검토를 거쳤는지 묻고 짧은 brief를 만들어드립니다.
>
> disclaimer가 아닙니다. 적시의 전문가 시간은 보통 실수보다 쌉니다.

답이 3이면 추가:

> 자격 있는 변호사·변리사를 찾아야 하면: 대한변호사협회 변호사검색(`https://www.koreanbar.or.kr`),
> 대한변리사회(`https://www.kpaa.or.kr`)가 빠른 출발점. 많은 사무소가 무료·저비용 초기 상담 제공.
> 영세사업자·발명자는 특허청 공익변리사 특허상담센터·지역지식재산센터, 한국지식재산보호원 분쟁 지원이 출발점.

#### 실무 환경

각 섹션의 에스컬레이션 질문이 올바로 분기하도록 일찍 한 번 묻는다:

> 다음 중 어느 것이 당신 실무에 가장 가깝나요? (이건 모든 스킬의 에스컬레이션 framing에 영향 — 사내는
> IP 책임자·GC routing, 단독·소규모는 "외부 변호사·변리사 상담", 특허법인은 파트너 routing.)
>
> - **단독 / 소규모 사무소 (계층 없음)** — 결재 체인 질문 skip, 동료·외부 자격자를 언제 부르는지 물음.
> - **특허법인 / 법무법인(IP)** — 결재 체인, 시간당 비용 임계점, 누가 위에서 사인하는지 물음. **매터 워크스페이스 활성화 고려.**
> - **사내 (인하우스 IP·법무)** — 에스컬레이션 매트릭스, IP 책임자/GC가 누구, 무엇이 사업부·경영진으로 가는지 물음.
> - **공공 / 지식재산 지원 / 클리닉** — 감독 구조·실무 제약 물음.
> - **위 중 어느 것도 아님** — 말씀해주세요. 적응하겠습니다.

답을 `## 우리는 누구인가`의 `**실무 환경:**`에 기록. 단독·소규모는 에스컬레이션을 "외부 변호사·변리사
상담 시점"으로 reshape. **특허법인·로펌·단독·소규모면 `## 매터 워크스페이스`의 `활성화`를 ✓로 설정 제안.**

#### 무엇이 연결되어 있나?

> 이 플러그인은 다음과 작업할 수 있습니다: **법망 MCP**(한국 법령·판례), IP 관리 시스템(Anaqua·CPA
> Global·PatSnap·특허법인 시스템), 문서 저장소(Google Drive·SharePoint·Box), Slack. 어느 커넥터가
> 구성되어 있는지 점검 — 필요한 기능은 동작하고 없는 기능은 침묵으로 실패하지 않고 수동으로 graceful하게
> fallback합니다. **특허청 KIPRIS(상표·특허·디자인 검색·등록원부)는 MCP가 없어 검색결과를 직접 제공하시게 됩니다.**

**구성된 것이 아닌 실제로 연결된 것을 점검.** `.mcp.json`에 나열 = *가용*. 실제 응답 = *연결됨*. 둘은
다르며 혼동하면 trust가 깨진다. 각 커넥터에 대해:
- 테스트 가능하면(간단한 MCP 호출) 성공 응답 시에만 ✓.
- 테스트 불가면 ⚪ "구성됨이나 검증 안 됨 — MCP 설정에서 확인" + 한 줄 how-to.
- 구성만으로 절대 ✓ 보고 안 함.

**법망 점검 구체:**
```
mcp__beopmang__law search "특허법"
```
응답에 결과 1건 이상이면 ✓. 연결 오류·timeout·빈 배열이면 ✗.

**KIPRIS는 MCP가 없다.** 통합 표에 `[N/A — MCP 없음]`으로 표기하고 "상표·특허·디자인 검색·등록원부는
법망 미커버. 사용자가 KIPRIS(kipris.or.kr) 검색결과를 직접 제공"이라고 폴백 명시. 절대 ✓·✗로 점검하지 않음.

연결 안 된 커넥터에 연결 방법 알림. 예: "법망이 연결되지 않았습니다. Claude Code:
`claude mcp add beopmang https://api.beopmang.org/mcp --transport http`. 이 플러그인은 법망 없이도 동작하지만 —
특허법·상표법·디자인보호법·저작권법·부정경쟁방지법·발명진흥법 인용 검증이 모두 `[모델 지식 — 검증 필요]`로 떨어집니다."

발견을 다음 형식으로 보고:

> - ✓ [통합] — 연결됨 (테스트됨)
> - ⚪ [통합] — 구성됨이나 검증 안 됨. MCP 설정에서 확인.
> - ✗ [통합] — 발견 안 됨. [기능]은 [수동 대안]으로 fallback. [연결 방법]. 나중에 셋업하면 `--check-integrations` 재실행.
> - N/A KIPRIS — MCP 없음. 상표·특허·디자인 검색결과·등록원부는 직접 제공.
>
> 전부 필요하진 않습니다. 핵심 기능은 파일 접근만으로 동작. **단, 법망 MCP가 없으면 인용 검증이 동작하지
> 않고, KIPRIS는 어차피 수동입니다 — 클리어런스·FTO·포트폴리오 작업 시 등록현황·심결은 직접 제공하셔야 합니다.**

### Part 7: 직무발명·영업비밀 (2-3분, 해당 시)

**직무발명 (발명진흥법):**
- 직무발명 보상규정이 있나요? 문서 경로? (없으면 갭 플래그.)
- 예약승계 약정 — 근로계약·취업규칙·별도 약정 / 미정비?
- 보상 체계 — 출원·등록·실시·처분 보상 / 협의 절차?

> **한국 특유.** 종업원의 직무발명은 발명진흥법상 종업원에게 원시귀속되고, 사용자는 통상실시권을 가지며
> **예약승계 + 정당한 보상**으로 권리를 승계합니다(§10·§15 `[검증 필요-핀포인트]`). 보상규정 미비·보상
> 부적정은 분쟁·소송의 핵심 원인 — 권리 귀속·승계를 단정 말고 `[검토]`.

> **보상규정을 안 올렸으면:** 규정을 paste하면 보상 체계·승계 약정을 추출합니다. 시드로 매우 가치 있음.

**영업비밀 (부정경쟁방지법):**
- 비밀관리 체계 — 비밀관리규정·접근통제·표시·서약서 / 미정비?
- KIPI 영업비밀 원본증명제도를 이용하나요?

> **비밀관리성이 권리의 요건.** 영업비밀 보호는 비밀로서 관리되고 있어야(비밀관리성) 인정됩니다 — 관리
> 부재는 권리 상실. 산출물 목적지 점검을 가장 엄격히 적용하도록 프로파일에 새깁니다.

`### 직무발명`·`### 영업비밀`에 기록.

## 실패 모드

- **모든 권리 영역이 활성이라 가정 안 함.** 먼저 묻고 live한 것만 깊이 인터뷰. 상표 클리어런스만 하는 팀에 OSS·직무발명 깊은 셋업 불필요.
- **권리자 입장을 하드코딩 안 함.** 입장(권리자/이용자/양쪽)을 명시적으로 묻고 기록. 같은 사실의 분석 방향이 여기서 갈린다.
- **일반 placeholder를 쓰지 않음.** 답이 모호하면("표준 집행") 무엇인지 물음. 트리거가 실제 트리거일 때만 유용.
- **KIPRIS를 법망처럼 다루지 않음.** 상표·특허·디자인 등록현황·심결은 법망 미커버 — 통합 표 N/A, 인용 시 `[KIPRIS — 직접 검증 필요]` / `[심판원 심결 — 직접 검증 필요]`.
- **법망 DOWN 인식.** 조문 인용은 호출 패턴(`mcp__beopmang__law get`·`mcp__beopmang__tools verify`)을 보이되 예시 인용은 검증 태그(`[검증 필요]` / `[검증 필요-핀포인트]` / `[모델 지식 — 검증 필요]`) 부착. 실제로 가져온 게 아니면 `[법망]` 태그 금지.
- **변리사·변호사 이원화 인식.** 역할이 변리사면 출원·심판 framing, 변호사면 침해소송·계약·형사 framing. 산출물이 어느 자격자 검토를 요하는지 플래그.
- **직무발명·갱신기한·존속기간을 단정 금지.** 권리 귀속·승계는 `[검토]`, 기한·수수료는 법망·특허청 고시 대조.
- **외국 출원·외국 준거법을 한국 프레임워크로 단정 안 함.** `[외국법 — 외국 변호사·대리인 검증 필요]` 플래그.
