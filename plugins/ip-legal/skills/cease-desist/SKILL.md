# /cease-desist

두 모드. 하나를 고른다:

- `/ip-legal:cease-desist --send` — 집행 입장에 맞춘 경고장·내용증명 초안. 발송 전 필수 게이트
  (권리 유효성 + 침해 성립 + **부당경고 역공 위험** + 승인 매트릭스).
- `/ip-legal:cease-desist --receive` — 받은 경고장을 옵션 메모(A–F)로 트리아지하고 권고.

플래그가 없으면 한 번 묻는다: "경고장을 *보내는* 건가요, 받은 경고장을 *트리아지*하는 건가요?"

## 지침

1. **`~/.claude/plugins/config/claude-for-legal-kr/ip-legal/CLAUDE.md` 로드.** `[PLACEHOLDER]`가
   남아 있으면 멈추고 안내: "이 플러그인은 셋업 전에는 유용한 출력을 줄 수 없습니다.
   `/ip-legal:cold-start-interview`를 실행하세요 — 이 스킬은 집행 입장·승인 매트릭스·실무
   프로파일에 의존합니다."

2. **매터 워크스페이스 확인.** `## 매터 워크스페이스`별: `활성화`가 `✗`면 skip. 활성이고 활성 매터
   없으면: "어느 매터입니까? `/ip-legal:matter-workspace switch <slug>` 실행하거나 `practice-level`."

3. **`$ARGUMENTS`로 디스패치:**
   - `--send` → 발송 모드. 다섯 핵심 사실 수집 → 상대방 실사 → 권리 유효성·침해 성립 점검 → 초안 →
     발송 전 게이트 → 출력.
   - `--receive` → 수신 모드. 핵심 사실 추출 → 주장 평가 → 우리 노출 평가 → 옵션 A–F 제시 → 권고 →
     트리아지 메모.
   - 플래그 없음 → 한 번 묻는다(위).

4. **게이트를 존중한다.** `--send`에서 발송 전 게이트는 최종 출력 작성 전 반드시 돈다. **근거 없는
   경고는 부정경쟁·업무방해·손해배상(부당경고) 역공의 빌미**가 되므로 게이트는 건너뛸 수 없다.

5. **관할 노트.** 침해·권리는 속지주의다. 권리가 외국에 등록되어 있거나 침해·상대방이 한국 밖이면
   초안 전 플래그한다 — 한국 경고장이 아니라(또는 함께) 외국 권리·외국 절차가 필요할 수 있다.
   `[외국법 — 외국 변호사·대리인 검증 필요]`

6. **자격 split.** 경고장·내용증명 발송, 침해소송·형사고소는 **변호사 영역**이다(변리사 영역인 출원·
   심판과 구별). 자격 없는 실무자에게는 게이트 통과 전 변호사 상담용 1쪽 brief를 준다.

## 예시

```
/ip-legal:cease-desist --send
/ip-legal:cease-desist --receive ~/Downloads/received-letter.pdf
/ip-legal:cease-desist
```

## 목적

경고장·내용증명은 빠르고 싸고 — 그만큼 위험하다. 한국에서 **근거 없는 경고장·영업방해성 통지는
부정경쟁(부정경쟁방지법상 신용훼손·영업비방), 업무방해(형법), 손해배상(민법) 역공**의 빌미가 될 수
있다(이른바 부당경고). 권리가 무효이거나 침해가 성립하지 않는데 경고장을 보내면, 상대방·상대방의
거래처에 보낸 통지가 거꾸로 우리 책임이 된다. 그래서 이 스킬은 발송 전 권리 유효성·침해 성립·승인
매트릭스를 강제 점검한다.

수신 모드는 받은 경고장을, 누가 시간을 낭비하기 전에, 옵션 메모로 분류한다.

> **외부 송부용 산출물(발송 모드).** 경고장·내용증명은 상대방에게 가는 문서다. 발송 문서 자체에는
> 내부 work-product 비밀유지 헤더를 **붙이지 않는다**(CLAUDE.md `## 산출물` 외부 송부용 규칙). 내부
> 초안·침해 분석·실사 메모·발송 전 brief는 헤더를 유지한다.

## 관할 가정

IP는 속지주의 — 권리는 등록·발생한 관할에서만 효력을 가진다. 한국 권리에 대한 한국 내 침해가 디폴트다.
권리가 외국에 등록되어 있거나, 침해 발생지·상대방이 한국 밖이면 초안 전 플래그한다 — 한국 경고장이
잘못된 도구이거나, 외국 권리·외국 절차의 통지와 페어링되어야 할 수 있다. `[외국법 — 외국 변호사·대리인
검증 필요]`

  유형별)**, 경고장 법적 위험 박스), `## 산출물`(헤더·역할), `## 누가 이 플러그인을 사용하나`(역할 —
  변호사 vs 자격 없는 실무자, 상담 가능한 변호사).
- 법망 — 권리 유효성·침해 성립 근거 조문. 인용마다:

> `[심판원 심결 — 직접 검증 필요]`. 저작권 등록은 한국저작권위원회 — `[저작권위 등록 — 직접 검증 필요]`.

## 발송 모드 — 경고장·내용증명 작성

### Step 1: 다섯 핵심 사실 수집

> 경고장을 작성하기 전 다섯 가지를 확정합니다:
>
> 1. **어느 IP 권리인가?** — 상표(등록번호·관할)·특허(등록번호)·디자인·저작권(등록 또는 무등록)·
>    영업비밀. 등록권리면 등록번호·존속 상태, 무등록이면 권리 발생 근거(저명상표·미등록 주지표지 등).
> 2. **구체적 침해 행위** — 누가, 무엇을, 어디서, 언제, 증거와 함께?
> 3. **상대방과의 관계** — 경쟁자 / 거래처·대리점 / 무관한 제3자?
> 4. **실제로 구하는 구제** — 사용 중지 / 폐기 / 손해배상 / 이전(도메인·계정) / 사과·정정?
> 5. **이 건에 한정한 입장 override**(있으면)?

### Step 2: 상대방 실사 (비선택 전제조건)

발송 전 실사는 선택이 아니다 — 부당경고 역공 위험을 평가하려면 상대방을 알아야 한다:

- 법인 명칭·설립·별칭
- 규모·자원대(임직원·매출·투자)
- IP 포트폴리오 점검(KIPRIS — 상대방이 무효심판·권리범위확인심판을 걸 역량·동기) `[KIPRIS — 직접 검증 필요]`
- 분쟁 이력(과거 소송·심판·경고장 대응 이력)
- 알려진 대리인(특허법인·로펌)
- **역공·심판청구 위험 등급** — 경고장 수령 시 ① 무효·권리범위확인 심판, ② 채무부존재확인·부당경고
  손해배상 소송, ③ 공정위 신고로 맞설 가능성
- 관계 위험(고객·투자자·파트너 overlap)

발견을 메모 블록으로 제시하고 **초안 전 사용자가 진행을 원하는지 확인**한다.

### Step 3: 권리 유효성·침해 성립 점검 (부당경고 방지의 핵심)

부당경고 역공의 빌미를 없애려면 두 가지가 colorable해야 한다:

- **권리 유효성** — 권리가 유효하게 존속하나? 등록 상태·존속기간·계류 중 무효심판 여부 점검
  `[KIPRIS — 직접 검증 필요]` `[심판원 심결 — 직접 검증 필요]`. 무효 가능성이 높은 권리로 경고장을
  보내는 것은 부당경고의 전형이다 `[검토]`.
- **침해 성립** — 주장된 사실관계에서 침해가 성립하나? 상표는 동일·유사 + 출처혼동, 특허는 청구항
  구성요소 대비, 저작권은 의거성·실질적 유사성, 디자인은 동일·유사. **침해 성립은 주관적 법률
  판단이다 — 단정하지 않고 `[검토]`**(CLAUDE.md `## 주관적 법률 판단의 결정 자세`). 침해가
  성립하는지 불확실하면 그 자리에 `[검토]`를 붙이고, 침해를 가정한 채 경고장을 작성하지 않는다.

둘 중 하나라도 약하면 멈추고 변호사 검토로 라우팅: "권리 유효성/침해 성립이 이 사실관계에서
약합니다. 근거 없는 경고는 부정경쟁·업무방해·손해배상(부당경고) 역공의 빌미입니다 — 발송 전 변호사
검토가 필요합니다." `[검토]`

### Step 4: 초안 (구체성)

구체성으로 초안한다: 등록번호, 날짜, URL, 증거. 구조:

- 발신인·수신인 표시(내용증명이면 3부 — 발신·수신·우체국 보관용 동일 문안)
- 권리의 특정(등록번호·등록일·존속 상태)
- 침해 행위의 특정(구체적 사실·증거·일시)
- 법적 근거 — 인용 placeholder `[CITE: X]` + 출처·검증 태그(`[법망]` / `[모델 지식 — 검증 필요]`).
  조문은 발송 전 법망 대조 검증.
- 구체적·번호 매긴 요구사항(사용 중지·폐기·답변기한 등)
- 집행 입장에 맞춘 결과 고지(불이행 시 — 단정·과장은 부당경고 위험)
- 답변 기한·연락처

**부당경고 위험 주의.** 결과 고지에서 과장·단정(예: 무효 가능성 있는 권리를 "명백한 침해"로 단정,
형사고소를 위협 수단으로 과장)은 부당경고·협박 빌미가 된다 `[검토]`. 사실과 권리 범위 안에서 절제한다.

사용자가 자격 없는 실무자면(`## 누가 이 플러그인을 사용하나`) 추가:

> 경고장·내용증명 발송과 침해 주장은 변호사 영역입니다(출원·심판은 변리사). 변호사와 검토하셨습니까?

> 전문 법무법인·특허법인(소송 대리는 변호사). 많은 곳이 초기 상담을 제공합니다.

게이트와의 명시적 engagement 없이 최종 출력을 작성하지 않는다.

## 수신 모드 — 받은 경고장 트리아지

### Step 1: 핵심 사실 추출

받은 경고장에서: 발신인, 수신인, 주장 권리, 주장 침해 행위, 법적 근거, 요구, 기한, 톤.

### Step 2: 주장 평가 (구조적 읽기, 의견 아님)

- **권리 유효성** — 등록·존속 상태, 무효·취소 사유 가능성 `[KIPRIS — 직접 검증 필요]`
  `[심판원 심결 — 직접 검증 필요]`
- **주장 사실관계에서 침해 성립 개연성** — 단정 말고 `[검토]`
- **과대주장(overbreadth) 점검** — 권리 범위를 넘는 요구인가
- **시기 문제** — 권리 행사의 실효·소멸시효·제척기간 가능성 `[검토]`
- **반격 기회** — 무효심판·권리범위확인심판(소극적)·채무부존재확인·부당경고 손해배상 가능성

### Step 3: 우리 노출 평가

- 우리가 실제 침해하고 있나? `[검토]`
- 준수 비용 vs 다툼 비용
- 발신인 신빙성(권리 troll인가 진성 권리자인가, 패턴 경고장인가)
- 부수적 이해관계(거래·브랜드·다른 권리)

### Step 4: 옵션 제시 (A–F)

- **A — 신속 준수.** 그들이 옳거나 다툴 가치가 없을 때. 트레이드오프: 행위 중단; SEO·계정·생계 영향.
- **B — 협상.** 라이선스·표시 변경·일부 시정 여지. 협상 통신 위생(화해 목적 통신).
- **C — 단호한 반박(거부).** 권리 무효·침해 불성립을 다툴 근거가 있을 때.
- **D — 무시 + 보존.** 위험이 작고 증거를 보존하며 관망. 단 응답 부재가 불리하게 작용할 수 있음 `[검토]`.
- **E — 선제 대응.** **채무부존재확인의 소** 또는 **권리범위확인심판(소극적)** 제기로 선제.
- **F — 권리 자체를 다툼.** **무효심판·취소심판**(특허·상표·디자인 — 특허심판원). 형사 위협이 있으면
  무고·부당경고 대응도 검토 `[검토]`.

각 옵션에 트레이드오프를 붙이고, 권고 하나를 두 문장 근거와 함께 제시한다.

## 이 스킬이 안 하는 것

- 경고장을 자율적으로 발송하지 않는다. 초안만. 변호사가 검토·발송.
- 묻지 않고 웹·모델 지식에서 인용을 backfill하지 않는다.
- 수신 측에서 메리트를 확정적으로 결정하지 않는다 — 구조적 읽기, 모든 권위는 검증 플래그.
- 발송 게이트를 우회하지 않는다.
- 형사고소·무효심판·소송 제기를 대신하지 않는다 — 변호사·특허법인 영역으로 라우팅.
- 사용자 확인 없이 매터 레코드를 생성하지 않는다.
