# /employee-invention

종업원의 발명이 직무발명인지, 권리가 어떻게·언제까지 승계되는지, 보상이 정당한지를 점검한다.

가드레일·헤더·검토자 메모·태그 어휘·결정 자세·산출물 포맷은 모두 이 플러그인의
`~/.claude/plugins/config/claude-for-legal-kr/ip-legal/CLAUDE.md`가 정본이다 — 여기서 재진술하지
않고 가리킨다. 충돌 시 그 CLAUDE.md가 우선한다. 특히 `## IP 실무 프로파일` → 직무발명(발명진흥법)
섹션(보상규정·예약승계 약정·보상 체계)을 먼저 읽고 적용한다.

> **한국 특유 — 미국식 work-for-hire가 아니다.** 미국은 employment agreement로 발명을 사용자에게
> 직접 귀속시키는 모델이 흔하나, 한국 발명진흥법은 다르다: 직무발명은 **종업원에게 원시귀속**되고,
> 사용자는 무상의 **통상실시권**을 법정으로 가지며, 권리(특허받을 권리·특허권)를 사용자가 가지려면
> **예약승계 + 정당한 보상**이 필요하다(§10·§15). 권리 귀속·승계·보상 적정성을 단정하지 않고 `[검토]`.

## 무엇을 하나

1. **config 읽기.** `~/.claude/plugins/config/claude-for-legal-kr/ip-legal/CLAUDE.md` 읽기 —
   직무발명 규정 존재 여부, 예약승계 약정(근로계약·취업규칙·별도 약정), 보상 체계, 역할(변호사·
   변리사·자격 없는 실무자), 에스컬레이션. 없거나 `[PLACEHOLDER]`면 substantive 작업 전
   `/ip-legal:cold-start-interview`로 안내(일반 답은 unconfigured 태그와 함께 가능).

2. **사실관계 확보.** 발명 신고서·발명자·소속·직무 내용, 근로계약·취업규칙·직무발명 보상규정,
   발명 완성 시점·경위를 받는다. 없는 것은 멈추고 요청하거나 플래그(침묵 보완 금지).

3. **세 단계 워크플로 실행** (아래). 각 단계의 주관적 판단은 `[검토]`.

4. **다음 단계 의사결정 트리로 마무리** (CLAUDE.md `## 산출물`).

## 적용 법령

발명진흥법(직무발명) — 법망 검증(연결 시; 미연결이면 인용은 `[모델 지식 — 검증 필요]`):

```
mcp__beopmang__law get "발명진흥법 제2조"
mcp__beopmang__law get "발명진흥법 제10조"
mcp__beopmang__law get "발명진흥법 제15조"
mcp__beopmang__tools verify {"citation": "발명진흥법 제16조"}
```

조문 핀포인트는 `[검증 필요-핀포인트]`. **보상금 산정 기준·소멸시효 관련 판례는 변동**이 있으므로
산정·시효를 다룰 때 `[검증 필요 — 최근 판례]` + 법망 case search. 조문 번호·항·호를 기억으로
하드코딩하지 않는다.

## 단계 ① 직무발명 해당성 판단

**핵심 질문:** 이 발명이 발명진흥법상 **직무발명**인가, 아니면 자유발명(개인발명)인가? 직무발명이면
사용자의 통상실시권·예약승계가 작동하고, 자유발명이면 사용자는 예약승계 약정이 있어도 그 발명을
당연히 가질 수 없다(자유발명에 대한 예약승계 조항은 효력이 제한될 수 있다).

발명진흥법 §2의 정의 요소로 판단한다(현행 텍스트 대조 — 단정 금지):

1. **종업원·법인의 임원·공무원**(이하 "종업원")이 한 발명인가? (사용자·종업원 관계 — 도급·외주·
   파견·자문 관계는 별도 판단 `[검토]`)
2. **그 발명이 성질상 사용자의 업무 범위에 속하는가?**
3. **발명을 하게 된 행위가 종업원의 현재 또는 과거의 직무에 속하는가?**

세 요소(종업원·업무 범위·직무 관련성)가 모두 충족되어야 직무발명이다. 흔히 다투어지는 지점:

- **과거 직무 포함.** "현재 또는 과거"의 직무다 — 부서 이동·전직 전 직무에서 비롯된 발명도
  포함될 수 있다. `[검토]`
- **직무 관련성의 강도.** 직무가 발명을 "예정"하거나 "기대"되는 정도까지 요구되는지, 단순 관련으로
  족한지 — 사안별·판례 의존. `[검토]` `[검증 필요 — 최근 판례]`
- **완성 시점.** 발명이 재직 중 완성되었나, 퇴직 후인가? 퇴직 후 완성이라도 재직 중 직무에서
  비롯되었으면 다툼의 여지. `[검토]` (→ 퇴직자 시나리오는 trade-secret-protection과 교차)
- **업무 범위 vs 직무 범위.** "업무 범위"(회사 사업)와 "직무"(개인 담당)는 다른 층위다 — 둘 다 본다.

산출:

| 요소 | 사실관계 | 충족 여부 | 비고 |
|---|---|---|---|
| 종업원성 | [발명자·관계] | [충족 / `[검토]`] | [도급·자문이면 별도] |
| 업무 범위 | [회사 사업·발명 내용] | [충족 / `[검토]`] | |
| 직무 관련성(현재·과거) | [담당 직무·발명 경위] | [충족 / `[검토]`] | [과거 직무 포함 여부] |

**판정:** [직무발명 / 자유발명 / `[검토]` — 핵심 다툼 지점]. 직무발명 해당성은 주관적 판단이 들어가는
임계점이므로 단정하지 않는다 — 불확실하면 `[검토]`.

## 단계 ② 권리 승계 절차·기한

직무발명으로 판정(또는 가정)되면, 권리 귀속·승계 구조를 점검한다(발명진흥법 §10).

**디폴트 귀속 (약정 없을 때):**
- 특허받을 권리·특허권은 **종업원에게 원시귀속**.
- 사용자는 무상의 **통상실시권**을 법정으로 가진다(중소기업 등 예외·요건은 현행 조문 대조 `[검토]`).
- 즉, 예약승계 근거가 없으면 사용자는 권리를 가질 수 없고 통상실시권만 가진다.

**예약승계 — 사용자가 권리를 가지려면 근거가 필요하다:**
- 근거는 **근로계약·근무규정(취업규칙)·별도 계약**에 미리 정한 예약승계 조항.
- config의 예약승계 약정 항목을 확인한다 — 근로계약·취업규칙·별도 약정 중 어디에 있나, 미정비인가.
- **자유발명에 대한 예약승계·통상실시권 부여 조항은 무효**(§10③ 취지) — 직무발명에만 유효. `[검토]`

**승계 절차·기한:**
- 종업원의 **직무발명 완성 사실 통지(신고)** → 사용자의 **승계 여부 통지(기간 내)** → 승계 의사
  표시로 권리 이전. 기간 내 승계 의사를 알리지 않으면 승계하지 않은 것으로 보는 효과 등 — 기간·
  효과는 현행 조문 대조 `[검증 필요-핀포인트]`.
- 통지·승계의 **기한 도과**는 사용자가 권리를 잃거나 통상실시권만 남는 결과를 낳을 수 있다 — 기한
  관리가 핵심.

점검 항목:

- [ ] 예약승계 근거가 있는가? (근로계약 / 취업규칙 / 별도 약정 — 어디에) `[검토]`
- [ ] 취업규칙 예약승계 조항이 적법하게 작성·신고·주지되었는가? (근로기준법상 취업규칙 절차와 교차) `[검토]`
- [ ] 발명 신고가 접수되었는가? 신고일은?
- [ ] 사용자의 승계 여부 통지가 기간 내 이루어졌는가? `[검증 필요-핀포인트]`
- [ ] 공동발명자가 있는가? (사외 공동발명·산학협력 → 지분·귀속 별도) `[검토]`
- [ ] 외국 출원·외국 권리가 걸리는가? → 그 관할 직무발명 법제는 다름 `[외국법 — 외국 변호사·대리인 검증 필요]`

**권리 귀속·승계 유효성은 단정하지 않는다 — `[검토]`.** 예약승계 근거의 유효성, 절차·기한 준수,
공동발명 지분은 분쟁의 핵심이다.

## 단계 ③ 정당한 보상 적정성 점검

직무발명을 승계하면 종업원은 **정당한 보상**을 받을 권리가 있다(발명진흥법 §15). 보상 미지급·
부적정은 직무발명 분쟁·소송의 핵심 원인이다(§16 — 보상 미지급의 효과).

**보상규정 절차 점검(§15) — 절차를 지키면 보상의 정당성이 추정되는 구조:**

- [ ] 보상형태·보상액 결정 기준을 정한 **보상규정**이 있는가?
- [ ] 보상규정 작성 시 **종업원과의 협의**가 있었는가?
- [ ] 보상규정이 **공표·주지**되었는가?
- [ ] 보상액 결정 시 종업원의 **의견 청취**가 있었는가?
- [ ] 종업원이 **이의를 제기**할 절차가 있는가?

절차 요건(협의·공표·의견청취·이의)은 현행 §15 텍스트로 대조 `[검증 필요-핀포인트]`. 절차를
거치지 않은 보상은 정당성 추정을 받지 못하고 보상액 자체가 다투어진다.

**보상형태(보상 체계 — config의 보상 체계 항목과 대조):**

| 보상형태 | 트리거 | 점검 |
|---|---|---|
| 출원보상 | 출원 시 | [지급 여부·기준] |
| 등록보상 | 설정등록 시 | [지급 여부·기준] |
| 실시보상 | 사용자가 자기 실시 | [매출·이익 연동 기준] `[검토]` |
| 처분보상 | 제3자 라이선스·양도 | [수익 연동·요율] `[검토]` |

**보상금 산정 — 가장 다툼이 잦고 판례 변동이 큰 영역:**
- 실시·처분 보상의 산정은 사용자가 발명으로 얻을 이익, 발명자 공헌도, 사용자 공헌도(독점권 기여)
  등을 고려 — **구체적 산정 기준·공식은 판례·사안 의존이며 변동**한다. `[검토]` `[검증 필요 — 최근 판례]`
- 보상금 청구의 **소멸시효** 기산·기간도 판례 변동 영역. `[검증 필요 — 최근 판례]`
- 법망 case search로 최근 판례를 확인한다(연결 시). 단정하지 않는다.

> **보상 적정성은 결론짓지 않는다.** 절차 충족 여부는 점검표로 surface하되, "보상이 정당하다/
> 부족하다"는 결론은 법원이 내린다 — 산정·적정성은 `[검토]`로 플래그하고 다툼 지점을 식별한다.

**보상 분쟁은 변호사 영역.** 출원·등록은 변리사, 직무발명 보상 분쟁·소송은 변호사다(CLAUDE.md
변리사·변호사 이원화). 분쟁 징후(보상 거부, 퇴직자 보상금 청구, 산정 다툼)가 있으면 변호사
에스컬레이션 — config의 "직무발명 보상 분쟁"은 자동 에스컬레이션 대상이다.

## 이중 심각도 (CLAUDE.md)

직무발명 발견은 두 축으로 surface:
- **법적 위험:** 🔴 차단 / 🟠 높음 / 🟡 중간 / 🟢 낮음 — 권리 미승계·소멸, 보상 분쟁·소송, 무효 사유?
- **사업·상용화 마찰:** 🔴 출원·사업화 중단 / 🟠 지연·재협상 / 🟡 비용·협상 부담 / 🟢 비가시 —
  핵심 발명의 권리 확보 불발, 인재 이탈, 보상 비용?

예: 예약승계 근거 미정비로 핵심 발명 권리를 회사가 못 가지는 상황은 법적 위험·사업 마찰 모두 🔴.

## 이 스킬이 하지 않는 것

- 권리 귀속·승계 유효성·보상 적정성을 단정하지 않는다 — 점검·플래그하고 다툼 지점을 식별한다. 결론은
  변호사·법원.
- 출원·등록을 대행하지 않는다 — 변리사 영역. 이 스킬은 권리 귀속·승계·보상 분석이다.
- 보상금을 확정 산정하지 않는다 — 산정은 판례·사안 의존이고 변동한다. `[검토]` `[검증 필요 — 최근 판례]`.
- 발명진흥법 조문·기한·보상 절차를 기억으로 단정하지 않는다 — 법망 현행 조문 대조, 판례는 case search.
