# 생성형 AI 표시의무(AI 기본법) 컴플라이언스

## 매터 컨텍스트

실무 수준 CLAUDE.md `## 매터 워크스페이스` 확인. 비활성이면 skip. 활성이면 묻기.

## 목적

AI 기본법(시행 **2026.1.22.**)은 생성형 인공지능에 **표시의무**를 부과한다 — 산출물이 생성형
AI로 만들어졌다는 표시, 그리고 실제와 구분이 어려운 **가상의 결과물(딥페이크 등)**에 대한 표시.
세부(대상 범위·표시 방법·면제·과태료)는 시행령·과기정통부 고시로 정해지며 **시행 직후라 정착 중**.

이 스킬은 (a) 제품이 표시의무 대상인지 판정, (b) 무엇을·어떻게 표시할지 권고, (c) 인접 법령
(불법합성물·선거 딥페이크 등)과의 교차를 점검한다.

**왜 별도 스킬인가:** 표시의무는 미국·EU 원본 플러그인에 등가물이 없는 한국 AI 기본법 특유의
사업자 의무다. 생성형 AI 제품(이미지·영상·음성·텍스트 생성)이 급증하면서 실무 수요가 크고,
형사처벌이 걸린 인접 법령(성폭력처벌법 불법합성물, 공직선거법 딥페이크)과 교차해 잘못 다루면
리스크가 크다.

## 핵심 조문

법망에서 현행 텍스트 가져오기 (신법이라 조문 번호·표시 방법·면제는 반드시 현행 확인):

```
mcp__beopmang__law get "인공지능 발전과 신뢰 기반 조성 등에 관한 기본법"
mcp__beopmang__law get "인공지능 발전과 신뢰 기반 조성 등에 관한 기본법 시행령"
mcp__beopmang__tools verify {"citation": "인공지능 발전과 신뢰 기반 조성 등에 관한 기본법"}
```

법망 미연결 시 모든 인용 `[모델 지식 — 검증 필요]`. **표시 방법·기술 기준은 과기정통부 고시로
정해지며 법망 미커버 가능** — `[과기정통부 — 직접 검증 필요]`, 과기정통부 사이트
(`https://www.msit.go.kr`) 직접 확인. 모든 AI 기본법 조문 인용에 `[AI 기본법 현행 텍스트 대조 — 검증 필요]`.

## 두 가지 표시 의무 (구분)

| 구분 | 대상 | 취지 |
|---|---|---|
| ① 생성형 AI 산출물 표시 | 생성형 AI가 만든 결과물(텍스트·이미지·음성·영상) | 이용자가 AI 생성물임을 인지 |
| ② 가상 결과물 표시 | 실제와 구분이 어려운 결과물(딥페이크·합성 영상·음성 등) | 실재 오인·기만 방지 |

①과 ②는 중첩되나 별개. 사실적 합성 영상은 ①·② 모두 해당 가능. 명백히 비현실적인 일러스트는
① 해당하나 ②는 아닐 수 있음 `[검토 — 현행 고시상 "가상 결과물" 정의 확인]`.

## 워크플로우

### Step 1: 제품 분석

- **산출물 유형:** 텍스트 / 이미지 / 음성 / 영상 / 혼합
- **실재 오인 가능성:** 사람·실제 사건과 혼동될 수 있는가? (사실적 인물 영상·음성 복제 등)
- **사업자 역할:** 우리가 생성형 AI를 **개발·제공**(AI개발사업자)하는가, 타사 모델을 **이용**
  (AI이용사업자)해 서비스하는가? 역할에 따라 의무 주체·범위 다름 `[AI 기본법 현행 텍스트 대조 — 검증 필요]`
- **노출 맥락:** B2C 공개 서비스 / 내부 도구 / API(다운스트림이 표시) / 콘텐츠 플랫폼
- **이용자:** 일반 이용자 / 사업자 / 미성년자 포함?

### Step 2: 표시의무 대상 판정

- 생성형 AI 산출물인가 → ① 표시 대상
- 실제와 구분 어려운 결과물인가 → ② 표시 대상(가중)
- 면제 사유 해당 여부 확인 `[검증 필요 — 현행 면제 범위, 예: 명백히 예술적·창작적 표현 등]`

결론: **① 표시 / ①+② 표시 / 면제 / 불확실(현행 고시 확인 필요)**.

### Step 3: 표시 방법 권고

현행 과기정통부 고시 확인 전제로, 일반 권고(고시 확정 시 갱신):

| 산출물 | 표시 방법(권고) |
|---|---|
| 텍스트 | 응답 내·인접에 "AI 생성" 고지, 챗봇은 대화 시작 시 AI임을 고지 |
| 이미지 | 가시적 라벨/워터마크 + 메타데이터(C2PA 등 출처표시 표준 검토) |
| 음성 | 가청 고지 또는 메타데이터, 합성 음성임을 인지 가능하게 |
| 영상 | 가시적 표시 + 메타데이터, 딥페이크는 명확·지속 표시 |

`[과기정통부 — 직접 검증 필요]` — 구체적 표기 문구·위치·기술 기준은 고시 의존.

### Step 4: 교차 법령 점검 (형사·선거 리스크)

생성형 AI 산출물은 AI 기본법 표시의무 외에 다음과 교차할 수 있다 — 표시의무 충족이 이들 책임을
면제하지 않음:

| 법령 | 트리거 | 비고 |
|---|---|---|
| 성폭력처벌법(불법합성물) | 사람 얼굴·신체·음성 합성 성적 영상물 | 형사처벌. 생성·반포 모두. `[검증 필요]` |
| 공직선거법 | 선거운동 목적 딥페이크 영상 등 | 선거일 전 일정 기간 제한·표시 의무 별도 `[검증 필요]` |
| 정보통신망법 | 명예훼손·허위정보 유통 | 합성물 유통 시 책임 |
| 저작권법 | 학습 데이터·생성물의 타인 저작물 침해 | ip-legal 영역 연계 |
| 개인정보보호법 | 실존 인물 이미지·음성 = 개인정보(생체인식정보 §23 가능) | privacy-legal 연계 |

해당 트리거가 보이면 플래그하고 형사·선거 리스크는 변호사 검토 `[검토]`.

## 흔한 실수

- **"명백히 가짜 일러스트라 표시 불필요" — ① 표시는 여전히 대상일 수 있음.** ②(실재 오인)만
  면제될 뿐 ①(AI 생성 고지)은 별개. 현행 고시 확인. `[검증 필요]`
- **"타사 모델 API만 쓰니 우리 의무 아님" — 위험.** AI이용사업자도 서비스 제공 주체로서
  표시 책임 가능. 역할별 의무 현행 확인. `[AI 기본법 현행 텍스트 대조 — 검증 필요]`
- **"AI 기본법 표시했으니 끝" — 형사·선거법 별개.** 표시의무 충족이 불법합성물·선거 딥페이크
  책임을 면제하지 않음.
- **"내부 도구라 표시 불필요" — 맥락 의존.** 산출물이 외부로 나가거나 의사결정에 쓰이면 재검토.
- **"워터마크는 기술 선택 자유" — 고시 기준 확인.** 표시 방법·기술 기준이 고시로 정해지면
  임의 방식이 미충족일 수 있음. `[과기정통부 — 직접 검증 필요]`
