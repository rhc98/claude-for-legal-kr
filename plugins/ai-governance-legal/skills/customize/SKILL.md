# /customize

## 실행 조건

사용자가 `/ai-governance-legal:customize` 입력. 콜드스타트 인터뷰 전체 재실행 없이, YAML
수동 편집 없이 AI 거버넌스 프로파일의 무엇 — 위험 성향·에스컬레이션 담당자·활용사례 등기부
항목·벤더 AI 입장·AI 정책 약속·영향평가 하우스 스타일·매터 워크스페이스 경로 — 을 변경하고
싶음.

## 할 일

1. **config 읽기.** `~/.claude/plugins/config/claude-for-legal-kr/ai-governance-legal/CLAUDE.md`
   (와 한 단계 위 `~/.claude/plugins/config/claude-for-legal-kr/company-profile.md`) 읽기.
   플러그인 config 없거나 `[PLACEHOLDER]` 값 남아 있으면:

   > 셋업을 아직 안 하셨습니다. 먼저 `/ai-governance-legal:cold-start-interview` 실행 —
   > customize는 이미 가진 프로파일을 조정하기 위한 것.

2. **맞춤화 가능 맵 보여주기.** 프로파일에 있는 것을 그룹화, 현재 값의 한 줄 요약과 함께:

   - **회사 / 우리는 누구인가** — 회사명·업종·관할·단계·실무 환경 *(모든 12 플러그인 공유 —
     `company-profile.md` 통해 변경 전파)*
   - **규제 풋프린트** — AI 기본법 / PIPA §37-2 / 분야법(금융·의료·채용) / 외국법(EU AI Act 등)
     중 실제 적용되는 것
   - **위험 성향** — 보수·중간·적극, 각각의 의미 (트리아지·영향평가 출력·벤더 레드라인 영향)
   - **인력** — 거버넌스 팀·AI 위험 담당(CISO/CPO/GC)·벤더 관계 담당·에스컬레이션 체인·승인권자
   - **AI 활용사례 등기부** — 승인·조건부·절대 안 됨 항목, 각 항목에 붙은 조건·레드라인
   - **거버넌스 등급** — 표준·상향·고위험 각 등급의 승인 경로와 예시 활용사례
   - **AI 시스템 인벤토리** — 시스템별 사업자 역할(AI개발사업자/AI이용사업자)과 유형
     (고영향/생성형/일반). 전용 편집기는 `/ai-governance-legal:ai-inventory`.
   - **벤더 AI 거버넌스** — 데이터 이용(학습 금지)·감사·산출물 책임·사고 통지·모델 변경 통지·
     인적 검토 권한·위수탁/국외이전 등 벤더 AI 플레이북 입장
   - **AI 정책 약속** — 이 플러그인이 실무 대조 감시하는 대내·대외 약속(금지 이용·안전조치·
     고지 의무·생성형 표시의무·allowlist/blocklist)
   - **AI 영향평가 하우스 스타일** — 섹션 순서·위험 점수 형식·이해관계자 framing·트리거
   - **워크플로우** — 인테이크 경로·산출물 폴더·매터 워크스페이스 경로·policy-monitor sweep cadence
   - **통합** — 문서 저장소·Slack·스케줄드 태스크·**법망 MCP** 상태·폴백

3. **무엇을 바꿀지 묻기.**

   > 무엇을 조정할까요? 섹션을 고르거나 자기 말로 변경 설명.

4. **변경 수집.** 사용자가 섹션을 고르면 그 섹션의 현재 값을 보여주고 새 값 묻기. 사용자가
   자유 형식으로 설명하면 어느 섹션이 영향인지 식별하고 mapping 확인.

5. **변경 전 검증.** 변경에 법적 인용·임계점·시행일·고영향 영역 분류가 있으면 법망 대조 검증:
   ```
   mcp__beopmang__law get "[조항]"
   ```
   인용이 안 맞으면 사용자에게 확인 ("당신은 X라 하셨는데, 법망의 현행 텍스트는 Y. 어느
   것으로 갈까요?"). **AI 기본법은 2026.1.22. 시행 신법이라 활용사례 분류·거버넌스 등급 ·
   고영향 영역 변경은 특히 법망 또는 과기정통부 고시 대조가 필요하다 —
   `[AI 기본법 현행 텍스트 대조 — 검증 필요]` 태그.** 법망 미연결이면 `[모델 지식 — 검증 필요]`
   로 떨어지고 그 사실을 사용자에게 알린다.

6. **변경 적용.** CLAUDE.md의 해당 섹션을 surgical하게 갱신한다 — 섹션 전체 재작성 금지,
   해당 라인만. 변경 전후 diff 보여주기.

7. **확인 + 검증 로그 기록.** 변경에 검증된 법적 인용이 포함되면:
   ```
   [YYYY-MM-DD] customize - [성명]이(가) [섹션]에서 [구체적 변경]
   ```
   `~/.claude/plugins/config/claude-for-legal-kr/ai-governance-legal/verification-log.md`에 추가.

8. **공유 프로파일 변경**(회사명·업종·관할·실무 환경·단계)은
   `~/.claude/plugins/config/claude-for-legal-kr/company-profile.md`에 쓰고 알린다:

   > 이 변경은 12개 플러그인 전부에 영향을 줍니다 — 관할 풋프린트를 읽는 모든 플러그인이
   > 이제 [새 값]을 봅니다.

## 흔한 맞춤화 패턴

**위험 성향 변경:** "중간에서 보수로" → 영향 다시 보여주고 사용자 확인. "더 많은 활용사례를
승인 대신 조건부로 플래그하고, 영향평가 후속 질문을 더 surface하고, 벤더 AI 레드라인을 더
보수적으로 권고합니다."

**에스컬레이션 담당자 추가·변경:** 에스컬레이션 매트릭스 전체 보여주기. "에스컬레이션을
라우팅하는 모든 스킬(`/use-case-triage`, `/vendor-ai-review`, `/reg-gap-analysis`)이 이제
해당 위험 밴드에 이 담당자를 포함합니다."

**활용사례 등기부 신규 항목:** "`/use-case-triage`가 다음 실행부터 이 항목에 매칭합니다.
기존 영향평가는 다시 쓰이지 않습니다 — 새 입장을 반영하려면 재실행하세요."

**거버넌스 등급 변경:** 예시 활용사례를 한 등급에서 다른 등급으로 이동(예: 채용 AI를 상향에서
고위험으로) → 트리아지·에스컬레이션 라우팅 영향. 고영향 영역 분류가 걸리면 `[검토]` + 법망 대조.

**벤더 AI 입장 단 한 줄 변경:** "학습 금지 폴백 수용에서 절대 안 됨으로" → 영향: 모든 미래
벤더 AI 검토.

**AI 정책 약속 갱신:** "생성형 AI 표시의무 적용 범위를 외부 콘텐츠에서 전체로 확대" →
policy-monitor 트리거 — 실제 AI/이용정책 문서 갱신 필요? 사용자에게 묻기.

**AI 시스템 인벤토리:** 시스템별 역할·유형 편집은 전용 편집기로 라우팅 — "이건
`/ai-governance-legal:ai-inventory`가 더 잘합니다. `add | edit <id> | classify <id>`."

**법망 MCP 추가·제거:** `.mcp.json` 직접 편집 안내 또는 자동화. **AI 기본법 신법 영역에서는
법망이 1순위 통합임을 다시 강조.**

## 가드레일

- **섹션을 절대 삭제하지 않는다.** 사용자가 무언가를 "제거"하고 싶으면 `[미설정]`으로
  설정하고 그것이 플러그인 동작에 무엇을 의미하는지 설명한다. ("에스컬레이션 체인을
  제거하면 `/use-case-triage`는 에스컬레이션 가치가 있는 항목을 플래그하지만 특정 담당자에게
  라우팅하지 않습니다.")
- **내부 불일치 플래그.** 변경이 프로파일을 불일치하게 만들면(예: 위험 성향 적극 + 에스컬레이션
  "전부 GC로"; "EU AI Act 적용" + "EU용으로 플래그된 시스템 없음"; 거버넌스 등급은 고위험인데
  인벤토리에 대응 시스템 없음), 긴장을 플래그하고 어느 쪽인지 묻는다.
- **AI 기본법 신법 경고.** 변경이 AI 기본법 조문·고영향 영역·하위 고시 임계점에 confident하게
  의존하면, 신법(2026.1.22. 시행, 시행령·고시 정착 중)임을 알리고 법망 또는 과기정통부 고시
  대조 후 진행. 하드코딩된 역할 × 유형 → 의무 매핑을 프로파일에 굳히지 않는다.
- **가드레일 약화 플래그.** 사용자가 가드레일을 끄려 하면("`[검토]` 플래그 그만 붙여",
  "인용 경고 빼", "비밀유지 헤더 건너뛰어") 그 가드레일이 무엇을 보호하는지 설명하고
  trade-off를 이해했는지 확인한다. 대부분의 가드레일은 조정 가능하나, 몇 개는 구조적이다:
  - `[검토]` 플래그 메커니즘(확신에 찬 잘못된 답 대신 변호사 판단이 필요한 지점을 알림) —
    load-bearing, 제거 불가.
  - 가져온 콘텐츠의 출처 attribution 태그 — load-bearing, 제거 불가.
  - 인용된 법령·고시의 `[검증 필요]` / `[AI 기본법 현행 텍스트 대조 — 검증 필요]` 태그 —
    load-bearing, 제거 불가.
- **한 번에 하나.** 인터뷰 전체를 다시 묻지 않는다. 사용자가 여러 변경을 원하면 순차적으로
  처리하고 각각 확인 후 다음으로.
