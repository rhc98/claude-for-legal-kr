# 소송·분쟁 실무 플러그인 (litigation-legal)

한국 민사·형사·행정·가사 소송과 보전·집행·분쟁 실무를 위한 Claude Code 플러그인. 사건 인테이크·
포트폴리오·기한 관리(제소·상소기간·소멸시효), 소장·답변서·준비서면 작성, 요건사실 정리표, 증인·당사자
신문 준비, 증거보전·문서제출명령, 문서제출거부 사유 검토, 내용증명·지급명령 대응, 보전처분(가압류·
가처분)·강제집행, 형사 고소·고발 트리아지, 전자소송 워크플로우, 조정·화해·중재를 다룬다.

원본 [`anthropics/claude-for-legal`](https://github.com/anthropics/claude-for-legal)의
상류 `litigation` 계열 플러그인(Apache 2.0)을 한국 소송법체계 — **민사소송법·민사집행법·형사소송법·
행정소송법·가사소송법·중재법·소액사건심판법** — 로 포팅한 결과물이다. 국가법령정보 MCP(`korean-law`)로
절차법·판례·헌재결정 위에서 동작한다.

**모든 산출물은 변호사 검토 전제의 초안이다** — 인용된 조문·판례는 국가법령정보 MCP로 검증해 표시하고,
판단해야 할 지점은 `[검토]` 플래그를 단다. 결정적 행위(서면 제출, 상대방·법원 송부, 상소·연장 신청,
에스컬레이션)는 명시적 확인 게이트를 통과해야 한다.

> **한국에는 미국식 디스커버리·deposition·attorney work-product 제도가 없다.** 상류는
> **미국 연방·주 민사소송(FRCP·docket·subpoena·deposition·privilege log)**이 전제다. 한국 포팅의 핵심
> 재맥락화는 **대륙법계 절차로의 재설계**다 — "어느 법원·연방/주 절차냐"라는 미국의 축이 한국에서는
> **사건유형(민사/형사/행정/가사/보전/집행) × 심급(1심/항소심/상고심)**의 조합으로 바뀐다. 적용 절차·
> 기한·서면·불복방법이 모두 이 조합에 따라 달라진다.

## 무엇을 하나

- **사건 라이프사이클** — 인테이크(당사자·사건유형·관할·소가·기한시계)부터 경과 업데이트, 브리핑,
  포트폴리오 대시보드, 종결까지. 다중 의뢰인 실무를 위한 매터 워크스페이스 격리.
- **기한 관리** — 항소·상고(2주)·즉시항고(1주)·지급명령 이의(2주)·행정 제소(90일)·재심(30일) 불변기간과
  소멸시효·제척기간을 한 곳에서 산정·추적. `deadline-watcher` 에이전트가 임박 기한을 주기적으로 알린다.
- **법원 제출 서면** — 소장·답변서·준비서면·상소이유서의 청구취지·청구원인·요건사실·항변을 하우스
  스타일로 초안. 요건사실 정리표(입증계획·증거공백 탐지)와 시간순 사실관계표로 뒷받침.
- **증거·증거조사(대륙법계 재설계)** — 증인·당사자신문 준비, 문서제출명령·사실조회·증인소환 트리아지,
  증거보전 신청, 문서제출거부 사유 검토. 미국식 deposition·subpoena·privilege log를 한국 증거법으로 재설계.
- **내용증명·지급명령** — 발송 전 인테이크, 최고서·내용증명 초안, 수신 문서 대응(지급명령 이의 포함).
- **한국 특유 트랙** — 형사 고소·고발, 보전처분(가압류·가처분), 강제집행, 전자소송, 소액사건, 조정·화해·중재.

**사건유형(민사/형사/행정/가사)·심급·원고/피고 입장이 이 플러그인의 핵심 변수다.** 같은 서면 이름(예:
"항소이유서")도 사건유형·심급에 따라 요건·기한이 다르므로, 검토·작성 스킬은 먼저 사건유형과 심급을 확인한다.

## 누구를 위한 것인가

| 역할 | 주 사용 영역 |
|---|---|
| **소송대리 변호사** | 서면 작성·요건사실·증인신문·보전·집행·상소, 사건 포트폴리오·기한 관리 |
| **법무법인 송무팀** | 다중 의뢰인 매터 워크스페이스, 사건 전반, 협상·화해·중재 |
| **사내 법무(직접 소송대리 제한)** | 사건 트리아지·기한 관리·서면 초안(변호사 검토 전제), 외부 대리인 관리 |
| **나홀로소송(본인소송) 당사자** | 절차·서식 안내, 서면 초안(리서치 노트 — 변호사 검토 필수), 소송구조 안내 |

> **소송대리에는 원칙적으로 변호사 자격이 필요하다(민사소송법 §87).** 사내 법무·자격 없는 실무자는
> 회사를 대리해 직접 소송을 수행할 수 없고(단독사건 등 예외는 법원 허가 §88), 나홀로소송은 당사자
> 본인만 가능하다. 역할에 따라 산출물 헤더가 달라진다 — 소송대리 변호사가 아닌 사용자의 작성물은
> "법률자문이 아님, 제출 전 변호사 검토 필수"로 처리된다.

## 설치

`.mcp.json`에 포함:

- **국가법령정보(korean-law)** — 한국 법령·판례·헌재결정 조회·검증. **1순위.** 없으면 모든 조문·판례
  인용이 `[모델 지식 — 검증 필요]`로 떨어진다. 환각방지 인용검증(`legal_analysis`)과 판례 생사확인
  Citator(`cite_check` — 전원합의체 판례변경·폐기 확인) 포함. **무료 OC 인증키 필요** —
  [open.law.go.kr](https://open.law.go.kr)에서 발급, 환경변수 `LAW_OC`로 설정.
- **Slack** — 기한 임박 알림·기일 결과·상소 여부·에스컬레이션 논의.
- **Google Drive** — 시드 문서(소장·답변서·준비서면 선례, 증거목록, 판결정본·기일통지서) 로드.
- **Box** — 사건기록·증거자료·서면 검토, 대량 문서 정리.

```
export LAW_OC=발급받은_OC_인증키
claude plugin install litigation-legal@claude-for-legal-kr
/litigation-legal:cold-start-interview
```

콜드스타트 인터뷰가 실무환경(사내 송무/법무법인/개인·단독/나홀로소송), 주력 분야(민사/형사/행정/가사),
역할(소송대리 변호사/사내 법무/나홀로소송 당사자), 소송 입장(원고/피고), 에스컬레이션 매트릭스,
하우스 스타일을 학습한다. 결과는 `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md`에
저장되며 플러그인 업데이트와 무관하게 유지된다. 회사 공통 정보는 상위
`company-profile.md`에 모여 모든 플러그인이 공유한다.

⚠️ **전자소송(ecfs.scourt.go.kr)·나의 사건검색은 MCP가 없다** — 사건현황·기일·송달 현황은 사용자가
기일통지서·판결정본·전자소송 화면을 직접 제공한다(아래 데이터 갭 참조).

## End-to-End 데모

**시나리오 A — 지급명령 수신 대응 (피고·채무자 입장)**

```
1. 지급명령이 날아왔다
   → /litigation-legal:demand-received
     수신 문서 분석 · 이의기간 2주(민사소송법 §470, 불변기간) 산정 · 응소 옵션
     [기간 계산 — 1차 소스 대조 검증 필요] [검토]

2. 이의신청 후 소송으로 이행 → 답변서 작성
   → /litigation-legal:brief-section-drafter  (답변서 · 항변 II — 소멸시효)

3. 항변의 요건사실·증명책임 매핑
   → /litigation-legal:claim-chart  (소멸시효 항변 · 입장: 피고)
     요건사실 ↔ 증거 ↔ 증거공백 탐지

4. 변론기일·제출기한 등록 → deadline-watcher가 D-7 임박 시 즉시 알림
   → /litigation-legal:deadline-tracker --register
```

**시나리오 B — 보전에서 집행까지 (원고·채권자 입장)**

```
1. 본안 전 재산 보전
   → /litigation-legal:provisional-remedy --seizure  (가압류)
     피보전권리 · 보전필요성 · 담보 · 본안 제소명령 · 부당보전 역공 위험 [검토]

2. 본안소송 → 소장 작성 → 요건사실 정리 → 증인신문 준비
   → brief-section-drafter (소장) → claim-chart (청구원인) → deposition-prep

3. 승소 판결 확정 → 강제집행
   → /litigation-legal:enforcement --creditor
     집행권원 · 집행문 · 재산명시(§61~)·조회 · 채권압류·추심·전부

4. 사건 종결 (상소기간 도과 확인 후)
   → /litigation-legal:matter-close
```

## 국가법령정보 데이터 갭 (litigation 특유)

국가법령정보로 **가져올 수 없는** 정보가 있다. 지어내지 않고, 사용자에게 1차 소스를 요청하며 태그로
플래그한다(CLAUDE.md·[`../../NOTICE`](../../NOTICE)와 일관):

- **전자소송(ecfs.scourt.go.kr)·나의 사건검색(사건현황·기일)은 공개 API가 없다.** 사건현황·기일·송달
  현황은 사용자가 기일통지서·판결정본·전자소송 화면을 직접 제공 → `[전자소송/나의사건검색 — 사용자 확인 필요]`.
  **`deadline-watcher` 에이전트는 이 시스템을 스크래핑하지 않는다** — 사용자가 등록한 로컬 등록부에서만 계산한다.
- **하급심 판결문 상당수가 미공개**다. 미공개 하급심을 인용하지 않고, 필요하면 판결정본을 요청한다
  → `[하급심 판결 — 미공개, 직접 확인 필요]`.
- **판례 인용은 `cite_check`로 강제 검증한다.** 전원합의체 판례변경으로 폐기·변경된 선례를 인용하지
  않도록 Citator로 유효성을 확인 → `[선례 유효성 — cite_check 확인]`.
- **기간 산정은 1차 소스 대조가 필요하다.** 기산점(송달일·선고일·고지일)·불변기간·추후보완이 사안별이라
  산정 결과에 → `[기간 계산 — 1차 소스 대조 검증 필요]`.
- **인지·송달료 요율·소액사건 소가 기준은 하드코딩하지 않는다** — 규칙 개정으로 변동, 현행 대조 필수.
- ✓ 잘 커버: 민사소송법·민사집행법·형사소송법·행정소송법·가사소송법·중재법·소액사건심판법 + 대법원·헌재 판례.

## 면책

⚠️ **한국법에는 미국식 attorney work-product 독트린(FRCP 26(b)(3))·디스커버리·deposition 제도가 없다.**
상류 미국 소송 플러그인을 **대륙법계 절차로 재설계**했다 — 증거조사는 법정에서 재판부 앞 증인신문·당사자
신문(법정 외 deposition 없음)으로, 증거확보는 법원을 통한 문서제출명령·사실조회(당사자 발부 subpoena
없음)로, 문서제출거부는 §344 개별 요건(privilege log 제도 없음)으로 이루어진다. 문서 상단의 "work-product"
표지는 비밀유지 의지의 표시일 뿐 **공개거부·문서제출명령 거부의 법적 근거를 만들지 않는다** — 한국에서
변호사 비밀유지는 좁게 인정되고, 사내변호사 작성물은 압수수색에서 보호 여부 자체가 다투어진다.

⚠️ **데이터 갭 — 전자소송·나의 사건검색·하급심 판결문은 사용자 확인이 필요하다.** 공개 API가 없거나
미공개인 정보는 지어내지 않고 위 태그로 플래그하며 1차 소스를 요청한다.

⚠️ **모든 산출물은 변호사 검토 전제의 초안이다.** 청구 인용 여부·항변 유효성·처분성·보전 필요성·상소이유
적법성 등 주관적 법률 판단은 `[검토]` 플래그로 surface하고 변호사가 좁힌다 — 법적 결론이 아니다.
소송대리 변호사가 아닌 사용자(사내 법무·나홀로소송)의 작성물은 제출 전 변호사 검토가 필수이며,
자력이 부족한 당사자는 소송구조(민사소송법 §128)·대한법률구조공단 지원을 안내받는다.

⚠️ **국가법령정보 인용은 참고용이다.** 조문·판례 인용은 국가법령정보 MCP로 검증해 태그하되, 폐기·변경
여부는 `cite_check`로 확인하고, 기산점·기간이 걸린 사안은 도과 시 복구가 불가능하므로 원본 대조를 가장
엄격히 적용한다. 외국 준거법·외국법원 계속·국제재판관할·외국판결 승인·집행 사안은
`[외국법·국제사법 — 외국 변호사 검증 필요]`로 플래그한다.
