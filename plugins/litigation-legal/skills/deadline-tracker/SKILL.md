# /deadline-tracker

## 이 스킬의 최우선 규율 (다른 모든 것에 우선)

**소송에서 기한 도과는 복구 불가능한 권리 상실이다 — 이 플러그인에서 가장 위험한 실패 모드.**
이 스킬은 그 위험을 관리한다. 따라서 다음 세 가지를 절대 어기지 않는다:

1. **기산점을 침묵으로 가정하지 않는다.** 불변기간의 기산점(판결정본 송달일 / 선고일 / 재판 고지일 /
   처분을 안 날)은 **사안별**이다. 사용자가 정확한 날짜를 제공하지 않으면 **산정하지 않고 공식만 제시**한다.
2. **기간·조문을 하드코딩하지 않는다.** 아래 표의 기간·조문은 국가법령정보 현행 조문 대조를 전제로 한
   출발점이다 — 매 산정 시 `get_law_text`로 현행 조문을 대조한다.
3. **모든 산정 결과에 `[기간 계산 — 1차 소스 대조 검증 필요]` + `[검토]`를 붙인다.** 기산점·불변기간
   여부·추후보완 가부는 변호사 판단이다.

정본은 실무 수준 CLAUDE.md `## 소송 플레이북 → 기한 관리 대원칙` · `## 데이터 갭` · `## 기한 트래커 설정`.

## 매터 컨텍스트

실무 수준 CLAUDE.md `## 매터 워크스페이스` 확인. `활성화`가 `✗`(사내 사용자의 디폴트)이면 이 단락 skip —
스킬은 실무 수준 컨텍스트 사용. 활성이고 활성 매터 없으면: "어느 매터입니까?

`on`이 아니면 다른 매터 파일 절대 안 읽음.

## 플레이북 먼저 로드

`~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md`를 읽는다. `## 기한 관리 대원칙`과
`## 데이터 갭`을 적용한다.

> **침묵의 보완 금지(CLAUDE.md `## 공유 가드레일`).** 이 산정에 필요한 조문(민사소송법 §396·§425·§444·
> §456·§470·§173, 행정소송법 §20, 민법 §162~§184)을 국가법령정보에서 확인하지 못하면 멈추지 않고
> `[모델 지식 — 검증 필요]`로 태그하고 진행하되, **기산점·기간은 반드시 사용자 날짜 확인 후에만 산정**한다.
>
> **국가법령정보 조회 예시:**
> ```
> mcp__korean-law__get_law_text "민사소송법 제396조"
> mcp__korean-law__get_law_text "민사소송법 제425조"
> mcp__korean-law__get_law_text "민사소송법 제444조"
> mcp__korean-law__get_law_text "민사소송법 제456조"
> mcp__korean-law__get_law_text "행정소송법 제20조"
> mcp__korean-law__get_law_text "민법 제162조"
> mcp__korean-law__legal_analysis {"mode": "verify_citations", "citation": "민사소송법 제396조"}
> ```
>
> **출처 태그:** `[국가법령정보]`(이 세션 가져온 경우만) · `[사용자 제공]`(기일통지서·판결정본 paste) ·
> `[모델 지식 — 검증 필요]`(디폴트). 산정 결과는 항상 `[기간 계산 — 1차 소스 대조 검증 필요]` + `[검토]`.

## 1단계 — 사건유형·심급 확인 (게이트)

**같은 서면 이름도 사건유형·심급마다 기한·기산점이 다르다.** 불명확하면 묻는다. 확인 없이 산정하지 않는다.

- **사건유형:** 민사 / 형사 / 행정 / 가사 / 보전 / 집행 `[사건유형 확인 — 검증 필요]`
- **심급:** 1심 / 항소심 / 상고심
- **기한의 종류:** 상소 / 즉시항고 / 이의 / 재심 / 제소 / 시효·제척기간

## 2단계 — 기한 카탈로그 (공식 — 하드코딩 아님, 현행 조문 대조 전제)

아래는 **출발점 카탈로그**다. 각 항목은 `get_law_text`로 현행 조문·기간·기산점을 대조한 뒤에만 적용한다.
기간은 자주 변동하는 요율·소가와 달리 비교적 안정적이나, **기산점과 불변기간 여부가 사안별**이라는 점이 핵심이다.

### A. 민사 상소·불복

| 불복수단 | 기간(대조 전제) | 기산점(사안별) | 근거(대조 전제) | 성질 |
|---|---|---|---|---|
| **항소** | 2주 | 판결정본 **송달일** | 민사소송법 §396 `[모델 지식 — 검증 필요]` | 불변기간 |
| **상고** | 2주 | 판결정본 **송달일** | 민사소송법 §425(§396 준용) `[모델 지식 — 검증 필요]` | 불변기간 |
| **즉시항고** | 1주 | 재판 **고지일** | 민사소송법 §444 `[모델 지식 — 검증 필요]` | 불변기간 |
| **지급명령 이의** | 2주 | 지급명령 **송달일** | 민사소송법 §470 `[모델 지식 — 검증 필요]` | — |
| **재심의 소** | 30일 | 재심사유를 **안 날** | 민사소송법 §456 `[모델 지식 — 검증 필요]` | 불변기간(제척기간 5년 별도 `[검증 필요]`) |
| **추후보완** | 2주 | 책임질 수 없는 사유가 **없어진 후** | 민사소송법 §173 `[모델 지식 — 검증 필요]` | 인정 여부 다툼 잦음 `[검토]` |

> **핵심 함정:** 사용자가 "선고일부터 2주"라고 말하면 정정한다 — 항소·상고 기산점은 **선고일이 아니라
> 판결정본 송달일**이다(CLAUDE.md `## 공유 가드레일` 예시 참조). 정본 수령일 확인이 선행이다. `[검토]`

> **형사·행정 상소기간은 별도.** 형사 항소·상고기간(형사소송법)·행정 항소는 사건유형별로 다르므로
> 해당 조문을 대조한다 — 위 민사 표를 형사·행정에 그대로 옮기지 않는다 `[사건유형 확인 — 검증 필요]`.

### B. 행정 제소기간

| 소송 | 기간(대조 전제) | 기산점(사안별) | 근거(대조 전제) |
|---|---|---|---|
| **취소소송 제소** | 90일 | 처분등이 있음을 **안 날** | 행정소송법 §20① `[모델 지식 — 검증 필요]` |
| **취소소송 제소(객관)** | 1년 | 처분등이 **있은 날** | 행정소송법 §20② `[모델 지식 — 검증 필요]` |

> 정당한 사유·행정심판 전치(임의·필요적) 여부에 따라 기산·예외가 달라진다. 행정심판을 거친 경우 재결서
> 정본 송달일 기산 등 별도 규율 — 개별법·§20 대조 `[검토]`. 행정심판 전치 여부는 개별법마다 다르다.

### C. 소멸시효·제척기간

| 유형 | 기간(대조 전제) | 근거(대조 전제) |
|---|---|---|
| **일반 채권 소멸시효** | 10년 | 민법 §162① `[모델 지식 — 검증 필요]` |
| **상행위 채권** | 5년 | 상법 §64 `[모델 지식 — 검증 필요]` |
| **3년 단기시효** | 3년 | 민법 §163 `[모델 지식 — 검증 필요]` |
| **1년 단기시효** | 1년 | 민법 §164 `[모델 지식 — 검증 필요]` |
| **불법행위 손해배상** | 안 날 3년 / 있은 날 10년 | 민법 §766 `[모델 지식 — 검증 필요]` |
| **판결로 확정된 채권** | 10년(단기시효 채권도 연장) | 민법 §165 `[모델 지식 — 검증 필요]` |

> **소멸시효 vs 제척기간은 성질이 다르다** — 중단·정지(민법 §168~§184)는 소멸시효에만 적용되고, 제척기간은
> 원칙적으로 중단이 없다. 기산점(권리를 행사할 수 있는 때 — 민법 §166)·중단사유(청구·압류·승인)·정지가
> 사안별이다. 어느 쪽인지, 중단·정지가 있었는지는 `[검토]`. **개별 특별법의 단기 제척기간·시효**(예:
> 각종 청구권)는 해당 법을 별도 대조 `[검증 필요]`.

## 3단계 — 산정 (날짜가 제공된 경우에만)

**날짜가 없으면 이 단계를 건너뛰고 "기산일 확인 후 산정"으로 표시하고, 사용자에게 필요한 1차 소스를 요청한다:**
판결정본 수령일(송달일), 기일통지서, 처분 통지·수령일, 재결서 정본 송달일 등.

날짜가 제공되면 산정한다. 산정 원칙(모두 현행 조문 대조 전제, 결과는 `[검토]`):
- **기간 계산은 민법·민사소송법의 기간 계산 원칙**(초일 불산입 원칙, 말일이 공휴일·토요일이면 익일 만료 등)을
  따르되, 이 원칙 자체를 국가법령정보에서 대조한다 `[모델 지식 — 검증 필요]`. 초일 산입 여부는 기간 종류에
  따라 다를 수 있어 `[검토]`.
- **불변기간은 법원이 늘리거나 줄일 수 없다**(추후보완만 예외). 따라서 산정된 만료일은 엄격히 적용한다.

| 항목 | 기산일(사용자 확인) | 기간 | 만료일(산정) | 잔여 | 심각도 |
|---|---|---|---|---|---|
| [예: 항소] | [정본 송달 YYYY-MM-DD] | 2주 | [산정] `[검토]` | [산정] | — |

심각도(기한도과는 언제나 🔴 — CLAUDE.md `## 공유 가드레일 → 이중 심각도`):
- 🔴 차단: 기한 경과 또는 D-3 이내
- 🟠 높음: D-7 이내
- 🟡 중간: D-14 이내
- 🟢 낮음: D-14 초과

> **기한 경과 확인 시:** "기한이 이미 경과했을 가능성이 있습니다. 추후보완(민사소송법 §173) 가부는 책임질
> 수 없는 사유가 있었는지에 대한 사안별 판단이며 다툼이 잦습니다 — 변호사 긴급 확인이 필요합니다. `[검토]`"

## 4단계 — 트래커 등록 (선택)

날짜가 확인된 기한만 트래커에 등록한다. `tracker_path`(CLAUDE.md `## 기한 트래커 설정`)의 YAML에 항목 추가.
deadline-watcher 에이전트가 이 파일과 `lead_times_days`를 읽어 알림한다. **날짜 미확인 항목은 등록하지 않고**
`[전자소송/나의사건검색 — 사용자 확인 필요]` 또는 `[기간 계산 — 1차 소스 대조 검증 필요]`로 남긴다.

등록 항목(예시 스키마):
```yaml
- matter: "[매터 slug 또는 practice-level]"
  type: "항소"            # 항소/상고/즉시항고/이의/재심/제소/시효
  case_type: "민사"        # 민사/형사/행정/가사/보전/집행
  basis: "민사소송법 §396"  # 대조 전제
  start_date: "YYYY-MM-DD" # 사용자 확인 기산일(송달일 등)
  start_event: "판결정본 송달"
  period: "2주"
  due_date: "YYYY-MM-DD"   # 산정 — [검토]
  verified: false          # 1차 소스 대조 시 true
```

> 트래커 상태를 narration하지 않는다("등록부에 추가했음..."). 추가하되, 추가했다고 말하지 않는다
> (CLAUDE.md `## 산출물`).

## 데이터 갭 (이 스킬 특유)

- **사건현황·기일·송달현황은 공개 API 없음.** 전자소송(ecfs.scourt.go.kr)·나의사건검색은 국가법령정보
  미커버 — 송달일·기일은 사용자가 기일통지서·판결정본을 paste해야 한다 `[전자소송/나의사건검색 — 사용자 확인 필요]`.

- **인지·송달료·소가는 이 스킬 범위 밖** — 자주 변동, `references/litigation-currency-watch.md` 참조.

> 1. **트래커 등록** — 확인된 기산일로 기한을 트래커에 등록하고 리드타임 알림 설정. *(날짜가 확인된 경우.)*

> 3. **상소 여부 에스컬레이션 메모** — 담당변호사에게 보낼 짧은 메모(원심 결과 + 기한 + 상소 여부 결정 요청). *(패소·일부패소인 경우 필수.)*
> 4. **관찰·대기** — 트래커·워치리스트에 추가. 언제 다시 볼지(다음 기일·기한) 메모와 함께.
