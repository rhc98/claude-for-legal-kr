# /cold-start-interview

1. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 확인 — populate되어 있고 `--redo`가 아니면 덮어쓰기 전 확인.
2. 아래 인터뷰 워크플로우 실행(Part 0 먼저 — 역할·통합 — 그 뒤 분야별): 실무환경·역할·주력 분야, 사건유형별 플레이북, 에스컬레이션 매트릭스, 기한 관리 원칙.
3. 시드 문서: 소장·답변서·준비서면 선례 + 최근 판결정본·기일통지서(10-20건). 읽기.
4. 사건유형별 적용범위 overlay + 에스컬레이션 매트릭스 구축.
5. 마이그레이션: populate된 CLAUDE.md(placeholder 없음)가 `~/.claude/plugins/cache/claude-for-legal-kr/litigation-legal/*/CLAUDE.md`에는 있는데 config 경로에 없다면, config 경로로 복사하고 무엇이 마이그레이션되었는지 사용자에게 보여줌.
6. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 작성(필요시 상위 디렉토리 생성). 요약 보여주기. 첫 작업 제안.

## `--check-integrations`

통합 가용성 점검(문서 저장소·Slack·**국가법령정보 MCP**)을 재실행하고 CLAUDE.md의 `## 연결 가능한 통합` 섹션을 갱신. 재인터뷰는 하지 않음.

점검 시: MCP 도구 호출이 실제로 성공한 경우에만 ✓ 보고. 구성만 되어 있고 미테스트 커넥터는 ⚪로 표시. `.mcp.json` 선언만으로 ✓ 보고하지 않음.

**국가법령정보(korean-law) MCP는 이 플러그인에서 1순위 통합이다.** 미연결이면:
- 모든 조문·판례 인용이 `[모델 지식 — 검증 필요]`로 떨어짐
- 기간 계산 조문(민사소송법 §396·§425, 행정소송법 §20 등) 검증 불가
- 판례 cite_check 불가

사용자에게 안내: "국가법령정보 MCP를 등록하세요: `claude mcp add korean-law "https://mcp.gomdori.app/law?oc=<OC키>" --transport http`. 무료 — open.law.go.kr에서 OC 인증키 발급 필요(1분). 자세한 내용은 `docs/LAW_MCP_INTEGRATION.md` 참조."

# 콜드스타트 인터뷰: 소송·분쟁 실무

## 목적

*이* 소송·분쟁 실무가 어떻게 일하는지 학습 — 실무환경이 어떻게 되는지(사내 송무/법무법인/단독/
나홀로소송), 주력 분야(민사/형사/행정/가사), 역할(소송대리 변호사/사내 법무/나홀로소송 당사자),
무엇을 에스컬레이션하는지, 기한 관리 원칙이 무엇인지.
`~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md`에 써서 다른 모든 스킬이
같은 이해에서 읽도록.

**한국 소송법은 전국 단일 관할이며 사건유형 × 심급이 핵심 변수다.** 미국 원본은 "어느 주냐"를
먼저 묻지만, 한국에서 가장 먼저 알아낼 것은 **사건유형(민사/형사/행정/가사)과 우리 입장(원고/피고)**이다
— 적용 절차·기한·서면·불복방법이 이 조합에 따라 달라진다. 인터뷰는 이 footprint를 다른 어떤 것보다 먼저 잡는다.

`~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 읽기:

- **populate됨 (placeholder도 pause comment도 없음)** → 이미 구성됨; `--redo`가 아니면 skip.

## 인터뷰

### Part 0: 누가 쓰고, 무엇이 연결되어 있나

#### 누가 쓰고 있나?

> 1. **소송대리 변호사** — 변호사 자격으로 직접 소송을 수행하는 변호사, 법무법인 어소시에이트, 외부 로펌 변호사.
> 2. **사내 법무(직접 소송대리 제한)** — 인하우스 법무팀, 직접 소송대리는 제한되지만 외부 변호사를 지시·감독하는 역할.
> 3. **나홀로소송(본인소송) 당사자** — 변호사 없이 본인이 직접 소송을 수행하는 당사자.
> 4. **기타** — 공인 자격 없는 실무자로 변호사 자문 접근 가능.

> **※ 소송대리에는 원칙적으로 변호사 자격이 필요하다(민사소송법 §87).** 단독사건 등 법원 허가의 경우 예외(§88). 나홀로소송은 당사자 본인만 가능.

답이 3 또는 4면, 한 번만:

> 모든 기능을 쓸 수 있습니다. 두 가지가 달라집니다:
> 1. **출력을 변호사 검토용 리서치로 framing합니다** — "제출해도 됨" 대신 "발견한 것과 제출 전 확인할 것".
> 2. **법적 결과가 있는 단계 전에 멈춥니다** — 소장 제출, 답변서 발송, 상소장 제출, 화해 수락. 변호사 검토를 거쳤는지 묻고 brief를 만들어드립니다.

답이 3(나홀로소송)이면 추가:

> **소송구조(민사소송법 §128) `[모델 지식 — 검증 필요]`** — 소송비용을 지출할 자력이 부족한 당사자에 대한 재판상 구조. **대한법률구조공단** — 법률상담·소송대리·서류작성 지원(자격·소득 요건). Claude는 소송대리를 대신하지 않습니다.

#### 실무 환경

> - **사내 송무 (법무팀·송무팀)** — 에스컬레이션은 GC·법무팀장·사업부 방향. 예산·비용 통제 중심.
> - **법무법인·로펌 (대·중·소형)** — 파트너 결재 체인, 의뢰인 보고, 매터별 격리.
> - **개인·단독 사무소** — 계층 없음. 에스컬레이션은 "외부 전문가 자문 시점"으로 매핑.
> - **나홀로소송 당사자** — 법무 체계 없음. 기한 추적·서면 초안 지원 중심.

#### 무엇이 연결되어 있나?

국가법령정보 MCP 테스트:
```
mcp__korean-law__search_law "민사소송법"
```
응답에 결과 1건 이상이면 ✓. 연결 오류·빈 배열이면 ✗.

발견을 다음 형식으로 보고:
> - ✓ [통합] — 연결됨 (테스트됨)
> - ⚪ [통합] — 구성됨이나 검증 안 됨
> - ✗ [통합] — 발견 안 됨. [기능]은 [수동 대안]으로 fallback. [연결 방법].
>
> **국가법령정보 MCP가 없으면 조문·판례 인용 검증이 모두 `[모델 지식 — 검증 필요]`로 떨어지고 기간 계산 조문 대조가 불가합니다.**

### Part 1: footprint — 사건유형 × 심급 × 입장

> **[사무소·법무팀]은 주로 어떤 사건을 다루나요?**
> 사건유형과 우리 입장이 모든 스킬의 출발점입니다.

없으면 하나씩:

- **주력 사건유형:** 민사(원고/피고) | 형사(고소·고발/피의자·피고인) | 행정(항고소송) | 가사 | 보전 | 집행 | 조정·중재 | 복합.
- **우리 측 기본 입장:** 원고·신청인(청구·집행) 중심 / 피고·피신청인(방어) 중심 / 양쪽.
- **주된 관할 법원:** 예: 서울중앙지방법원, 행정법원, 가정법원.
- **외부 대리인 사용 여부:** 항상 선임 / 대규모만 / 직접 수행(사내·나홀로).
- **전자소송 이용 여부:** 이용 / 미이용. `[전자소송/나의사건검색 — 사용자 확인 필요]` — 기일·송달 확인은 사용자가 직접.

**사건유형별 footprint.** 각 유형에 대해:

- **민사:** 소가 규모 대략, 청구 유형(금전·부동산·이행청구 등), 주요 반복 상대방.
- **형사:** 고소·고발 중심 vs 피의자·피고인 방어 중심, 범죄 유형.
- **행정:** 주력 처분 유형(허가취소·과징금·영업정지 등), 행정심판 전치 여부 빈도.
- **가사:** 이혼·재산분할·양육 vs 상속·친권 등.
- **보전·집행:** 가압류·가처분 주도 vs 피신청 방어.

### Part 2: 검토 트리거 + 에스컬레이션 (3-4분)

> "**에스컬레이션 임계점을 지금 구축하시겠어요?** 스킬이 훨씬 나아집니다 — 일반 임계점 대신 당신 실무의 실제 선을 압니다."

없으면:

**어떤 사건이 에스컬레이션을 자동으로 트리거하나요?**
- 소가 금액 임계점?
- 형사 피소·구속·압수수색 시 자동?
- 불변기간(상소기간·즉시항고·행정소송 제소기간) 임박 시 자동?
- 상소 여부 결정 시?
- 가압류·가처분 신청·피신청 시?

**기한 관리 원칙:**

> "소송에서 기한 도과는 복구 불가능한 권리 상실입니다. 어떻게 추적하고 있나요?"
> - 캘린더 도구·판례관리시스템·수동 관리?
> - 기일통지서·판결정본은 누가 수령·기록?
> - 불변기간 임박 알림 리드타임은?

**하우스 스타일:**
- 검토·메모 톤(간결·상세)?
- 의뢰인·경영진 보고 방식?
- 서면 선례 포맷 선호?

### Part 3: 시드 문서 (3-4분)

> **가장 중요한 부분 — 정책이 아니라 팀이 실제로 어떻게 일하는지 보고 싶습니다.**
>
> 1. **최근 소장·답변서·준비서면 선례.** 현재 포맷. 하우스 서면 스타일을 읽습니다.
> 2. **판결정본·기일통지서.** 기일·기한 기록 방식을 파악합니다.
> 3. **최근 사건 메모·브리핑 (있으면).** 10건이 좋은 하한.

이 문서들에서:
- 소장·답변서: 청구취지·원인 서술 방식, 증거목록 포맷.
- 준비서면: 쟁점 정리 방식, 인용 스타일.
- 판결정본: 판단 방식, 인용 판례.
- 기일통지서: 기일 기록 방식, 다음 기일 예정.

## 데이터 갭 — 침묵의 보완 금지 (세 값)

인터뷰 중 조문·기간·판례 유효성·사건현황이 부족할 때 답은 세 개다:

1. **플래그와 함께 보완.** 모델 지식·국가법령정보에서 가져오되 항목에 태그(`[모델 지식 — 검증 필요]`)를 붙이고 진행.
2. **말하지 않고 멈춤.** 1차 소스(취업규칙 유사 — 소장·준비서면 선례·기일통지서)를 붙여 달라거나 가리켜 달라고 부탁하고, 줄 때까지 진행하지 않음.
3. **플래그만 — 사용하지 않음.** 분석을 바꾸지 않지만 독자가 알아야 하는 정보(개정 검토 중 조문, 전원합의체 계속)를 `[모델 지식 — 검증 필요]` 태그로 surface하되, 그것으로 분석을 바꾸지 않음.

이미 알고 있는 의심을 침묵하는 것은 확신에 찬 단언만큼이나 오도한다.

## 실패 모드

- **사건유형·심급을 가정하지 마라.** 확정되지 않으면 묻고, 조합이 모든 후속 분석의 게이트다.
- **임의해고(at-will) 유사 개념 없음.** 소송에는 "그냥 소취하하면 된다"가 없다 — 상대방 동의·기한·비용까지 점검.
- **일반 입장에서 플레이북을 쓰지 마라.** 경험이 적으면 config에 명시: `[POSITIONS UNTESTED — 이 실무는 아직 충분한 사건을 다루지 않음. 출발점이지 settled 입장이 아니라고 취급.]`
- **하급심 판결은 국가법령정보 미커버라고 명시.** `[하급심 판결 — 미공개, 직접 확인 필요]` 태그.
- **외국 준거법·외국 당사자를 한국 절차법으로 단정하지 마라.** `[외국법·국제사법 — 외국 변호사 검증 필요]` 플래그.
- **기간 계산을 기억으로 하드코딩하지 마라.** 기산점·불변기간·추후보완 가부는 사안별 — `[기간 계산 — 1차 소스 대조 검증 필요]`.
