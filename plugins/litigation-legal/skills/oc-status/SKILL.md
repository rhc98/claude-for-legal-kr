# /oc-status

> **한국 소송 맥락 — 용어 정리:**
> - **OC(opposing counsel, 상대방 대리인):** 이 스킬에서는 **상대방 측 대리 변호사**를 의미.
> - **우리 측 외부 대리인:** matter.md의 `외부 대리인(OC)` 필드로 별도 관리.
> - **협상·화해:** 재판상화해(민사소송법 §220)·조정·당사자 간 합의. 한국에서 소송 중
>   화해는 일반적이며, 조정전치주의(가사·소액)도 있음.

1. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 로드 →
   실무 프로파일, 에스컬레이션 체인, 하우스 스타일.
2. 아래 워크플로우와 참조표를 따른다.
3. 플래그 파싱:
   - `--all` — 활성 전체 매터
   - `--slug=<slug>` — 특정 매터
   - `--no-email` — 이메일 초안 생략
4. `matters/_log.yaml`에서 대상 매터 필터링.
5. 각 매터의 `matter.md`·`history.md` 로드 → 상대방 대리인 정보·협상 현황 추출.
6. 상대방 대리인 현황 롤업 + 매터별 상세.
7. 필요 시 통신 초안 생성.

## OC 상태 트래킹 워크플로우

### 1단계 — 상대방 대리인 정보 추출

각 매터의 `matter.md`에서:
- **상대방:** 상대방 당사자 전원.
- **상대방 대리인(opposing counsel):** 대리 변호사·사무소명. 미선임이면 `본인소송` 또는 `미확인`.
- **마지막 접촉:** 최근 통신·서면·기일에서의 상대방 대리인 발언·입장.
- **협상·화해 상태:** 진행 중·미시작·성립·결렬.

`history.md`에서 상대방 관련 이벤트 추출:
- 상대방 서면 수령·기일 발언·협상 제안·화해 시도.

### 2단계 — 협상·화해 상태 분류

| 상태 | 설명 | 주의 사항 |
|---|---|---|
| **미시작** | 협상 시도 없음 | — |
| **탐색 중** | 비공식 접촉·화해 타진 | 발언이 증거로 쓰일 수 있음 `[검토]` |
| **제안 있음** | 한쪽 또는 양쪽 제안 | 수락·거절·역제안 결정 필요 `[검토]` |
| **협상 진행** | 적극 협상 중 | 에스컬레이션 임계점 확인 |
| **보류** | 협상 일시 중단 | 재개 조건 확인 |
| **성립** | 화해·합의 성립 | 화해조서·합의서 확인, `/matter-close` 준비 |
| **결렬** | 협상 결렬 | 소송 계속, 전략 재검토 |

> **한국 소송에서 화해 발언의 증거 사용.** 화해 교섭 중 발언이 불리한 증거로 쓰일 수 있는지는
> `[검토]` — 민사소송법 §149(증언거부) 등과 연결해 검토 필요. 미국의 FRE 408과 같은 명시적
> 규정이 한국에는 없음 `[모델 지식 — 검증 필요]`.

### 3단계 — 통신 이력 요약

각 매터별 상대방 대리인과의 주요 통신:
- **발신:** 우리 측 내용증명·서면·이메일 발송 이력.
- **수신:** 상대방 서면·내용증명·이메일 수령 이력.
- **기일:** 기일에서의 상대방 주요 주장·입장.
- **비공개 접촉:** 전화·면담 등 비공개 협의 이력.

### 4단계 — 다음 단계 결정 지원

**협상 관련 결정은 모두 `[검토]`** — 화해 수락·거절·역제안, 조정 신청 여부, 강제 조정 대응.

| 상황 | 검토 포인트 |
|---|---|
| 상대방 화해 제안 수령 | 제안 조건 분석, 수락·거절·역제안 결정 `[검토]` |
| 협상 결렬 후 기일 접근 | 입장 강화, 증거 보완 포인트 |
| 상대방 대리인 변경 | 새 대리인 파악, 전략 재검토 |
| 본인소송 상대방 | 조정 전치 여부, 나홀로소송 당사자 특성 고려 |

## 통신 초안 (선택)

`--no-email` 없으면: 상대방 대리인에게 발송할 협의 요청·화해 타진 이메일 초안 생성.

> **발송 전 검토 필수.**
> "상대방 대리인에게 보낼 통신 초안입니다. 발송 전 아래를 확인하세요:
> 1. 협상 발언이 향후 불리한 증거로 쓰일 수 있는지 `[검토]`.
> 2. 에스컬레이션 임계점(실무 프로파일 참조) 확인 — 화해 금액·조건 권한 범위 내인지.
> 3. 외부 대리인(우리 측 OC)이 있는 경우 발송 전 OC 검토.
> 미검토 통신은 이론·전략·양보를 의도치 않게 기록에 남길 수 있습니다."

통신 초안 형식:
```
수신: [상대방 대리인 성명·이메일]
발신: [우리 측 담당자]
제목: [사건명] — [목적]
날짜: [YYYY-MM-DD]

[본문]

---
※ 이 통신은 분쟁 해결 목적의 화해 교섭 과정에서 작성되었습니다. [검토 — 증거 사용 가능성 변호사 확인]
```

## 이상 플래그

자동 플래그:
- **협상 제안 수령 후 14일 이상 미응답** — 결정 필요.
- **화해 성립 + matter-close 미실행** — `/litigation-legal:matter-close` 실행 권장.
- **상대방 대리인 미확인 + 고위험 사건** — 대리인 정보 확보 필요.
- **본인소송 상대방 + 형사 또는 보전 사건** — 절차 주의 `[검토]`.

## 이 스킬이 하지 않는 것

- **협상 조건 결정.** 수락·거절·역제안은 변호사 판단 `[검토]`.
- **화해 발언의 증거 사용 가능성 단정.** 한국법상 명시 규정 없음 `[모델 지식 — 검증 필요]`.
- **상대방 대리인 직접 조회.** 대리인 정보는 사용자 입력 기반.
- **통신 미검토 발송.** 초안은 검토 후 발송 — 발송 자체는 사용자 결정.
