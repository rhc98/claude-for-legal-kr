# /matter-briefing

1. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 로드 →
   사건유형별 플레이북, 리스크 캘리브레이션, 에스컬레이션 체인.
2. 아래 워크플로우와 참조표를 따른다.
3. `$ARGUMENTS`에서 slug 추출 — 없으면 활성 매터 사용(없으면 묻는다).
4. `matters/<slug>/matter.md` + `matters/<slug>/history.md` 로드.
5. 브리핑 산출물 생성.

> **매터가 인테이크되지 않은 경우:**
> "[slug]를 매터 로그에서 찾을 수 없습니다. `/litigation-legal:matter-intake`를 먼저 실행해
> 이해상충 점검과 매터 워크스페이스 설정을 완료하세요. 인테이크되지 않은 사건의 브리핑은
> 만들지 않습니다 — 이해상충 점검이 게이트입니다."

## 브리핑 워크플로우

### 1단계 — 매터 로드 및 coverage 확인

`matter.md`·`history.md`를 읽고 검토자 메모 **읽기 범위:**에 coverage 기록. 파일이 없거나
업데이트가 30일 이상 오래된 경우 플래그: "최근 업데이트 없음 — history.md가 [마지막 날짜] 이후
갱신되지 않았습니다. 현황 업데이트가 필요하면 `/litigation-legal:matter-update`를 실행하세요."

### 2단계 — 사건유형·심급 확인

`matter.md`의 사건유형·심급 확정. **침묵으로 가정하지 않는다.** 미확정이면 브리핑 전에 확인.
심급에 따라 쟁점 분석 방향이 다르다:

| 심급 | 브리핑 초점 |
|---|---|
| 1심 | 요건사실·증거·변론주의, 소명·증거 갭, 기일 일정 |
| 항소심 | 원심 판단의 사실오인·법령위반, 항소이유(민사소송법 §394) `[검증 필요]`, 불복 범위 |
| 상고심 | **법률심** — 사실주장 제한, 상고이유(§423~) `[검증 필요]`, 심리불속행 위험 |
| 수사(형사) | 피의자 방어, 증거법칙, 구속·보석 현황, 기소 여부 전망 |
| 보전 | 피보전권리·보전의 필요성 소명 상태, 담보 현황, 본안 연계 |
| 집행 | 집행권원 확보, 재산조사 현황, 청구이의·제3자이의 위험 |

### 3단계 — 쟁점·요건사실 정리

사건유형에 따른 핵심 쟁점:

**민사(원고):** 청구원인의 요건사실 특정 — 어느 요건을 입증할 수 있는가, 어디에 갭이 있는가.
입증계획(서증·증인·감정)과 현재 상태.

**민사(피고):** 원고 청구원인에 대한 항변 — 소멸시효·상계·동시이행·권리남용·계약해석.
입증책임 분배(`[검토]`) + 반소·독립당사자참가 검토.

**형사(피의자·피고인):** 구성요건해당성·위법성·책임 요건 분석. 증거법칙(위법수집증거·자백보강).
양형 인자(`[검토]`). 자백·진술 관련은 **특히 신중** — 진술은 절대 강제하지 않는다.

**행정(항고소송):** 처분성·원고적격·협의의 소익·제소기간 적법성. 처분의 위법사유(비례원칙·
재량일탈·절차위법 등). 집행정지 검토.

**가사:** 조정전치 경과, 재산분할·양육·친권 쟁점, 사전처분·이행명령 현황.

**보전·집행:** 피보전권리·보전의 필요성·소명도(보전), 집행권원 유효성·재산 현황(집행).

**판례 인용 시:**
```
mcp__korean-law__legal_analysis {"mode": "cite_check", "citation": "대법원 YYYY. M. D. 선고 YYYY다NNNNN 판결"}
```
→ `[선례 유효성 — cite_check 확인]` 플래그.

### 4단계 — 증거·기록 현황

- **제출·확보된 증거:** 서증·증인·감정 목록 및 현황.
- **미확보·필요 증거:** 갭 및 확보 방법.
- **상대방 제출 증거:** 반박 포인트.
- **하급심 판결 미공개:** 인용이 필요한 하급심 판결이 국가법령정보 미공개인 경우 → `[하급심 판결 — 미공개, 직접 확인 필요]`.

### 5단계 — 기일·기한 현황

**불변기간은 최우선 점검.**

```
# 조문 검증 예시
mcp__korean-law__get_law_text "민사소송법 제396조"
```

| 기한 유형 | 날짜 | 기산점 | 상태 | 태그 |
|---|---|---|---|---|
| 다음 기일 | [날짜] | — | `[전자소송/나의사건검색 — 사용자 확인 필요]` | — |
| 상소기간 2주 | [판결정본 송달일 + 2주] | 송달일 | `[기간 계산 — 1차 소스 대조 검증 필요]` | 불변기간 |
| 즉시항고 1주 | [고지일 + 1주] | 고지일 | `[기간 계산 — 1차 소스 대조 검증 필요]` | 불변기간 |
| 행정소송 90일 | [처분 안 날 + 90일] | 처분 안 날 | `[기간 계산 — 1차 소스 대조 검증 필요]` | 불변기간 |

> **날짜가 없으면 기간을 산정하지 않는다.** 공식과 필요한 정보(송달일·정본 수령일)를 제시하고
> 사용자에게 확인 요청. `[전자소송/나의사건검색 — 사용자 확인 필요]` — 기일통지서·판결정본 paste 요청.

### 6단계 — 리스크 재평가

실무 프로파일의 **이중 심각도** 기준 적용:

- **법적 위험:** 🔴 기한도과·패소·권리소멸 / 🟠 불리한 심증·증거배제 / 🟡 절차지연·비용 / 🟢 경미
- **분쟁·관계 마찰:** 🔴 관계파탄·확전 / 🟠 협상여지 축소 / 🟡 비용·시간 / 🟢 비가시

**기한도과는 언제나 🔴.** 업스트림 심각도 바닥(floor) 유지.

### 7단계 — 미해결 쟁점·질문

다음 단계를 결정하기 전에 알아야 할 것:
- 확보 안 된 증거·서류
- 의뢰인에게 확인해야 할 사실
- 상대방 대리인과 협의 필요한 절차
- 에스컬레이션 필요 여부

## 데이터 갭 — 침묵의 보완 금지 (세 값)

조문·기간·기산점·판례 유효성·사건현황·기일이 부족할 때 답은 세 개다:

1. **플래그와 함께 보완.** 모델 지식·다른 소스에서 가져오되 항목에 태그(`[모델 지식 — 검증 필요]`)를 붙이고 진행.
2. **말하지 않고 멈춤.** 1차 소스(판결정본·기일통지서·소장·준비서면)를 붙여 달라거나 가리켜 달라고 부탁하고, 줄 때까지 진행하지 않음.
3. **플래그만 — 사용하지 않음.** 분석을 바꾸지 않지만 독자가 알아야 하는 정보(계류 중 전원합의체 심리, 개정 검토)를 `[모델 지식 — 검증 필요]` 태그로 caveat에 surface하되, 그것으로 분석을 바꾸지 않음.

이미 알고 있는 의심을 침묵하는 것은 확신에 찬 단언만큼이나 오도한다.
