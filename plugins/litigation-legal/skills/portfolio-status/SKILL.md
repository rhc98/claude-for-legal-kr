# /portfolio-status

1. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 로드 →
   리스크 캘리브레이션(`risk:` 필드 해석 방법).
2. 아래 워크플로우와 참조표를 따른다.
3. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/matters/_log.yaml` 파싱.
   기본값으로 종결 매터 제외(포함은 `--all`).
4. 롤업 생성: 리스크 분포, 14/30/60일 내 기한·기일, 30일 이상 미갱신 사건, 소가 합계, 심급 분포.
5. 이상 플래그 — critical 표시 전체, `next_deadline` 도과 사건, 중·고위험인데 OC 미선임 사건.

---

## 플래그

- `--all` — 종결 포함
- `--risk=high` (또는 `critical` / `medium` / `low`) — 리스크 등급 필터
- `--stale` — `last_updated` > 30일 사건만
- `--type=민사` (또는 형사·행정·가사·보전·집행·조정·중재·상소심) — 사건유형 필터
- `--stage=항소심` (또는 1심·상고심·수사·보전·집행) — 심급 필터
- `--owner=[이름]` — 내부 담당자 필터

---

## 포트폴리오 롤업 워크플로우

### 1단계 — `_log.yaml` 파싱

`_log.yaml`을 읽고 각 행에서:
- `slug`, `name`, `type`, `stage`, `status`, `risk`, `next_deadline`, `last_updated`, `materiality`, `oc`, `owner`, `court`, `opened` 추출.
- `status: closed` 행은 `--all` 없으면 제외.

`_log.yaml`이 없거나 비어 있으면:
> "매터 로그(`_log.yaml`)가 없거나 비어 있습니다. `/litigation-legal:matter-intake`로
> 사건을 먼저 등록하세요."

### 2단계 — 기한 현황 분류

> **기한도과는 언제나 🔴.** `next_deadline` 필드가 있는 사건의 날짜를 오늘과 비교.

| 구분 | 기준 | 표시 |
|---|---|---|
| 도과 | `next_deadline` < 오늘 | 🔴 **도과** |
| 7일 이내 | `next_deadline` ≤ 오늘 + 7 | 🔴 **긴급** |
| 14일 이내 | `next_deadline` ≤ 오늘 + 14 | 🟠 임박 |
| 30일 이내 | `next_deadline` ≤ 오늘 + 30 | 🟡 주의 |
| 30일 초과 | — | 🟢 여유 |
| next_deadline 없음 | null | ⚪ 기일 미등록 `[전자소송/나의사건검색 — 사용자 확인 필요]` |

> **기한 태그:** 모든 `next_deadline` 날짜는 `[기간 계산 — 1차 소스 대조 검증 필요]`. 기산점·
> 불변기간 여부는 각 사건의 matter.md 및 기일통지서·판결정본 확인 필요.

### 3단계 — 리스크 분포

실무 프로파일 리스크 캘리브레이션 기준으로 분류:

```
🔴 critical/high: [N]건
🟠 high:          [N]건  
🟡 medium:        [N]건
🟢 low:           [N]건
```

### 5단계 — 장기 미갱신 사건

`last_updated` > 30일 사건 목록. 사건이 아직 진행 중인지 확인 필요.

| Slug | 사건명 | 마지막 갱신 | 경과일 | 담당자 |
|---|---|---|---|---|

### 6단계 — 소가 합계

- **전체 소가 합계:** `materiality` 필드 합산 (비산정 항목 제외).
- **리스크 등급별 소가:** critical·high·medium·low 각각.
- **소가 주의:** 소가 산정 기준은 민사소송등인지법 현행 고시 대조 필요 `[검증 필요]`. 산정된
  소가는 사용자 입력값 기반 — 법원 결정 소가와 다를 수 있음.

### 7단계 — 이상 플래그

자동 플래그 항목:

1. **불변기간 7일 이내 임박** — 🔴 자동 에스컬레이션 대상.
2. **`next_deadline` 도과** — 기일·기한이 지난 사건. 종결 또는 갱신 필요.
3. **중·고위험 + OC 미선임** — 외부 대리인 선임 여부 확인 필요.
4. **30일 이상 미갱신 + 활성** — 현황 업데이트 필요.
5. **사건유형·심급 미확정** — `type` 또는 `stage`가 null인 사건.
6. **소가 비산정 + 고위험** — 소가 확인 필요.

## 이 스킬이 하지 않는 것

- **기간 계산 단정.** `next_deadline`은 사용자 입력값 — 기산점·불변기간 여부는 각 사건 확인 필요.
- **전자소송·사건검색 기일 조회.** 공개 API 없음 → `[전자소송/나의사건검색 — 사용자 확인 필요]`.
- **소가 법원 결정값 보장.** 산정값은 사용자 입력 기반 `[검증 필요]`.
- **하급심 판결 미공개 내용 인용.** → `[하급심 판결 — 미공개, 직접 확인 필요]`.
