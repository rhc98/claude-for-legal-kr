## 목적지 점검

내용증명·최고서는 **상대방에게 발송되는 외부 문서**다. work-product 헤더 미부착 — 이 스킬의
가장 중요한 목적지 규칙. 발송 전 승인 매트릭스(에스컬레이션 체인)와 아래 법적 위험을 확인한다.

발송이 초래할 수 있는 위험:
- **역공·반발** — 상대방이 반대 주장을 선제 제기하거나 채무부존재확인의 소 제기
- **시효 중단 효과 제한** — 이행최고만으로는 6개월 이내 소 제기 등 후속조치 없으면 시효 중단 효과 없음 `[검토]`
- **채무 승인 위험** — 내용증명 문언이 우리 측 채무를 묵시적으로 인정하지 않는지 확인

정본: 실무 수준 CLAUDE.md `## 공유 가드레일 → 목적지 점검`.

## 목적

완성된 인테이크(`demand-intake` 산출물)를 받아:
- 발송 전 체크리스트(특권·어드미션·시효 중단 요건) 게이트를 통과하고,
- 유형별 한국식 최고서·내용증명 초안을 작성하며,
- 발송 후 체크리스트를 작성하고,
- 매터 생성 여부를 제안한다.

**이 게이트는 초안 작성 전에 실행한다. 사용자가 각 항목에 응답하지 않으면 진행하지 않는다.**

```
발송 전 체크리스트 — [slug]

1. 대외비 필터 (특권 필터)
   인테이크의 특권 필터: [목록]
   이 중 어느 것도 초안에 나타나지 않음을 확인하셨습니까?  [예/아니오]

2. 어드미션 위험 (채무 승인·화해 언급 위험)
   인테이크의 어드미션 위험: [목록]
   각 항목의 문언이 통제·제거되었습니까?  [항목별 예/아니오]

3. 화해 언급 위험
   인테이크: [플래그된 위험, 있으면]
   이 최고서가 별도 청구를 묵시적으로 수락하거나 상계처럼 읽히지 않습니까?  [예/아니오]

4. 시효 중단 요건 확인 `[모델 지식 — 검증 필요]`
   - 이행최고(민법 §174): 최고일부터 6개월 이내 소 제기·가압류·가처분 등 후속조치 계획이 있습니까?
   - 시효중단 효과를 원하는 경우 후속조치 일정을 확인하십시오.
   인테이크: [시효 중단 목적 여부]
   확인.  [예/아니오]

5. 특권 노출 스캔
   초안 중 어느 문장이 우리 측 내부 법률 분석(결론이 아닌 분석 내용)을 드러냅니까?  [예/아니오]
   예이면 — 초안 작성 전 재표현.

6. 톤 포스처
   인테이크: [관계유지형 / 표준형 / 강경형]
   이것이 동사 선택·프레이밍·결과 언어를 결정합니다. 확인.  [예/아니오]

7. 사실 정확성
   초안의 모든 사실은 확인된 것이어야 한다. "아마 맞을 것"이 아닌 — 확인됨.
   아직 확인되지 않은 사실이 있으면 나열하세요 — `[검증: ___]` 인라인 플래그로 처리됩니다.

## 유형 선택

인테이크의 `최고 유형`을 읽는다. 시드 문서(업로드된 선례 또는 과거 내용증명)가 있으면 그것을
템플릿으로 사용 — 소프트 템플릿보다 시드 문서가 우선.

### 소프트 템플릿 (시드 문서 없을 때만)

**지급 최고·내용증명 (금전채무 이행최고):**
1. 당사자 및 관계 (1 문단)
2. 청구권원 — 계약 조항 / 청구서 / 발생 사실, 날짜
3. 채무불이행 — 미지급 금액, 기한, 경위
4. 이행 청구 — 구체적 금액, 기한, 지급 방법
5. 불이행 시 결과 — 법적 조치(지급명령·소 제기), 이자(법정이율·약정이율), 비용
6. 시효 중단 최고 문언 (해당 시 — 민법 §174) `[모델 지식 — 검증 필요]`
7. 내용증명 발송 사실 및 서명·날인

**채무불이행 최고·이행 최고 (계약 이행 / 치유 기간):**
1. 당사자 및 계약 특정
2. 채무불이행 사실 — 조항, 위반 행위, 날짜, 증거
3. 이행 청구 및 치유 기간 — 구체적 이행 내용, 기한 (민법 §387 `[모델 지식 — 검증 필요]`)
4. 기한 도과 시 결과 — 계약해제·해지(민법 §544 `[모델 지식 — 검증 필요]`), 손해배상 청구
5. 서명·날인

**계약해지 최고·통보:**
1. 당사자 및 계약 특정
2. 해지 사유 — 귀책 사유·조항
3. 해지의 의사표시 및 효력 발생 시점
4. 잔여 의무 처리 (원상회복·손해배상)
5. 서명·날인

**시효 중단 최고 (순수 시효 중단 목적):**
1. 당사자 특정
2. 청구권 특정 — 발생원인, 금액 또는 이행 내용
3. 시효 중단을 위한 최고임을 명시 (민법 §174) `[모델 지식 — 검증 필요]`
4. 6개월 이내 후속조치(소 제기 등) 예정 사실
5. 서명·날인

**지식재산·영업비밀·부정경쟁 중단 요구 (C&D형):**
1. 당사자 및 우리 측 권리 특정
2. 침해 행위 — 구체적 행위·날짜·증거
3. 중단 요구 — 즉시 중단, 삭제, 사용 내역 제출, 서면 확인
4. 이행 기한
5. 불이행 시 결과 — 소 제기, 가처분 신청, 법정 손해배상
6. 증거 보전 요청 (관련 문서·데이터·시스템)
7. 서명·날인

## 작성 규칙

0. **지급명령 안내.** 인테이크의 청구 유형이 금전 지급인 경우, 내용증명 발송과 병행하여
   지급명령(민사소송법 §462 이하 `[모델 지식 — 검증 필요]`) 신청을 고려할 수 있음을 사용자에게
   안내한다. 지급명령은 채무자가 이의신청(2주, 불변기간)하지 않으면 확정판결과 동일한 효력.
   `[기간 계산 — 1차 소스 대조 검증 필요]`

1. **구체성이 수사보다 중요하다.** "2026년 3월 14일 귀사는 X를 발송하였습니다"가
   "귀사는 반복적으로 부당하게 X를 발송하였습니다"보다 낫다. 수사어구는 사실이 빈약하다는 신호.

2. **확인된 사실만.** 초안의 모든 사실은 소스에 추적 가능해야 한다. 확인되지 않은 사실은
   `[검증: 구체적 내용]` 인라인 플래그.

3. **법적 인용은 플레이스홀더.** `[인용: 조문/조항/판례]` 형태로. 인용을 발명하지 않는다.
   인테이크에서 제공된 근거가 있으면 그대로 사용.
   국가법령정보로 조문 텍스트를 가져와 확인 — 모델 지식에만 의존하지 않는다.

4. **결과 언어는 톤 포스처에 맞춘다.**
   - `관계유지형`: "상호 원만한 해결을 기대합니다."
   - `표준형`: "[N]일 이내 이행하지 않으실 경우 법적 조치를 검토하겠습니다."
   - `강경형`: "[N]일 이내 이행하지 않으실 경우 즉시 법적 조치(소 제기·가압류 등)를 취하겠습니다."

5. **인라인 대안 표현.** 톤이 달라질 수 있는 곳은 컴팩트한 대안을 제시:
   > *첨부 청구서의 [금액]이 미지급 상태입니다.* [또는 더 강경하게: *귀사는 기한 내 지급 의무를 이행하지 않았습니다.*]

6. **내용증명 형식 주의.** 내용증명은 우편법시행규칙상 요건을 충족해야 한다 `[모델 지식 — 검증 필요]`.
   3통 작성(발송인·수신인·우체국 각 1통), 작성자 기명·날인. 법인인 경우 대표자 날인.

7. **시효 중단 문언 정밀도.** 시효 중단을 목적으로 하는 최고는 민법 §174에 따라 6개월 이내에
   소 제기·가압류·가처분·승인 등 후속조치가 없으면 시효 중단 효력이 없다 `[모델 지식 — 검증 필요]`.
   최고서 본문에 이 사실을 명시하고, 발송 후 체크리스트에 후속조치 기한을 등재한다.

8. **인용 출처 태그.** 모든 법령 인용에 출처 태그 부착. `[국가법령정보]` / `[모델 지식 — 검증 필요]`.
   판례 인용 시 `cite_check`:
   ```
   mcp__korean-law__legal_analysis {"mode": "cite_check", "citation": "대법원 [사건번호]"}
   ```

### 발송 게이트 (초안 끝에)

> **⚠️ 발송 전 확인**
> 이 문서는 법적 효과를 갖는 외부 문서입니다. 발송은 시효 중단·계약해지·분쟁 확전 등 법적
> 결과를 초래합니다. 발송 전 담당 변호사의 검토를 받으셨습니까?
