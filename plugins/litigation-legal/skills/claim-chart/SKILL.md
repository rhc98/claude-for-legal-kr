# /claim-chart

1. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md`를 읽는다 → 역할, 산출물 헤더,
   결정 자세, 문서 저장 위치.
2. 매터 워크스페이스가 활성이면 활성 매터 확인·선택; `matter.md` 로드(입장, 사건유형, 심급, 사건 이론).
3. 아래 워크플로우와 레퍼런스를 따른다.
4. **입장 확인** — 원고(청구원인 구성) / 피고(항변 구성). 요건사실과 증명책임 분배가 입장에 따라 다르다.
5. 청구원인(또는 항변)을 요건사실로 분해. 각 요건사실을 증거·증명책임 주체·상대방 항변에 매핑. 셀마다
   증거 핀포인트. `=`·`+`·`-`·`@`·탭·CR로 시작하는 셀 값은 앞에 어퍼스트로피(`'`)를 붙여 중화(CSV·엑셀 주입 방지).
6. **증거 공백(gap) 목록 산출** — 최우선 산출물.
7. 마크다운, CSV(값 + `_sources` 동반 파일), 엑셀·시트를 사용자 선호에 따라. 내부 검토용이면 산출물 헤더.
8. 매터 활성이면 매터의 `claim-charts/` 폴더에, 아니면 실무 수준 `claim-charts/` 폴더에 쓴다. 매터 활성이면
   `history.md`에 한 줄 추가.
9. 요약 readout 반환: 청구원인/항변, 입장, 사건유형, 심급, 요건사실 수(상태별), 증거 공백 목록, 파일 경로,
   그리고 "모든 셀은 검증 대상"이라는 리마인더.

## 이것이 무엇인가 — 요건사실 정리표

**이것은 요건사실 정리표(입증계획 도구)지 승소 예측이 아니다.** 한국 민사소송에서 승패는 요건사실이
주장·증명되었는가에 달려 있다(변론주의·증명책임 분배). 이 표는:

- **청구원인의 요건사실**을 빠짐없이 분해하고,
- 각 요건사실의 **증명책임을 누가 지는지**(원고/피고) 표시하고,
- 각 요건사실에 **대응 증거**(서증·인증·간접사실)를 매핑하고,
- 상대방의 **예상 항변·부인**을 붙이고,
- 무엇이 빠졌는지(**증거 공백**) surface한다.

**이것은 특허 claim chart가 아니다.** 청구항 대비·균등론·구성요소 완비는 이 스킬의 대상이 아니다.
특허·상표·디자인 침해 대비는 ip-legal 플러그인의 `/ip-legal:infringement-triage`·`claim-chart` 소관이다.
사용자가 특허 침해 대비표를 원하면 말한다: "특허 claim chart는 ip-legal 플러그인 소관입니다 —
`/ip-legal:` 스킬을 쓰세요. 이 스킬은 일반 민사 청구의 요건사실 매트릭스입니다."

## 요건사실론·증명책임 — 프레임워크

**요건사실(要件事實)** = 법률효과의 발생·변경·소멸에 필요한 법률요건에 해당하는 구체적 사실. 소송에서
당사자가 주장·증명해야 하는 대상이다. `[검토]` — 어느 사실이 요건사실인지는 실체법 해석 문제로 사안별.

**증명책임 분배(원칙):** 각 당사자는 자기에게 유리한 법률효과의 요건사실을 증명한다(법률요건분류설).
`[모델 지식 — 검증 필요]`
- **원고:** 권리근거규정의 요건사실(청구원인).
- **피고:** 권리장애·권리소멸·권리저지규정의 요건사실(항변 — 소멸시효·변제·상계·동시이행 등).

**주요사실 vs 간접사실:**
- **주요사실**(요건사실) — 법률요건에 직접 해당. 변론주의상 당사자가 주장해야 법원이 판단(주장책임).
- **간접사실** — 주요사실을 추인하게 하는 사실. 증거와 함께 주요사실의 존부를 추론하는 자료.
- 표에서 이 둘을 구분한다 — 주요사실은 주장·증명 모두, 간접사실은 증명의 자료.

> **증명책임 분배·요건사실 특정은 실체법 해석이다.** 개별 청구원인(계약·불법행위·부당이득 등)의
> 요건사실은 관련 실체법(민법 등)과 판례로 정해진다. 국가법령정보에서 근거 실체법·절차법을 대조하고,
> 요건사실 특정은 `[검토]`로 플래그한다 — 이 표가 요건사실을 확정하지 않는다.

## 워크플로우

### 1. 입장·청구원인/항변 확인

- **입장:** 원고(청구원인) / 피고(항변)?
- **청구원인 또는 항변:** 무엇에 대한 요건사실표인가? (예: 대여금 반환청구 / 불법행위 손해배상 /
  소유권이전등기 / 소멸시효 항변 / 상계 항변)
- **사건유형·심급:** 민사 원칙. 형사·행정·가사면 요건사실론이 그대로 적용되지 않을 수 있다 `[검토]`.

### 2. 요건사실 분해

청구원인(또는 항변)을 요건사실로 분해한다. 근거 실체법을 국가법령정보로 대조:

```
mcp__korean-law__get_law_text "민법 제598조"    # 예: 소비대차 (대여금)
mcp__korean-law__get_law_text "민법 제750조"    # 예: 불법행위
mcp__korean-law__get_law_text "민법 제162조"    # 예: 소멸시효 (항변)
mcp__korean-law__legal_analysis {"mode": "verify_citations", "citation": "민법 제750조"}
```

**침묵의 보완 금지(CLAUDE.md `## 공유 가드레일`).** 근거 실체법·절차법을 국가법령정보에서 확인하지 못하면
멈추지 않고 `[모델 지식 — 검증 필요]`로 태그하고 진행한다. 태그 없이 모델 지식을 확신처럼 제시하지 않는다.

각 요건사실을 표의 한 행으로. **요건사실 특정 자체가 `[검토]`** — 실체법 해석이므로 변호사가 확정한다.

### 3. 매핑

각 요건사실 행에 대해:
- **증명책임 주체** — 원고/피고 중 누가 이 사실을 증명하는가.
- **주요사실/간접사실** — 이 사실이 요건사실(주요사실)인지, 이를 추인하는 간접사실인지.
- **대응 증거** — 이 사실을 뒷받침하는 서증(갑/을 제N호증)·인증·간접사실. 각 셀 핀포인트.
- **상대방 예상 항변·부인** — 상대방이 이 요건사실을 어떻게 다툴지(부인/항변/부지).
- **상태** — 증거 있음(supported) / 약함(weak) / 증거 없음·공백(gap).

### 4. 증거 공백 목록 (최우선 산출물)

**상태가 gap(증거 없음)·weak(약함)인 요건사실을 별도 목록으로 뽑는다.** 이것이 이 표의 최우선 산출물 —
"이 청구를 세우려면(이 항변을 방어하려면) 무엇이 더 필요한가". 각 공백에:
- 어느 요건사실이 비었는지,
- 어떤 증거가 필요한지(서증·증인·감정·사실조회),
- 어떻게 확보할지(증인신문·문서제출명령·사실조회 — `/litigation-legal:subpoena-triage`,
  증거보전 — `/litigation-legal:evidence-preservation` 라우팅).

## 이 스킬이 하지 않는 것

- 요건사실을 확정한다 — 실체법 해석은 `[검토]`, 변호사 영역.
- 증명책임 분배를 단정한다 — 원칙만 제시, 사안별은 `[검토]`.
- 청구가 인용될지·항변이 유효할지 예측한다.
- 특허 청구항 대비·균등론을 한다 — ip-legal 소관.
- 증거를 지어낸다 — 없으면 증거 공백으로 surface.
- 증거의 증거능력·증명력을 판정한다 (법원 영역).
