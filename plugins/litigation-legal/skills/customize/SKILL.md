# /customize

1. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 읽기.
   - 없거나 populate 안 됨: "먼저 `/litigation-legal:cold-start-interview`를 실행하세요."
2. `$ARGUMENTS`에서 변경 대상 섹션·항목 파악.
3. 현재 값 보여주고 새 값 묻기.
4. 변경 사항 확인 후 파일 업데이트.
5. 관련 스킬이 이 변경에 따라 어떻게 달라지는지 한 줄 요약.

---

## 변경 가능한 항목

### 실무환경 (`## 우리는 누구인가`)

**무엇이 바뀌나:** 에스컬레이션 routing 방향, 매터 격리 필요 여부, work-product 헤더.

```
현재: [사내 송무 / 법무법인 송무 / 개인·단독 / 나홀로소송]
새 값: ?
```

변경 시:
- `사내 송무` → 매터 워크스페이스 off 권장, 에스컬레이션은 GC·법무팀장 방향.
- `법무법인 송무` / `개인·단독` → 매터 워크스페이스 on 권장, 의뢰인별 격리.
- `나홀로소송 당사자` → 산출물 헤더 `리서치 노트 — 법률자문이 아님. 제출 전 변호사 검토 필수`로 전환.

### 역할 (`## 누가 이 플러그인을 사용하나`)

**무엇이 바뀌나:** work-product 헤더, 서면 framing, 법적 결과 단계 전 체크포인트.

```
현재: [소송대리 변호사 / 사내 법무 / 나홀로소송 당사자 / 기타]
새 값: ?
```

역할별 헤더:
- **소송대리 변호사:** `대외비 · 변호사·의뢰인 비밀유지 대상 — 소송대리 변호사의 법률자문·소송수행 목적 (변호사법 §26)`
- **사내 법무:** `내부 검토 자료 — 소송대리 변호사의 검토 필요. 압수수색·문서제출 시 보호 여부는 사안별`
- **나홀로소송 당사자:** `리서치 노트 — 법률자문이 아님. 제출 전 변호사 검토 필수`

### 주력 분야 (`## 소송 플레이북 / 사건 관리 원칙` → 사건유형별 접근)

**무엇이 바뀌나:** 해당 사건유형의 검토 트리거·에스컬레이션·하우스 입장.

변경 가능한 유형:
- 민사(원고) / 민사(피고)
- 형사(고소·고발) / 형사(피의자·피고인)
- 행정(항고소송)
- 가사
- 보전(가압류·가처분)
- 집행
- 조정·중재

각 유형에 대해:
```
현재 하우스 입장: [PLACEHOLDER 또는 기존 값]
새 입장: ?
자동 에스컬레이션 트리거: ?
```

> **주의:** 형사(피의자·피고인) 플레이북 변경 시 — 자백·진술 관련 전략은 **특히 신중**. 하우스 입장으로 "자백 권유"를 설정하지 않는다.

### 에스컬레이션 임계점 (`## 에스컬레이션`)

**무엇이 바뀌나:** matter-intake·matter-update·matter-briefing이 에스컬레이션 노트를 자동 생성하는 기준.

```
현재 임계점:
- 소가 금액: [N원 이상 → [담당자]]
- 자동 에스컬레이션: [불변기간 임박 / 형사 피소 / 보전처분 / 상소 여부 결정]
새 값: ?
```

**임계점과 무관하게 자동 에스컬레이션 (고정 — 변경 불가):**
- 불변기간 7일 이내 임박 (상소기간·즉시항고·행정소송 제소기간)
- 패소·일부패소 판결 선고 후 상소 여부 결정
- 형사 구속·압수수색 집행
- 가압류·가처분 신청 또는 피신청

> "이 네 가지는 실무 프로파일 설정과 무관하게 항상 에스컬레이션됩니다. 소송에서 기한도과·형사 강제처분은 복구 불가 — 이 트리거는 고정입니다."

### 기한 알림 리드타임 (`## 기한 트래커 설정`)

**무엇이 바뀌나:** deadline-watcher 에이전트가 불변기간 도래 전 몇 일에 알림을 보내는지.

```yaml
현재:
  lead_times_days: [14, 7, 3, 1]

새 값: ?
```

> **불변기간은 도과 시 복구 불가.** 리드타임을 줄이는 것은 권장하지 않습니다. 1일 알림은 최소값으로 유지하세요.

### 하우스 스타일 (`## 하우스 스타일` — 있는 경우)

**무엇이 바뀌나:** 브리핑·메모 길이, 의뢰인 보고 포맷, 서면 선례 포맷.

```
현재: [간결 / 상세 / 불릿 중심 / 서술형]
새 값: ?
```

### 매터 워크스페이스 설정 (`## 매터 워크스페이스`)

**무엇이 바뀌나:** 다중 의뢰인 격리 활성화 여부, 활성 매터.

```
현재:
  활성화: [✓/✗]
  활성 매터: [slug 또는 없음]
  매터 간 컨텍스트: [on/off]

새 값: ?
```

> **법무법인·단독 사무소 → 활성화 ✓ 권장.** 사내 사용자 → ✗ 유지.
> **형사·보전 사건이 포함된 경우 매터 간 컨텍스트는 항상 off 권장** — 상대방 비밀·정보장벽.

### 국가법령정보 연결 상태 (`## 연결 가능한 통합`)

**무엇이 바뀌나:** 인용 검증 태그 수준 (✓ vs `[모델 지식 — 검증 필요]`).

MCP 연결 재점검: `/litigation-legal:cold-start-interview --check-integrations`

## 변경 실행 워크플로우

1. **현재 값 확인:** CLAUDE.md에서 해당 섹션 읽기.
2. **새 값 제시:** 사용자에게 현재 값 보여주고 새 값 묻기.
3. **충돌 점검:**
   - 역할 변경이 work-product 헤더와 충돌하지 않는지.
   - 에스컬레이션 임계점 변경이 자동 트리거와 충돌하지 않는지.
   - 실무환경 변경이 매터 워크스페이스 설정과 정합하는지.
4. **변경 확인:** "아래를 변경합니다 — 맞나요?"
5. **CLAUDE.md 업데이트:** 해당 섹션만 편집. 다른 섹션은 건드리지 않는다.
6. **영향 요약:** "이 변경으로 [스킬 X]가 [이렇게] 달라집니다."

---

## 이 스킬이 하지 않는 것

- **전체 재인터뷰.** 단일 항목 변경만. 전체 재실행은 `/litigation-legal:cold-start-interview --redo`.
- **자동 에스컬레이션 트리거 비활성화.** 불변기간 임박·형사 강제처분·패소 판결은 고정.
- **사건유형별 플레이북 전체 재구성.** 한 유형의 입장·임계점만 변경. 전체는 `--redo`.
- **변경 없이 파일 저장.** 변경 내용이 기존 값과 동일하면 "변경 사항 없음"으로 종료.
