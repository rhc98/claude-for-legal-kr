# /matter-intake

1. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 로드 → 리스크 캘리브레이션(트리아지용), 이해상충 확인 방법, 에스컬레이션 체인.
2. 아래 워크플로우와 참조표를 따른다.
3. 통일 인테이크 실행: 사건 식별, 이해상충 점검, 사건유형·심급 확인, 리스크 트리아지, 소가·담보·인지, 외부 대리인, 내부 담당자, 기일·기한, 초기 입장.
4. Slug 생성(사건명 기반 — 소문자·하이픈·연도, 예: `2026-갑을건설-공사대금`).
5. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/matters/[slug]/matter.md` 작성 — 서술형 인테이크 전체.
6. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/matters/[slug]/history.md` 작성 — 인테이크를 첫 항목으로 seed.
7. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/matters/_log.yaml`에 구조화 행 추가.
8. 사용자 확인: "아래 행을 씁니다 — 수정할 것 있나요?"

## 인테이크 워크플로우

### 1단계 — 사건 식별

- **사건명(우리 쪽 호칭):** 짧고 기억하기 쉬운 내부 이름.
- **당사자:** 우리 측(원고·피고·신청인·피신청인·고소인·피의자 등) + 상대방 전원.
- **법원·기관:** 관할 법원·검찰·경찰·중재기관 등.
- **사건번호:** 있으면 기재. 미확정이면 공란.
- **사건유형·심급:** 아래 `## 사건유형·심급 참조표` 확인 — **침묵으로 가정하지 않는다.** 모르면 묻는다.

### 2단계 — 이해상충 점검

> **⚠️ 이해상충 점검은 이 스킬의 게이트다.** 실무 프로파일(`CLAUDE.md`)의 점검 방법을 따른다. 방법이 없으면:
> 1. 상대방 전원을 기존 활성·아카이브 매터의 당사자·상대방과 비교.
> 2. 충돌 발견 시 **즉시 플래그**: "[다른 매터]에서 이 상대방을 대리/맞서고 있음. 이해상충 점검 필요 — 변호사 확인 전 인테이크 중단."
> 3. 충돌 없으면: "이해상충: 기존 매터와 당사자 중복 없음." 기록.

다중 의뢰인 실무(법무법인·단독 사무소)만 해당. 사내 사용자(한 회사만 다루는 경우)는 이 점검을 간략화할 수 있음.

### 3단계 — 사건유형·심급 확정

**사건유형 + 심급** 조합이 이후 모든 분석의 게이트다. 아래 참조표에서 확정:

| 사건유형 | 우리 입장 | 심급 | 근거 절차법 |
|---|---|---|---|
| 민사 | 원고 / 피고 | 1심 / 항소심 / 상고심 | 민사소송법 |
| 형사 | 고소·고발인 / 피의자·피고인 | 수사 / 1심 / 항소심 / 상고심 | 형사소송법 |
| 행정 | 원고(처분취소·무효확인 등) / 피고(행정청) | 1심 / 항소심 / 상고심 | 행정소송법 |
| 가사 | 원고·청구인 / 피고·상대방 | 1심 / 항소심 | 가사소송법 |
| 보전 | 채권자·신청인 / 채무자·피신청인 | — (보전처분) | 민사집행법 |
| 집행 | 채권자 / 채무자 | — (강제집행) | 민사집행법 |
| 조정·중재 | 신청인 / 피신청인 | — | 중재법·민사조정법 |
| 상소심 | (1심 입장 승계) | 항소심 / 상고심 | 민사소송법 §390~ |

> **같은 사실도 원고냐 피고냐에 따라 분석 방향이 전혀 다르다.** 인테이크에서 입장을 명시한다.

### 4단계 — 관할·소가·준거법

- **관할:** 합의관할·법정관할·전속관할 여부 확인 `[모델 지식 — 검증 필요]`. 민사소송법 §2~§25 `[검증 필요]`. 관할 위반이면 본안 전 항변 포인트.
- **소가·인지:** 소가 산정 방식은 사건유형별로 다름. **요율·구체 금액은 하드코딩하지 않는다** — 민사소송등인지법 현행 고시 대조 필요 `[검증 필요]`. "소가 [N]원이면 인지 대략 [X]원"은 제시하되 반드시 `[기간 계산 — 1차 소스 대조 검증 필요]` 플래그.
- **준거법:** 외국 요소(외국 당사자·외국에서의 이행·외국법 약정) 있으면 감지, `[외국법·국제사법 — 외국 변호사 검증 필요]` 플래그.

```
# 관할 검증 예시
mcp__korean-law__get_law_text "민사소송법 제2조"
mcp__korean-law__legal_analysis {"mode": "verify_citations", "citation": "민사소송법 제36조"}
```

### 5단계 — 리스크 트리아지

실무 프로파일의 리스크 캘리브레이션 적용. 없으면 아래 기준 사용:

| 리스크 | 기준 |
|---|---|
| 🔴 차단·긴급 | 불변기간 임박(상소기간·즉시항고기간·행정소송 제소기간 등), 보전처분 긴급 집행, 구속·압수수색 대응, 기한 도과 시 권리 소멸 |
| 🟠 높음 | 대규모 소가, 형사 피소, 가압류·가처분 신청·피신청 |
| 🟡 중간 | 일반 민사분쟁, 행정심판·항고소송, 가사 조정 |
| 🟢 낮음 | 소액사건, 내용증명 대응, 조정·화해 기대 |

**기한도과는 언제나 🔴.** 불변기간(상소기간·즉시항고·행정소송 제소기간) 임박 여부를 먼저 확인한다.

#### 이중 심각도(dual severity)

실무 프로파일 `## 이중 심각도` 기준:
- **법적 위험:** 승패·권리 소멸·불변기간에 미치는 영향
- **분쟁·관계 마찰:** 상대방·의뢰인 관계, 화해·조정 여지

### 6단계 — 외부 대리인·내부 담당자

- **외부 대리인(OC):** 담당 변호사·사무소, 선임 여부, 위임장 여부.
- **내부 담당자:** 송무팀 담당자, 에스컬레이션 상위 라인.
- **소송구조:** 자력 부족 당사자인 경우 소송구조(민사소송법 §128) `[모델 지식 — 검증 필요]`·대한법률구조공단 안내.

### 7단계 — 기일·기한 등록

아래 기한은 비교적 안정적이나 **기산점·불변기간 여부·추후보완 가부는 사안별** — 항상 `[기간 계산 — 1차 소스 대조 검증 필요]` 플래그.

| 기한 유형 | 기간 | 기산점 | 근거 |
|---|---|---|---|
| 항소·상고기간 | 2주 | 판결정본 송달일 | 민사소송법 §396·§425 `[검증 필요]` |
| 즉시항고 | 1주 | 재판 고지일 | 민사소송법 §444 `[검증 필요]` |
| 지급명령 이의 | 2주 | 지급명령 송달일 | 민사소송법 §470 `[검증 필요]` |
| 행정소송 제소기간 | 90일 | 처분 안 날 | 행정소송법 §20 `[검증 필요]` |
| 추후보완 | 2주 | 사유 없어진 후 | 민사소송법 §173 `[검토]` |

> **날짜가 제공되지 않으면 기간을 산정하지 않는다.** 공식과 필요한 정보(송달일·정본 수령일)를 제시하고 사용자에게 확인 요청.
>
> **전자소송·나의사건검색 기일 정보는 공개 API 없음** — 사용자가 기일통지서·판결정본을 paste해야 함 `[전자소송/나의사건검색 — 사용자 확인 필요]`.

```
# 기간 조문 검증 예시
mcp__korean-law__get_law_text "민사소송법 제396조"
mcp__korean-law__legal_analysis {"mode": "verify_citations", "citation": "행정소송법 제20조"}
```

### 8단계 — 초기 입장

- **우리 측 입장:** 원고 청구원인·요건사실 특정 / 피고 항변 방향 / 형사 방어전략·소명 / 행정처분 취소 사유 등.
- **소명·증거 초기 평가:** 있는 자료 수준(소장·증거·판결정본·기일통지서) 및 부족한 것.
- **에스컬레이션 여부:** 실무 프로파일 `## 에스컬레이션` 임계점 대조 — 형사 피소·보전처분·불변기간 임박은 자동 에스컬레이션.

## 사건유형·심급 참조표

**각 조합의 절차법·서면·기한은 다르다.** 스킬은 아래 조합을 확정한 뒤 해당 절차를 적용한다. 인테이크에서 확정되지 않으면 substantive 분석 전 묻는다.

| 조합 | 핵심 절차 | 주요 기한 |
|---|---|---|
| 민사 1심(원고) | 소장 → 변론준비·변론 → 판결 | 재판기일, 준비서면 제출기한 |
| 민사 1심(피고) | 답변서 → 변론 → 판결 | 답변서 제출, 변론기일 |
| 민사 항소심 | 항소장·항소이유서 → 변론 | **항소기간 2주**(판결정본 송달일) `[검증 필요]` |

## 데이터 갭 — 침묵의 보완 금지 (세 값)

조문 전체 텍스트·기간·기산점·판례 유효성·사건현황·기일이 부족할 때 답은 세 개다:

1. **플래그와 함께 보완.** 모델 지식·다른 소스에서 가져오되 항목에 태그(`[모델 지식 — 검증 필요]`)를 붙이고 진행.
2. **말하지 않고 멈춤.** 1차 소스(판결정본·기일통지서·소장)를 붙여 달라거나 가리켜 달라고 부탁하고, 줄 때까지 진행하지 않음.
3. **플래그만 — 사용하지 않음.** 분석을 바꾸지 않지만 독자가 알아야 하는 정보를 `[모델 지식 — 검증 필요]` 태그로 caveat에 surface하되, 그것으로 분석을 바꾸지 않음.

이미 알고 있는 의심을 침묵하는 것은 확신에 찬 단언만큼이나 오도한다.

## 이 스킬이 하지 않는 것

- **이해상충 분석 실행.** 점검은 사용자가 선언한 당사자를 기존 매터와 비교하는 것 — 법적 충돌 판단은 변호사 몫.
- **소가·인지 하드코딩.** 요율은 민사소송등인지법 현행 고시에 따르고, 스킬은 산정 공식만 제시하며 결과에 `[검증 필요]` 플래그.
- **기산점·불변기간 단정.** 날짜 미확인 시 산정하지 않고 공식과 필요 정보를 제시.
- **사건현황·기일 조회.** 전자소송·나의사건검색은 공개 API 없음 — 사용자가 기일통지서·판결정본을 paste해야 함.
