# /matter-close

1. `~/.claude/plugins/config/claude-for-legal-kr/litigation-legal/CLAUDE.md` 로드 →
   에스컬레이션 체인, 상소 결정 권한.
2. 아래 워크플로우와 참조표를 따른다.
3. `$ARGUMENTS`에서 slug 추출 — 없으면 활성 매터 사용(없으면 묻는다).
4. `matters/<slug>/matter.md` + `matters/<slug>/history.md` 로드.
5. 종결 인터뷰 실행.
6. `history.md`에 종결 항목 추가, `_log.yaml` 갱신(`status: closed`), 매터 아카이브.

사건이 끝난다. 결과는 포트폴리오가 생성하는 가장 가치 있는 데이터 포인트 —
미래 사건의 리스크 프레임워크를 캘리브레이션한다. 사건을 종결하면 구조적으로
결과를 포착해 기록이 아카이브에 머물지 않고 유용하게 된다.

## 종결 인터뷰

### 1단계 — 종결 유형 확인

종결 유형은 아래 중 하나. 유형마다 확인할 항목이 다르다.

| 종결 유형 | 근거 | 확인 항목 |
|---|---|---|
| 판결 확정 | 민사소송법 §498 `[검증 필요]` | 판결정본 수령일, 상소기간 도과 또는 상소포기 확인 |
| 재판상화해·조정성립 | 민사소송법 §220·§231 `[검증 필요]` | 화해·조정 조서 확정일, 이행 조건 |
| 소취하 | 민사소송법 §266 `[검증 필요]` | 취하 날짜, 상대방 동의 여부(소송계속 후) `[검토]`, 재소 제한 여부 |
| 항소·상고 포기 | 민사소송법 §393·§425 `[검증 필요]` | 포기서 제출일, 상대방 포기 여부 |
| 상소기간 도과 | 민사소송법 §396·§425 `[검증 필요]` | 판결정본 송달일 + 2주 경과 확인, 추후보완 여부 `[검토]` |
| 불기소·기소유예(형사) | 형사소송법 §255 `[검증 필요]` | 불기소 처분 통지일, 항고·재정신청 여부 |
| 무죄·유죄 확정(형사) | 형사소송법 §383~ `[검증 필요]` | 판결 확정일, 상소 여부 |
| 중재판정 확정 | 중재법 §35·§38 `[검증 필요]` | 판정 수령일, 취소소송 제기 여부 검토 `[검토]` |
| 행정심판 재결 확정 | 행정심판법 §49 `[검증 필요]` | 재결 통지일, 항고소송 제기 여부 |
| 집행 완료 | 민사집행법 §39~ `[검증 필요]` | 집행 완료일, 청구이의 여부, 배당 완료 |
| 합의 종결 | — | 합의서 서명일, 부제소합의 여부 `[검토]`, 이행 조건 |
| 기타 | — | 사유 기술 |

### 2단계 — 상소기간·불변기간 도과 확인

> **상소기간 도과는 종결의 게이트다.** 아직 상소기간이 남아 있는데 종결하면 권리를 포기하는 것.
> 반드시 확인 후 종결.

```
# 상소기간 조문 검증
mcp__korean-law__get_law_text "민사소송법 제396조"
mcp__korean-law__legal_analysis {"mode": "verify_citations", "citation": "민사소송법 제498조"}
```

| 상소 유형 | 기간 | 기산점 | 불변기간 | 태그 |
|---|---|---|---|---|
| 항소 | 2주 | 판결정본 **송달일** | 예 | `[기간 계산 — 1차 소스 대조 검증 필요]` |
| 상고 | 2주 | 판결정본 **송달일** | 예 | `[기간 계산 — 1차 소스 대조 검증 필요]` |
| 즉시항고 | 1주 | 재판 **고지일** | 예 | `[기간 계산 — 1차 소스 대조 검증 필요]` |
| 추후보완 | 2주 | 사유 없어진 후 | — | `[검토]` — 인정 여부 다툼 多 |

> **날짜가 없으면 기간을 산정하지 않는다.** 공식과 필요한 정보(판결정본 수령일)를 제시하고
> 사용자에게 확인 요청. `[전자소송/나의사건검색 — 사용자 확인 필요]`.

상소기간이 아직 남아 있으면:
> "판결정본 수령일로부터 아직 [N]일이 남아 있습니다. 상소 여부를 결정하셨나요?
> 상소포기서 제출 또는 기간 도과 확인 후 종결을 권장합니다.
> 지금 종결하면 상소권을 포기하는 것과 같습니다. 계속 진행하시겠습니까? `[검토]`"

### 3단계 — 결과 포착

- **최종 결과:** 승소·패소·일부승소·화해·취하·불기소 등 구체적 내용.
- **최종 노출·회수:** 소가 대비 실제 인용·집행 금액 또는 형사 선고 결과.
- **비용:** 소송비용 확정 여부, 변호사 보수 산정 여부 `[검토]`.
- **소송비용 확정:** 판결 확정 후 소송비용 확정 신청 여부 검토 `[모델 지식 — 검증 필요]`.
- **집행권원 확보:** 승소한 경우 집행문 부여·강제집행 계획 여부.

### 4단계 — 교훈 기록

> **교훈은 포트폴리오의 기억이다.** 사용자가 교훈 섹션을 skip하면 빈칸으로 두지 발명하지 않는다.

- **잘 된 것:** 전략·증거·절차 중 효과적이었던 것.
- **개선할 것:** 다음에 달리 할 것.
- **예상 외 발견:** 상대방 전략·판례 해석·절차 포인트 중 예상과 달랐던 것.
- **향후 유사 사건 주의 사항:** 리스크 캘리브레이션 업데이트 제안.

### 5단계 — 레거시 보유 항목

종결 후에도 유지해야 할 사항:
- **판결정본·화해조서 보관:** 집행 또는 이해상충 점검 목적.
- **이해상충 아카이브:** 종결 매터의 당사자는 향후 이해상충 점검 시 검색 대상.
- **재소 제한(소취하 후):** 소취하 후 동일 소 재소 제한 여부 `[검토]` — 민사소송법 §267 `[검증 필요]`.
- **집행권원 보존:** 집행 계획이 있는 경우 집행권원·집행문 보관.

## 종결 처리

### `history.md` 종결 항목

```markdown
## [YYYY-MM-DD] — 사건 종결

**종결 유형:** [판결 확정 | 재판상화해 | 소취하 | 상소포기 | 상소기간 도과 | 불기소 | 중재판정 확정 | 합의 종결 | 기타]
**최종 결과:** [구체적 결과 — 승소·패소·화해 금액 등]
**최종 노출·회수:** [소가 대비 인용 금액 또는 비산정]
**상소기간 확인:** [도과 확인 YYYY-MM-DD | 상소포기서 제출 | 판결 확정 날짜]

**교훈:**
- 잘 된 것: [또는 빈칸 — 사용자가 skip한 경우]
- 개선할 것: [또는 빈칸]
- 향후 주의 사항: [또는 빈칸]
```

### `_log.yaml` 갱신

```yaml
status: closed
closed: "[YYYY-MM-DD]"
outcome: "[승소 | 일부승소 | 패소 | 화해 | 취하 | 불기소 | 기타]"
final_exposure: "[금액 또는 null]"
last_updated: "[YYYY-MM-DD]"
```

### 아카이브

`matters/<slug>/` → `matters/_archived/<slug>/` 이동.
활성 매터였으면 CLAUDE.md `활성 매터:`를 `없음 — 실무 수준 컨텍스트만`으로 설정.

## 이 스킬이 하지 않는 것

- **매터 삭제.** 종결된 매터는 `_log.yaml`과 디스크에 남음 — 포트폴리오 판단의 훈련셋.
- **재개.** 종결된 사건이 돌아오면(상소, 관련 소송) 새 매터를 열고 종결 매터 slug를 `matter.md`에 참조.
- **교훈 발명.** 사용자가 교훈 섹션을 skip하면 빈칸으로 둠.
- **상소기간 단정.** 기산점(판결정본 수령일)이 확인되지 않으면 산정하지 않음.
- **소송비용 확정 절차 대행.** 확정 신청 여부를 안내하되 절차 판단은 변호사 몫 `[검토]`.
